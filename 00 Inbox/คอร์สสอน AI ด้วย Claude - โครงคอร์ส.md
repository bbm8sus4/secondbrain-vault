# คอร์สสอน AI ด้วย Claude — โครงคอร์ส

## ข้อมูลพื้นฐาน
- **กลุ่มเป้าหมาย:** พนักงานออฟฟิศทั่วไป (non-technical)
- **สื่อหลัก:** Claude บน Warp terminal (Claude Code)
- **เป้าหมายผู้เรียน:** ใช้ Claude ทำงานประจำ / เขียน prompt เก่ง / สร้าง automation / เข้าใจ AI ภาพรวม

## จุดที่ต้องระวัง
- พนักงานออฟฟิศกลัว terminal เป็นอันดับหนึ่ง → ห้ามเปิดคอร์สด้วย Warp ทันที
- ทางออก: สร้างความมั่นใจก่อน แล้วค่อยพาเข้า terminal ตอนเห็นคุณค่าแล้ว
- terminal = จุดขาย เพราะทำให้เขาสร้าง automation จริง ไม่ใช่แค่แชทเล่น

## โครงคอร์ส (4 Module เรียงเป็นบันได)
> เรียง "ความกลัวน้อยสุด → คุณค่ามากสุด" — ทุก Module จบด้วยของที่ใช้ได้พรุ่งนี้

### Module 1 — เข้าใจ AI กว้างๆ
- AI ทำอะไรได้/ไม่ได้ ในภาษาคนทำงาน
- ลองใช้ Claude บนเว็บ/แอปก่อน (ยังไม่แตะ terminal)

### Module 2 — เขียน prompt ให้เก่ง
- โครงสร้าง prompt: บทบาท + บริบท + ตัวอย่าง + รูปแบบผลลัพธ์
- ใบงานจากงานจริง: อีเมล, สรุปประชุม, รายงาน

### Module 3 — ใช้ Claude ทำงานประจำ
- **จุดเปลี่ยน:** พาเข้า Warp + Claude Code ตรงนี้
- เริ่มจากงานเดียวที่ทำซ้ำทุกวัน ให้เห็นผลทันที

### Module 4 — สร้าง automation/workflow
- ต่อยอดจาก Module 3 เป็น workflow หลายขั้น
- ปลายทาง: กลับไปทำเองที่ออฟฟิศได้

## Project Instructions (paste ลง Claude Project ได้เลย)
```
คุณคือผู้ช่วยออกแบบและสอนคอร์ส AI สำหรับพนักงานออฟฟิศทั่วไป (non-technical)
สื่อหลักที่ใช้สอนคือ Claude บน Warp terminal (Claude Code)

หลักการ:
- อธิบายด้วยภาษาคนทำงาน ห้ามศัพท์เทคนิคโดยไม่จำเป็น
- ทุกบทเรียนต้องจบด้วยตัวอย่างใช้งานจริงที่เอาไปทำได้ทันที
- ลำดับการสอน: เข้าใจ AI → เขียน prompt → ใช้ทำงานประจำ → สร้าง automation
- ผู้เรียนกลัว terminal ให้สร้างความมั่นใจก่อนเสมอ

Output ที่ต้องการ: outline บทเรียน, สคริปต์สอน, ใบงาน, ตัวอย่าง prompt
```
