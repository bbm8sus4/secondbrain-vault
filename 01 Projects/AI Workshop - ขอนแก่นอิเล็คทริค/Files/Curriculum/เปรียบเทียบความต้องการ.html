ผู้เข้าอบรมต้องการอะไร และพร้อมแค่ไหน

สรุปจากแบบสำรวจผู้เข้าอบรมจริง เพื่อใช้ออกแบบหลักสูตรให้ตรงกับความต้องการและระดับพื้นฐานของผู้เรียน

บริษัท ขอนแก่นอิเล็คทริค (ระบบไฟ · โซลาร์เซลล์) ผู้เข้าอบรม 8 คน (การตลาด 7 · ช่าง 1) หลักสูตร Claude · ทำเว็บ · ฐานข้อมูล · GitHub

บทสรุปผู้บริหาร

ผู้เรียนตั้งเป้าสูง แต่พื้นฐานยังเป็นผู้เริ่มต้น

ผู้เข้าอบรมอยากทำได้ถึงขั้นสร้างเว็บและเชื่อม AI กับข้อมูลเองได้ แต่พื้นฐานส่วนใหญ่ยังเป็นผู้เริ่มต้น — หลักสูตรต้องออกแบบให้รองรับช่องว่างนี้

กลุ่มผู้เรียนเป็น สายการตลาด 7 ใน 8 คน งานที่อยากให้ AI ช่วยมากที่สุดคือ สร้างภาพประกอบ · คอนเทนต์โซเชียล · ทำเว็บ/landing page และตั้งเป้าสูง — 6 ใน 8 คนอยากทำเว็บ deploy เองได้, 5 คนอยากเชื่อม AI กับ Excel/ฐานข้อมูล

แต่ความเป็นจริงคือส่วนใหญ่เป็น ผู้เริ่มต้น: 5 ใน 8 คนไม่เคยเขียนโค้ด · ไม่มีใครใช้ Claude เป็นประจำ · ความมั่นใจในการใช้ AI เฉลี่ย 2.6/5 · 6 ใน 8 คน “กังวลว่าจะตามไม่ทัน” · และ 5 ใน 8 คนติดตั้งโปรแกรมเองไม่ได้หรือไม่แน่ใจ

ทางออก: แบ่งระดับผลลัพธ์ (tiered) · เน้น use case การตลาด ที่ตรงกับงานจริงของผู้เรียน · และ เตรียมเครื่อง/บัญชีก่อนวันอบรม

ข้อเสนอเพื่อปิด gap

  1. แบ่งระดับผลลัพธ์ (tiered) แทนการบังคับทุกคนเท่ากันCore track (ทุกคนทำได้จริง) = ใช้ Claude + คอนเทนต์ + landing page แบบ no-/low-code · Advanced track (ต่อ, นัท) = deploy จริง + ฐานข้อมูล + GitHub
  2. โฟกัส use case การตลาด เป็นแกนหลักคอนเทนต์ · ภาพประกอบ · landing page โปรโมชัน — ตรงกับงานจริงของผู้เรียน 7/8 ใช้เป็นโจทย์จริงตลอดคลาส
  3. ลดฐานข้อมูล + GitHub เป็น “เข้าใจภาพรวม + ดู demo”สำหรับคนส่วนใหญ่ ไม่ใช่ทำเอง — ไม่ใช่ความต้องการหลักของผู้เรียน วางเป็นเป้าระยะถัดไป ไม่ใช่จบใน 1 วัน
  4. เตรียมก่อนวันงาน (สำคัญ)ติดตั้ง Claude/Git/Node ล่วงหน้า (5/8 ติดปัญหาสิทธิ์) · เปิดบัญชี Claude ให้ครบ · แจก cheat sheet ศัพท์อังกฤษ/error
  5. จัดการความกังวล “เรียนไม่ทัน” (6/8)เดินช้า มีจุดพัก · จับคู่ buddy (ต่อ/นัท ช่วยเพื่อน) · ตั้งผลลัพธ์เป็นขั้น ให้แต่ละคนก้าวตามจังหวะของตัวเอง
  6. 1 วันอาจไม่พอสำหรับ “deploy + ฐานข้อมูล ทำเองได้จริงทุกคน”ทางเลือก — ลดขอบเขตให้ลึกเฉพาะที่ใช้จริง หรือเพิ่ม follow-up clinic 1 รอบหลังอบรม
