# Part 1
## พื้นฐาน Claude

**AI Workshop · บริษัทขอนแก่นอิเล็คทริค**
09:30 – 10:45 · 75 นาที

---

## จบชั่วโมงนี้ ทุกคนจะทำอะไรได้

1. เปิด Claude ใช้งานเป็นครั้งแรก (หรือถูกวิธี ถ้าเคยเปิดแล้ว)
2. สั่งงานให้ Claude ช่วยเรื่องงานประจำได้
3. เขียน prompt ที่ได้ผลดีตั้งแต่ครั้งแรก ไม่ต้องลองผิดลองถูกนาน
4. รู้ว่าอะไรควรใช้ Claude อะไรไม่ควร

<div class="ok">

**สิ่งที่จะได้ติดมือกลับไป:** prompt 3 อันที่ใช้กับงานตัวเองได้เลย

</div>

## ก่อนไป — Claude คืออะไร ในภาษาคน

- เป็น "ผู้ช่วยที่พิมพ์คุยด้วยได้" คล้าย ChatGPT
- เก่งเรื่อง **เขียน · สรุป · แปล · หาเหตุผล · ค่อยๆ คิด**
- ไม่ใช่กูเกิล ไม่ได้ค้นเน็ตทันที (โดยพื้นฐาน) — เก่งเรื่อง "คิดจากที่คุณบอก"
- อ่านไทยได้เข้าใจ พิมพ์กลับเป็นไทยได้ ไม่ต้องแปลก่อน

> **จำไว้คำเดียว:** Claude คือ "เพื่อนร่วมงานที่อ่านเก่ง เขียนไว ไม่รู้จักบริษัทเรา" — เราต้องเล่าให้ฟังก่อน

---

## ทำไมต้อง Claude (ไม่ใช่ ChatGPT / Gemini)

<div class="tag">เลือกอันเดียวก่อน</div>

- **เขียน/สรุปภาษาไทยได้เนียนกว่าค่าเฉลี่ย** โดยเฉพาะภาษาธุรกิจ
- **อ่านไฟล์ยาวได้เยอะ** — ใส่เอกสาร 50 หน้าให้สรุปได้ในทีเดียว
- **ไม่เดาสุ่มเวลาไม่รู้** — จะบอกว่า "ไม่แน่ใจ" มากกว่าคู่แข่ง
- **ตัวฟรีก็ใช้งานได้จริง** — พอสำหรับงานส่วนใหญ่ในองค์กร

> ทุกตัวคล้ายกัน 80% เลือกอันไหนก็เริ่มได้ วันนี้เราเริ่มที่ Claude เพราะมี app ให้ใช้ทันที

## เปิดใช้ครั้งแรก — 3 ขั้น

1. เปิดเบราว์เซอร์ ไปที่ **claude.ai**
2. Sign up ด้วยอีเมล Google ของบริษัท (คลิก "Continue with Google")
3. เจอหน้าจอ chat — พร้อมใช้งาน

<div class="warn">

**ระวัง:** อย่าเผลอสมัครด้วยอีเมลส่วนตัว ถ้าจะใช้กับงานบริษัท

</div>

## หน้าจอ Claude — 4 ส่วนที่ต้องรู้

| ส่วน | อยู่ตรงไหน | ใช้ทำอะไร |
|---|---|---|
| **กล่องพิมพ์** | ล่างสุด | พิมพ์คำสั่ง/คำถาม |
| **แชทเก่า** | แถบซ้าย | ย้อนดูงานที่เคยทำ |
| **New chat** | ปุ่มบนซ้าย | เริ่มเรื่องใหม่ (แยกเรื่อง = ผลลัพธ์ดีกว่า) |
| **แนบไฟล์** | คลิปหนีบในกล่องพิมพ์ | ให้อ่านเอกสาร/รูป/ตาราง |

> **เคล็ดลับ:** งานคนละเรื่อง = แชทคนละอัน ไม่งั้น Claude จะสับสน

## ลองเลย 1 — พิมพ์ทักทาย (3 นาที)

พิมพ์ประโยคนี้ในกล่องแชท:

```
สวัสดี ฉันชื่อ [ชื่อคุณ] ทำงานตำแหน่ง [ตำแหน่ง]
ที่บริษัทขอนแก่นอิเล็คทริค วันนี้อยากให้ช่วยเรื่องงาน
```

แล้วดูว่า Claude ตอบว่ายังไง

<div class="ok">

**เป้าหมาย:** ทำให้ทุกคนเห็นว่ามันตอบเป็นภาษาไทยได้ + เริ่มไม่เกร็ง

</div>

---

## หลักการสั่งงานที่ได้ผล — 4 อย่างที่ต้องมี

<div class="tag">CRIT</div>

- **C — Context** เล่าว่าเราคือใคร บริษัทอะไร งานประเภทไหน
- **R — Role** ให้ Claude สวมบทบาท (เช่น "เป็นนักเขียน copy")
- **I — Instruction** สั่งให้ชัด ทำอะไร ผลลัพธ์แบบไหน
- **T — Target** ให้ตัวอย่าง / รูปแบบ / ความยาว

> **จำเป็นเมนู:** งานสั้นๆ ใช้ 1-2 อย่างพอ · งานสำคัญใส่ครบ 4

---

## แย่ vs ดี — เขียน caption Facebook

<div class="warn">

**Prompt แย่:**
"เขียน caption FB ให้หน่อย"

</div>

<div class="ok">

**Prompt ดี:**
"ช่วยเขียน caption Facebook สำหรับโปรโมชั่นเครื่องใช้ไฟฟ้าลด 30%
กลุ่มเป้าหมายคือแม่บ้านอายุ 35-50 ปี ภาคอีสาน
โทนอบอุ่น เข้าถึงง่าย ไม่ทางการ
ความยาว 2-3 บรรทัด ต้องมี CTA ให้คลิกลิงก์"

</div>

**ต่างกันตรงไหน?** → context + กลุ่มเป้าหมาย + โทน + ความยาว + CTA

---

## แย่ vs ดี — สรุปประชุม

<div class="warn">

**Prompt แย่:**
"สรุปให้หน่อย" (แล้ววาง transcript)

</div>

<div class="ok">

**Prompt ดี:**
"ช่วยสรุปประชุมนี้เป็น 3 ส่วน:
1. หัวข้อที่คุยและมติ (bullet)
2. งานที่มอบหมาย: ใคร ทำอะไร ภายในเมื่อไหร่
3. เรื่องที่ยังไม่ได้ข้อสรุป ต้องคุยรอบหน้า
ใช้ภาษาสุภาพ ไม่ยาวเกิน 1 หน้า A4"

</div>

**ต่างกันตรงไหน?** → บอกโครง output ชัด, ระบุความยาว, ระบุโทน

## Claude เก่ง 5 เรื่องนี้ในงานประจำ

| งาน | ตัวอย่างใช้จริง |
|---|---|
| **เขียน** | ร่างอีเมลตอบลูกค้า · เขียน caption · เขียน job description |
| **สรุป** | สรุป transcript ประชุม · สรุปเอกสาร PDF ยาว · สรุปแชทลูกค้า |
| **แปล** | ไทย↔อังกฤษ · เรียบเรียงใหม่ให้เนียนกว่า Google Translate |
| **จัดโครง** | ทำ outline บทความ · จัดหัวข้อ presentation · วางแผนงาน |
| **คิดร่วม** | ระดมไอเดียชื่อสินค้า · ตั้งชื่อแคมเปญ · หา angle การขาย |

---

## Claude ไม่เก่ง / ต้องระวัง 4 เรื่อง

<div class="warn">

- **ตัวเลข/บัญชี** — คำนวณผิดได้ ใช้ Excel ยังปลอดภัยกว่า
- **ข้อมูลปัจจุบัน** — ไม่รู้ข่าววันนี้ ราคาสินค้าล่าสุด สต๊อกจริง
- **ข้อมูลลูกค้า/ความลับ** — อย่าวาง เลขบัตร ปชช. พาสเวิร์ด สัญญาลับ
- **การตัดสินใจสำคัญ** — ใช้เป็นผู้ช่วยคิด อย่ายกให้ตัดสินใจแทน

</div>

> **กฎเหล็ก:** ถ้ามันตอบมาแล้ว "รู้สึกเก่งเกินไป" ให้ถามซ้ำ / ตรวจข้อมูล

---

## ลองเลย 2 — สรุปงาน 1 ชิ้นของตัวเอง (10 นาที)

**โจทย์:** เลือก 1 อัน แล้วให้ Claude ช่วย

- อีเมล/แชทลูกค้าที่ต้องตอบ → ร่างคำตอบ
- เอกสาร/รายงานที่ต้องอ่าน → วาง แล้วให้สรุป 5 bullet
- ข้อความประกาศ/ประชาสัมพันธ์ที่ต้องเขียน → ให้เขียนร่าง 2 เวอร์ชัน

**กติกา:** ต้องใส่ context + บอกความยาว + บอกโทน

<div class="ok">

**ผู้ช่วยห้อง:** ต่อ · นัท เดินดู ใครติดยกมือ

</div>

---

## เคล็ดลับที่คนไม่ค่อยรู้

1. **ให้ตัวอย่างดีกว่าอธิบายยาว** — วาง caption เก่า 2-3 อัน แล้วบอก "เขียนแบบนี้แต่เรื่องใหม่"
2. **บอกให้ถามกลับก่อน** — "ถ้าอยากรู้อะไรเพิ่มก่อนเริ่ม บอกได้"
3. **ให้ทำหลาย option** — "ทำ 3 แบบ สั้น/กลาง/ยาว" เลือกได้ดีกว่าอันเดียว
4. **แก้ทีละจุด** — ไม่พอใจตรงไหน บอกจุดนั้น ไม่ต้องพิมพ์ใหม่ทั้งหมด
5. **จำใส่ Project** — งานที่ทำบ่อย เปิด Project แล้วใส่ context ไว้ ไม่ต้องเล่าใหม่ทุกครั้ง

## Prompt library — เริ่มสะสมของตัวเอง

**เก็บ prompt ที่ใช้แล้วเวิร์ก** เพื่อใช้ซ้ำ (ไม่ต้องคิดใหม่ทุกครั้ง)

**เก็บที่ไหนก็ได้:**
- Google Doc / Notes ในมือถือ
- ไฟล์ใน Google Drive ของทีม
- Claude Project (แนะนำ ถ้าจะใช้กับ Claude อย่างเดียว)

**แต่ละ prompt ควรมี:**
- ชื่อสั้นๆ ("ร่างอีเมลตอบร้องเรียน")
- ตัว prompt เอง
- ตัวอย่าง output ที่ใช้ได้จริง
