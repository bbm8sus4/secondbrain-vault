# Part 4
## Cowork + Claude Code บน Warp

**AI Workshop · บริษัทขอนแก่นอิเล็คทริค**
Advanced track · ช่วงบ่าย

<br>

*จาก "คุยกับ Claude" → "ให้ Claude ลงมือทำงานให้"*

---

## ก่อนอื่น — เราอยู่ตรงไหนแล้ว

เช้านี้เราทำ **Chat** ได้แล้ว — พิมพ์คุย · เขียน · สรุป · แปล
Claude ตอบเป็น "ข้อความ" ให้เรา copy ไปใช้เอง

> **บ่ายนี้ไปอีกขั้น:** ให้ Claude **ลงมือทำงานกับไฟล์จริง** ในเครื่องเรา
> ไม่ใช่แค่ตอบ — แต่สร้างไฟล์ · แก้ไฟล์ · จัดโฟลเดอร์ · ทำเว็บ ให้เลย

มี 2 ประตูสู่โลกนี้:

- **Cowork** — ผ่านหน้าแอป กดปุ่ม (ไม่ต้องพิมพ์คำสั่ง)
- **Claude Code บน Warp** — ผ่าน terminal (แรงกว่า · ทำ automation ได้)

---

## Chat vs "ให้ทำงานเอง" — ต่างกันยังไง

| | Chat (เช้านี้) | Cowork / Claude Code (บ่ายนี้) |
|---|---|---|
| **Claude ทำอะไร** | ตอบเป็นข้อความ | ลงมือทำกับไฟล์จริง |
| **ผลลัพธ์** | เรา copy ไปวางเอง | ได้ไฟล์/เว็บออกมาเลย |
| **งานหลายขั้น** | สั่งทีละอัน | สั่งครั้งเดียว ทำครบ |
| **เหมาะกับ** | เขียน · คิด · ปรึกษา | จัดไฟล์ · ทำเอกสารเป็นชุด · ทำเว็บ |

> **เปรียบเทียบ:** Chat = "ที่ปรึกษาบอกวิธี" · Cowork/Code = "ลูกน้องลงมือทำให้"

# 1 · Claude Cowork
## ให้ Claude ทำงานให้ — โดยไม่ต้องแตะ terminal

---

## Cowork คืออะไร

โหมดใน**แอป Claude เดสก์ท็อป** ที่ Claude ทำงานเป็นขั้นๆ ให้เรา
บนไฟล์และโฟลเดอร์จริงในเครื่อง — **กดปุ่มเอา ไม่ต้องพิมพ์คำสั่ง**

<div class="ok">

**พูดง่ายๆ:** เหมือนจ้างผู้ช่วยมานั่งหน้าคอมเรา
เราบอกงานเป็นภาษาคน เขาเปิดไฟล์ · ทำ · เซฟ ให้เอง

</div>

- เห็น Claude ทำงานทีละสเต็ป (โปร่งใส ตรวจได้)
- ทำงานที่ต้องยุ่งกับหลายไฟล์พร้อมกันได้
- เหมาะกับคนที่ **ยังไม่อยากแตะ terminal** — เป็นสะพานที่ดี

---

## Cowork ทำอะไรให้ทีมการตลาดได้บ้าง

| งานจริง | สั่งว่าอะไร |
|---|---|
| **รวมรายงาน** | "อ่านไฟล์ยอดขาย 3 ไฟล์นี้ สรุปเป็นรายงานเดียว" |
| **จัดไฟล์** | "จัดรูปโปรโมชันในโฟลเดอร์นี้ แยกตามเดือน" |
| **ทำคอนเทนต์เป็นชุด** | "เขียน caption 10 โพสต์จากตารางนี้ เซฟเป็นไฟล์" |
| **แปลงเอกสาร** | "เอา Word นี้ ทำเป็นสไลด์สรุป 1 หน้า" |

> **จุดต่างจาก Chat:** งานพวกนี้ถ้าใช้ Chat ต้อง copy เข้า-ออกหลายรอบ
> Cowork ทำครบในที่เดียว แล้วได้ไฟล์กลับมาเลย

---

## เริ่มใช้ Cowork — 4 ขั้น

<div class="step">

**1)** เปิด **แอป Claude เดสก์ท็อป** (ไม่ใช่หน้าเว็บ)

</div>

<div class="step">

**2)** เลือกโหมด **Cowork** แล้วชี้ไปที่**โฟลเดอร์งาน**ที่จะให้ทำ

</div>

<div class="step">

**3)** พิมพ์สั่งงานเป็น**ภาษาคน** เหมือนบอกลูกน้อง

</div>

<div class="step">

**4)** Claude ขอ**อนุญาต**ก่อนแก้ไฟล์ → เรากด **อนุมัติ/ไม่อนุมัติ** ทีละสเต็ป

</div>

<div class="warn">

**กฎเหล็ก:** ชี้ไปที่**สำเนา**ก่อนเสมอ · อ่านสิ่งที่มันจะทำก่อนกดอนุมัติ

</div>

---

# 2 · Claude Code บน Warp
## ปลายทาง — ทำ automation ได้จริง

---

## ทำไมต้องขึ้น terminal

Cowork เก่งแล้ว แต่ **Claude Code บน terminal แรงกว่า**:

- ทำงานกับ**ทั้งโปรเจกต์**ได้ (ไม่ใช่ทีละไฟล์)
- ต่อ **GitHub · deploy เว็บ · ฐานข้อมูล** ได้ครบ
- ตั้ง **automation ทำซ้ำ**ได้ — งานที่ทำทุกวัน สั่งครั้งเดียว

<div class="ok">

**นี่คือเหตุผลที่บริษัทอยากให้เรียน terminal:**
มันเปลี่ยนจาก "ใช้ AI ช่วย" → "ให้ AI ทำงานแทนเป็นระบบ"

</div>

> อย่าเพิ่งกลัวคำว่า terminal — หน้าถัดไปจะเห็นว่ามันง่ายกว่าที่คิด

---

## Warp คืออะไร — terminal ที่ไม่น่ากลัว

**Warp** = โปรแกรม terminal ยุคใหม่ ออกแบบมาให้คนทั่วไปใช้ได้

| terminal เก่า | Warp |
|---|---|
| จอดำ พิมพ์ผิดนิดพัง | พิมพ์เหมือนแชท มีปุ่มกด |
| ต้องจำคำสั่ง | มี AI ช่วยเติมให้ |
| error งงเป็นภาษาต่างดาว | อธิบาย error ให้เข้าใจ |

> **จำภาพเดียว:** Warp = "กล่องพิมพ์คุย" ที่ต่อกับเครื่องเราได้จริง
> เราจะพิมพ์ **ภาษาคน** ไม่ใช่โค้ด

---

## Claude Code คืออะไร

**Claude ที่รันอยู่ใน terminal** — เห็นทั้งโปรเจกต์ ทำงานให้ครบวงจร

- เปิดในโฟลเดอร์งาน → พิมพ์ `claude` → เริ่มสั่งงานได้เลย
- สั่งเป็น**ภาษาคน**: "ทำเว็บโปรโมชันเครื่องซักผ้า แล้ว deploy ให้หน่อย"
- Claude อ่านไฟล์ · เขียนโค้ด · แก้ · ทดสอบ · deploy ให้เอง

<div class="ok">

**สำคัญ:** เราไม่ต้องเขียนโค้ดเป็น — เราเป็น "คนสั่ง" Claude เป็น "คนทำ"
หน้าที่เรา = อธิบายให้ชัด + ตรวจผลงาน

</div>

---

## Cowork vs Claude Code — เลือกยังไง

| | Cowork | Claude Code (Warp) |
|---|---|---|
| **หน้าตา** | แอป กดปุ่ม | terminal พิมพ์ |
| **ความยาก** | ง่าย เริ่มได้เลย | ต้องคุ้นนิดหน่อย |
| **พลัง** | งานไฟล์/เอกสาร | ทั้งโปรเจกต์ + เว็บ + automation |
| **เหมาะกับ** | งานประจำวัน | สร้างของจริงขึ้น production |

> **เส้นทางแนะนำ:** เริ่ม **Cowork** ให้คุ้นก่อน → พร้อมแล้วค่อยขยับขึ้น **Warp**
> ไม่ต้องรีบ — คนละจังหวะกันได้

---

## ลองเลย — งานจริง 1 ชิ้น (Advanced)

**โจทย์:** ทำ landing page โปรโมชันของตัวเอง แล้วเอาขึ้นเว็บจริง

```
เปิด Warp ในโฟลเดอร์ว่าง → พิมพ์ claude → แล้วสั่ง:

"ทำ landing page โปรโมชันเครื่องซักผ้าลด 30%
กลุ่มเป้าหมายแม่บ้านขอนแก่น ใส่ปุ่มติดต่อทาง LINE
โทนสีน้ำเงิน-ขาว ดูน่าเชื่อถือ แล้ว deploy ขึ้นให้ได้ลิงก์"
```

<div class="ok">

**เป้าหมาย:** ได้**ลิงก์เว็บจริง**ที่เปิดในมือถือได้ ภายในคาบนี้
ต่อ/นัท เป็นพี่เลี้ยง เดินช่วยใครติด

</div>

---

## กฎความปลอดภัย — ทั้ง Cowork และ Warp

<div class="warn">

- **ตรวจก่อนอนุมัติ** — อ่านว่ามันจะทำอะไรก่อนกด "ใช่" ทุกครั้ง
- **อย่าให้ยุ่งกับไฟล์สำคัญ** — ทำงานบนสำเนา/โฟลเดอร์แยกก่อน
- **ข้อมูลลับ / PDPA** — อย่าเอาข้อมูลลูกค้าจริง เบอร์ บัตร ปชช. เข้าไป
- **ลบแล้วหายจริง** — คำสั่งลบ/เขียนทับ ให้ยั้งคิดก่อนเสมอ

</div>

<div class="ok">

**ตัวช่วย:** ทั้ง 2 โหมดจะ**ขออนุญาตก่อนแตะของสำคัญ** — จุดนั้นคือจุดที่เราต้องตั้งสติอ่าน

</div>

---

## Recap — จำ 5 อย่างพอ

1. **Chat = ตอบ · Cowork/Code = ลงมือทำให้** กับไฟล์จริง
2. **Cowork** = ในแอป กดปุ่ม ไม่ต้องพิมพ์คำสั่ง — สะพานที่ดีสำหรับคนกลัว terminal
3. **Warp + Claude Code** = terminal ที่ไม่น่ากลัว สั่งภาษาคน ทำได้ถึง deploy เว็บจริง
4. **เราเป็นคนสั่ง Claude เป็นคนทำ** — ไม่ต้องเขียนโค้ดเป็น แค่สั่งชัด + ตรวจผล
5. **ตรวจก่อนอนุมัติเสมอ** + ไม่เอาข้อมูลลับเข้า + ทำบนสำเนา
