Claude AI Workshop · Needs Analysis

เจ้าของธุรกิจต้องการอะไร เทียบกับสิ่งที่ผู้เข้าอบรมต้องการ — สอดคล้องกันแค่ไหน

สรุปจากแบบสำรวจจริง เปรียบเทียบ “มุมเจ้าของ/ผู้ว่าจ้าง” กับ “มุมพนักงานผู้เข้าอบรม” เพื่อใช้ออกแบบหลักสูตรให้ตรงทั้งสองฝ่าย

บริษัท ขอนแก่นอิเล็คทริค (ระบบไฟ · โซลาร์เซลล์) ผู้ว่าจ้าง Ketthip · CEO ผู้เข้าอบรม 8 คน (การตลาด 7 · ช่าง 1) งบ ฿30,000–50,000 · 1 วัน

บทสรุปผู้บริหาร

สอดคล้องบางส่วน — เป้าหมายตรง แต่มี gap วิกฤต

“ความอยาก” ของทั้งสองฝ่ายตรงกันมาก แต่ “ความพร้อม + เวลา” ยังเป็นช่องว่างที่ต้องจัดการก่อนวันอบรม

เจ้าของอยากให้ทีม ทำเองได้จริง ทั้งใช้ AI · ทำเว็บขึ้น production · ใช้ฐานข้อมูล ภายใน 1 วัน — และที่น่ายินดีคือ ผู้เรียนเองก็ตั้งเป้าสูงตรงกัน (6 ใน 8 คนอยากทำเว็บ deploy เองได้, 5 คนอยากเชื่อม AI กับ Excel/ฐานข้อมูล)

แต่ความเป็นจริงคือผู้เรียนส่วนใหญ่เป็น ผู้เริ่มต้น: 5 ใน 8 ไม่เคยเขียนโค้ดเลย · ไม่มีใครใช้ Claude เป็นประจำ · ความมั่นใจใช้ AI เฉลี่ย 2.6/5 · และ 6 ใน 8 “กลัวเรียนไม่ทัน” หากดันให้ทุกคน “ทำเว็บ + ฐานข้อมูลเองได้จริง” ใน 1 วัน มีความเสี่ยงสูงที่กลุ่มผู้เริ่มต้นจะตามไม่ทันและหลุดกลางทาง

ทางออก: คงเป้าหมายร่วม แต่ แบ่งระดับผลลัพธ์ (tiered) + โฟกัส use case การตลาด ซึ่งเป็นจุดที่ทั้งสองฝ่ายตรงกันที่สุด

ตารางเทียบความสอดคล้อง

ประเด็นเจ้าของต้องการสถานะจริงของผู้เรียนความสอดคล้อง
ใช้ AI ในงานประจำ ทีมใช้ AI คล่อง ลดงานซ้ำ อยากได้มาก — ภาพ/คอนเทนต์/ค้นหา/แปล เป็นความต้องการอันดับต้น สอดคล้องสูง
โฟกัสการตลาด เพิ่มหัวข้อ “AI กับการตลาด” 7 ใน 8 เป็นสายการตลาด use case ตรงกันเอง สอดคล้องสูง
ทำเว็บ / landing page deploy ขึ้น production เองได้จริง 6/8 อยากทำได้ แต่ 5/8 ไม่เคยเขียนโค้ด — อยากแต่ทักษะยังไม่ถึง ตรงเป้า แต่ทักษะไม่ถึง
เครื่องมือ Claude ใช้ Claude เป็นหลัก ไม่มีใครใช้ Claude (ใช้ ChatGPT/Gemini) ต้อง onboard ตั้งแต่ต้น ต้องปูพื้นก่อน
การติดตั้งโปรแกรม ผู้เรียนติดตั้งเองได้ 5/8 ติดตั้งเองไม่ได้ / ไม่แน่ใจ ปัญหา logistics วันงาน ต้องเคลียร์ก่อน
ระดับ “ทำเองได้จริง” ทุกหัวข้อ คาดหวังระดับสูงสุดทุกหัวข้อ ผู้เริ่มต้นเป็นส่วนใหญ่ · 6/8 กลัวเรียนไม่ทัน ความคาดหวังสูงกว่าความพร้อมมาก gap วิกฤต
ฐานข้อมูล (Supabase/Firebase) ใช้ฐานข้อมูลในงานได้ ไม่มีใครระบุ DB เป็นงานจริง · 5/8 ไม่เคยโค้ด เป็น owner-push ไม่ใช่ learner-pull เสี่ยงไม่ตรงความต้องการ
GitHub / เวอร์ชันคอนโทรล อยากให้เรียนรู้ ไม่มีใครพูดถึง / สนใจ นามธรรมเกินไปสำหรับสายการตลาด เสี่ยงไม่ตรงความต้องการ
เวลา 1 วัน vs ขอบเขต 4+ หัวข้อ ภายใน 1 วัน กลุ่มผู้เริ่มต้น ต้อง pacing ช้า เนื้อหาแน่นเกินเวลา เสี่ยงทำไม่ครบ
สอดคล้อง — เดินหน้าได้เลย ต้องปรับ/เตรียมก่อน เสี่ยง — ต้องตัดสินใจกับเจ้าของ

ข้อเสนอเพื่อปิด gap

  1. แบ่งระดับผลลัพธ์ (tiered) แทนการบังคับทุกคนเท่ากันCore track (ทุกคนทำได้จริง) = ใช้ Claude + คอนเทนต์ + landing page แบบ no-/low-code · Advanced track (ต่อ, นัท) = deploy จริง + ฐานข้อมูล + GitHub
  2. โฟกัส use case การตลาด เป็นแกนหลักคอนเทนต์ · ภาพประกอบ · landing page โปรโมชัน — จุดที่เจ้าของและผู้เรียน 7/8 ตรงกันที่สุด ใช้เป็นโจทย์จริงตลอดคลาส
  3. ลดฐานข้อมูล + GitHub เป็น “เข้าใจภาพรวม + ดู demo”สำหรับคนส่วนใหญ่ ไม่ใช่ทำเอง — คุยกับเจ้าของว่าทักษะนี้เป็นเป้าระยะถัดไป ไม่ใช่จบใน 1 วัน
  4. เตรียมก่อนวันงาน (สำคัญ)ติดตั้ง Claude/Git/Node ล่วงหน้า (5/8 ติดปัญหาสิทธิ์) · เปิดบัญชี Claude ให้ครบ · แจก cheat sheet ศัพท์อังกฤษ/error
  5. จัดการความกังวล “เรียนไม่ทัน” (6/8)เดินช้า มีจุดพัก · จับคู่ buddy (ต่อ/นัท ช่วยเพื่อน) · ตั้งผลลัพธ์เป็นขั้น ไม่กดดันให้ทุกคนถึงปลายทางพร้อมกัน
  6. เสนอเจ้าของ: 1 วันอาจไม่พอสำหรับ “deploy + DB ทำเองได้จริงทุกคน”ทางเลือก — ลด scope ลงให้ลึกเฉพาะที่ใช้จริง หรือเพิ่ม follow-up clinic 1 รอบหลังอบรม เพื่อให้ตัวชี้วัด 3 เดือนเป็นจริง
