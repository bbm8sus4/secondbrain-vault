1

หน้า UI Claude (Chat — เว็บ/แอป)

หน้าที่ทุกคนเจอก่อน — พื้นฐานที่สุด

  • กล่องพิมพ์ (ล่างสุด) — พิมพ์คำถาม/คำสั่ง · Enter ส่ง · Shift+Enter ขึ้นบรรทัดใหม่
  • ปุ่มคลิปหนีบ (แนบไฟล์) — อัปโหลด PDF · รูป · Excel · Word ให้ Claude อ่าน
  • ตัวเลือกโมเดล — สลับ Opus (เก่งสุด) / Sonnet (สมดุล) / Haiku (เร็ว-ประหยัด)
  • ปุ่ม New chat — เริ่มเรื่องใหม่ (งานคนละเรื่อง = แชทคนละอัน)
  • แถบซ้าย — Recents — ประวัติแชทเก่า · ค้นหาย้อนดูงานเดิมได้
  • Projects — โฟลเดอร์เก็บ context ถาวร ใส่ไฟล์+บรีฟครั้งเดียว ไม่ต้องเล่าซ้ำ
  • แผง Artifacts (ขวา) — โชว์ผลงาน (เอกสาร/เว็บ/ตาราง) แบบ live · มีปุ่ม copy · download · publish
  • ปุ่มใต้คำตอบ — copy · retry (ตอบใหม่) · edit (แก้คำถามเรา) · ให้คะแนน (ดี/ไม่ดี)
  • เมนูโปรไฟล์ (มุมล่างซ้าย) — ตั้งค่าภาษา · ธีมสว่าง/มืด · บัญชี
2

Claude Cowork

ให้ Claude ทำงานกับไฟล์ในเครื่อง — ผ่านแอป กดปุ่มเอา ไม่ต้องพิมพ์คำสั่ง

  • เข้าโหมด Cowork — เลือกจากแอปเดสก์ท็อป (ไม่ใช่หน้าเว็บ)
  • เลือกโฟลเดอร์งาน (workspace) — ชี้ว่าจะให้ทำงานกับไฟล์ชุดไหน
  • กล่องสั่งงาน — พิมพ์งานเป็นภาษาคน เหมือนบอกลูกน้อง
  • แผงขั้นตอน (step-by-step) — เห็น Claude ทำทีละสเต็ป โปร่งใส ตรวจได้
  • ปุ่ม Allow / Deny — อนุมัติก่อน Claude แตะไฟล์ทุกครั้ง
  • มุมมองไฟล์ที่เปลี่ยน (preview/diff) — ดูก่อน–หลัง ก่อนกดยอมรับ
  • ปุ่ม Stop / แทรก — หยุดกลางคัน หรือสั่งเพิ่มระหว่างทำ
3

Claude Code (โหมด Code ในแอป — GUI)

พลังเท่า terminal แต่ยังเป็นหน้าต่างกดปุ่ม สำหรับคนไม่อยากพิมพ์คำสั่ง

  • เปิดโปรเจกต์/โฟลเดอร์ — ให้ Claude เห็นทั้งโปรเจกต์ ไม่ใช่ทีละไฟล์
  • หน้าต่างสนทนา + รายการไฟล์ — สั่งงานด้านหนึ่ง เห็นไฟล์อีกด้าน
  • Permission prompt — ขออนุญาตก่อนแก้ไฟล์/รันคำสั่ง
  • มุมมอง diff — เทียบโค้ดเก่า-ใหม่ ก่อน apply
  • ปุ่ม Accept / Reject — ยอมรับหรือย้อนการเปลี่ยนแปลงทีละจุด
4

Claude Code บน terminal Warp

ปลายทาง power user — automation + deploy เว็บจริง

  • เปิด Warp ในโฟลเดอร์โปรเจกต์ — พิมพ์ claude เพื่อเริ่ม session
  • กล่องพิมพ์คำสั่ง — สั่งเป็นภาษาคน ("ทำเว็บโปรฯ แล้ว deploy")
  • Permission mode (Shift+Tab) — ถามก่อนทำ / ทำเองอัตโนมัติ / โหมดวางแผน
  • อนุมัติ y/n — ตอบยอมให้แก้ไฟล์หรือรันคำสั่ง
  • มุมมอง diff — เห็นสิ่งที่จะเปลี่ยนก่อนตกลง
  • @ แท็กไฟล์ — ชี้ไฟล์เฉพาะให้ Claude โฟกัส
  • Slash commands — /model เลือกโมเดล · /clear ล้างบริบท · /init อ่านโปรเจกต์ · /help
  • Skills — คำสั่งสำเร็จรูปที่ทีมสร้างเอง เรียกด้วย /ชื่อ
  • Esc หยุด · resume/--continue — เบรกกลางคัน · ทำงานเดิมต่อจากที่ค้าง
หมายเหตุ: ส่วน terminal Warp (4) แม่นยำ เพราะเป็นตัวเดียวกับที่ใช้งานจริง · ส่วนหน้าเว็บ/แอป + Cowork (1–3) ปุ่มอาจขยับตามเวอร์ชัน — แนะนำเช็คหน้าจอจริงตอนซ้อมก่อนสอน
