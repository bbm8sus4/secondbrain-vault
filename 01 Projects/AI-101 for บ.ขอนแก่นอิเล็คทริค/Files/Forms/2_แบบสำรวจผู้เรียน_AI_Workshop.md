# ชุดที่ 2 — แบบสำรวจผู้เรียน ก่อนการอบรม (Pre-training Survey)

**สำหรับ:** ผู้เรียน 15 ท่าน (บัญชี · การตลาด · วิศวกร · ช่าง)
**วัตถุประสงค์:** ประเมินพื้นฐาน + ความพร้อมด้านเทคนิค เพื่อแบ่งกลุ่ม/ปรับเนื้อหา
**เวลาตอบ:** ~5 นาที

> หัวฟอร์ม: *"แบบสอบถามนี้ใช้เพื่อเตรียมหลักสูตรให้ตรงกับงานและพื้นฐานของทุกท่าน รบกวนตอบตามจริงครับ ไม่มีถูกผิด — ใช้เพื่อออกแบบการอบรมเท่านั้น"*

**2.** แผนก · `[ตัวเลือกเดียว: บัญชี/การเงิน · การตลาด · วิศวกร · ช่าง/เทคนิค · อื่นๆ]` · ⚑

**6.** ระดับการอ่านภาษาอังกฤษ · `[ตัวเลือกเดียว: น้อย · พอได้ · ดี]` · ⚑

## ส่วนที่ 3 — ความพร้อมด้านเทคนิค ⭐ (สำคัญ เพราะคอร์สมี deploy เว็บ + ฐานข้อมูล)

**7.** ระบบปฏิบัติการเครื่องที่จะใช้ในวันอบรม · `[ตัวเลือกเดียว]` · ⚑
- Windows · macOS · Linux · ไม่แน่ใจ

**8.** สิทธิ์ติดตั้งโปรแกรมบนเครื่อง (จำเป็นต่อการลง Claude/Git/Node) · `[ตัวเลือกเดียว]` · ⚑
- ติดตั้งเองได้ · ต้องให้ IT ลงให้ · ไม่แน่ใจ

**9.** ประสบการณ์เขียนโค้ด / ทำเว็บ · `[ตัวเลือกเดียว]` · ⚑
- ไม่เคยเลย · เคยเห็น HTML/CSS นิดหน่อย · เขียนโปรแกรม/ทำเว็บได้

**10.** เคยใช้เครื่องมือเหล่านี้ไหม (เลือกที่เคยใช้) · `[เลือกได้หลายข้อ]`
- GitHub / Git · Command line (Terminal) · ฐานข้อมูล SQL · ฐานข้อมูล NoSQL · ยังไม่เคยเลย

**11.** มีบัญชีเหล่านี้แล้วหรือยัง · `[เลือกได้หลายข้อ]`
- GitHub · Vercel · Netlify · Supabase · Firebase · MongoDB · ยังไม่มีเลย

**12.** ความสบายใจกับเครื่องมือสายเทคนิค (Git, Terminal, ฐานข้อมูล) · `[สเกล 1–5: อึดอัดมาก → สบายมาก]` · ⚑

### 🔑 ข้อ 7–12 คือกุญแจ
ใช้คัดระดับผู้เรียน — ถ้าเจอว่าคนส่วนใหญ่ตอบ "ไม่เคยเขียนโค้ด / ไม่มีสิทธิ์ติดตั้งโปรแกรม" จะรู้ทันทีว่าต้อง **แบ่งกลุ่ม** หรือ **ปรับความลึก** ของหัวข้อ deploy เว็บ + ฐานข้อมูล
