# Strategic Analysis — KuanGolf

> COO-to-COO assessment after deep research on local assets + live website (2026-06-21)

## TL;DR

**Product ดี · Brand ดี · Tech ดี · แต่ยังไม่มี business motion** — ตอนนี้คือ "side project ที่ทำให้สวยแล้วทิ้งไว้"

**Recommendation**: Path B — Park แต่ไม่ทิ้ง · กลับมาเล่นเมื่อ EasySlip/BoostSMS stable

## จุดที่ดีจริง (Strengths)

| # | จุดแข็ง | ทำไมสำคัญ |
|---|---|---|
| 1 | **Niche fit ชัด** | แก้ pain point จริงของก๊วนไทย (สมุดเปียก, นับ Snake ไม่ทัน) |
| 2 | **Differentiator ป้องกันได้** | ต่อแต้ม halve-on-tie + Snake 3 variants — คู่แข่งฝรั่งทำไม่ได้ |

## จุดที่น่าห่วง (Concerns) — เรียงจากร้ายแรง

### 🚨 1. ไม่มี go-to-market motion
- Facebook 2 โพสต์ 0 engagement
- ไม่มี Instagram · ไม่มี ads · ไม่มี SEO
- 30-day brief เขียนไว้ตั้งแต่ พ.ค. 2026 — 1 เดือนผ่านไป **ยังไม่ได้โพสต์เลยสักโพสต์**
- → product พร้อม แต่ไม่มีคนรู้ว่ามันมีอยู่

### ⚠️ 2. Monetization path ขาดตอน
- Free ตอนนี้ **เล่นไม่จำกัด + Side Game ครบ + แชร์ LINE ได้**
- → คนทั่วไปไม่มีเหตุผลอัปเป็น Pro
- ของล่อ Pro = แค่ GPS + ประวัติย้อนหลัง = nice-to-have ไม่ใช่ must-have
- ARPU ฿999/ปี = ต้องการ scale มาก กว่าจะคุ้ม

### ⚠️ 3. Pricing ยังไม่นิ่ง
- llms.txt บอก ฿790 · /sale FAQ บอก ฿999 · /pricing ปกติไม่โชว์ Lifetime
- → ทีมยังเทสราคาอยู่ = ยังไม่มี product-market fit จริง
- (AI bots ตอบลูกค้าผิดอยู่ตอนนี้ เพราะอ่าน llms.txt เก่า)

### ⚠️ 4. TAM จำกัด
- นักกอล์ฟไทย 40-65 ใน "ก๊วน" + ยอมจ่าย ฿999/ปี + ดาวน์โหลดแอป — เหลือเท่าไหร่?
- คู่แข่งหลักไม่ใช่แอปอื่น แต่คือ **กระดาษ + LINE chat ฟรี** (ทำลายยากที่สุด)

### ⚠️ 5. Resource concern
- COO juggling Thunder + EasySlip + BoostSMS + EasyCRM + Friday Bot + อื่นๆ
- Watcharin Kaewmoung (thunder.in.th) build คนเดียว?
- → ถ้าไม่มี owner เต็มเวลา ตัวนี้จะค่อยๆ ตาย

## Fork in the road — 3 ทางเลือก

### Path A — Commit จริง
**ทำเมื่อ**: เชื่อใน TAM + มีคนเต็มเวลา + Thunder/EasySlip stable แล้ว

**Action items**:
- หา **owner คนเดียวเต็มเวลา** (marketing + community)
- รัน 30-day brief ที่มีอยู่ทันที (content + ads)
- จำกัด Free ให้แคบลง (เช่น 2 ก๊วน/เดือน) → ดัน conversion
- Partnership 3-5 สนามกอล์ฟใน Bangkok → product trial
- **Target**: 1,000 paid users ใน 6 เดือน = ฿1M ARR
- ถ้าทำไม่ได้ → ตัดสินใจ park

**Risk**: ดูด attention ออกจาก revenue-generating products ที่ใหญ่กว่า

### Path B — Park แต่ไม่ทิ้ง ⭐ **RECOMMENDED**
**ทำเมื่อ**: priority อื่นมากกว่า (now) + ไม่อยากเสีย sunk cost

**Action items**:
- Maintenance mode · ไม่ปิด ไม่ลงทุนต่อ
- **Update llms.txt** ให้ price ตรงเว็บ (1 ชม.)
- **สร้าง /docs/KNOWLEDGE_BASE** ตามที่ llms.txt อ้าง (1 วัน)
- ทิ้ง LINE OA @kuangolf เปิดไว้รับ inbound
- รอจังหวะ — เมื่อ EasySlip/BoostSMS stable แล้วค่อยกลับมาทำเต็มสูบ

**Rationale**: คุณมี product ที่ทำเงินอยู่แล้ว 4-5 ตัว · KuanGolf ยังไม่ proof = ไม่ใช่ที่ที่ควรเทใจตอนนี้

### Path C — Sell / Spin-off
**ทำเมื่อ**: ไม่มี internal owner + อยาก clean focus

**Action items**:
- ถ้า Watcharin (หรือคนอื่น) อยากเป็น owner → ให้ equity + spin ออกเป็นบริษัทใหม่
- Thunder ถือหุ้น minority + ไม่ต้องเสีย attention
- Royalty / revenue share กลับมา

**Risk**: ถ้าเขา scale ได้ขึ้นมา จะเสียดาย equity

## Decision matrix

| มิติ | Path A | Path B | Path C |
|---|---|---|---|
| Attention cost | สูง | ต่ำ | ศูนย์ |
| Cash burn | กลาง-สูง | ต่ำมาก | ศูนย์ |
| Upside potential | สูง (ถ้า PMF จริง) | กลาง (รอ window) | กลาง (minority stake) |
| Downside risk | เสีย focus + cash | sunk cost | เสีย control |
| ความเหมาะกับสถานการณ์ปัจจุบัน | ต่ำ | **สูง** | กลาง |

## Quick wins ถ้าเลือก Path B (เสร็จใน 1 สัปดาห์)

1. **Update llms.txt** → AI bots ตอบลูกค้าถูก (Pro ฿999, Free unlimited)
2. **สร้าง /docs/KNOWLEDGE_BASE** ตามที่ลิงก์
3. **Custom Facebook URL** เปลี่ยน `profile.php?id=...` เป็น `/KuanGolfApp`
4. **เพิ่ม Schema.org markup** (SoftwareApplication) → ติด search ง่ายขึ้น
5. **เปิด Auto-reply LINE OA** บอกราคา + คุณสมบัติพื้นฐาน

ทั้งหมดไม่เกิน 1 day work · ทำแล้วจบ ไม่ต้อง maintain ต่อ
