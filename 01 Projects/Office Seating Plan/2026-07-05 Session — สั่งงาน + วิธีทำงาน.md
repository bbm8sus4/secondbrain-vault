## 🛠 วิธีทำงาน (Working Method)

**ไฟล์ที่แก้จริง**
- แก้ที่ **`~/Work/office-seating-deploy/dist/index.html`** (single-file HTML ~410KB, ไม่มี build step)
- ⚠️ session นี้ macOS (ghostty) **บล็อกการอ่าน/เขียน `~/Desktop`** (EPERM) — เลย build+deploy จาก `dist/` แทน ซึ่งเป็นตัว deploy จริงอยู่แล้ว · ไฟล์ `~/Desktop/office-seating-overview.html` ตามหลังชั่วคราว รอ sync ตอนสิทธิ์ปกติ
- `_worker.js` = Cloudflare Worker (password gate `thunder2026` + API `/api/state` บน KV = shared state ทุกคนเห็นชุดเดียว)

**ขั้นตอนมาตรฐานทุกครั้งที่แก้**
1. แก้ CSS/JS/HTML ใน dist/index.html
2. **Syntax check**: `node -e` วน `<script>` ทุกบล็อก `new Function()` + เช็ค div balance (`grep -o '<div'` = `</div>`)
3. **Deploy**: `npx wrangler pages deploy dist --project-name=seatmap-bb85fd28 --commit-dirty=true`
4. **Verify ด้วย Chrome MCP** (ผ่าน HTTP curl → `http://127.0.0.1:12306/mcp`, tab id 1331140608):
   - reload tab (`location.reload(true)`) รอ CDN cache หาย (~7-9s) แล้ว poll หา function ใหม่ก่อนวัด
   - วัดผลจริง (DOM measure / computed style / เรียก function) — ไม่เดา
5. เช็ค **console** ว่า app-origin exceptions = 0 (ข้าม error จาก browser extension)

**กติกาเหล็กเรื่องข้อมูล (สำคัญมาก)**
- ❌ **ห้ามเขียนทับ KV จริง** ตอนเทสต์ — เคยพลาดทับข้อมูลจริงมาก่อน (ผู้ใช้โกรธมาก)
- ✅ เทสต์ logic ที่แก้ state โดย **stub `window.save`/`saveOrg` + snapshot state → restore** หลังเทสต์ (ข้อมูลจริงไม่ถูกแตะ)
- ✅ ยืนยัน 47-49 คน / ver ไม่เปลี่ยน ทุกครั้งหลังเทสต์
- การเปลี่ยนข้อมูลจริง (เช่นตั้ง WOOT/PHUM เป็น owner) ทำได้เมื่อ **ผู้ใช้สั่งชัดเจน** เท่านั้น (ไม่ใช่การเทสต์)

**สไตล์การตอบ**: ไทยธรรมชาติ กระชับ ขึ้นด้วยผลลัพธ์ก่อน ลงท้าย "ครับ" · ผู้ใช้ = ผู้บริหาร ทำเร็ว สั่งรัว ต้องการ zero-defect + verify จริงทุกครั้ง

## 📋 รายการสั่งงาน 2026-07-05 (ตามลำดับ)

### กลุ่ม UI/Layout
- **Toolbar ทุกหน้าให้เท่ากัน** ยึดหน้า org เป็นหลัก (full-width sticky bar + inner 1480/22px) — seating/manage ทำตาม
- **Manage table**: ยุบคอลัมน์ 8→6 (รวม ห้อง+โปรดักส์+ผังองค์กร), ต่อมาเหลือ **แค่ "ห้อง"** โชว์เลขห้อง (203) hover เห็นชื่อเต็ม, ตัด tag ออก
- คอลัมน์ "สถานะ" → **"สัญญาจ้าง"** (โชว์ประเภทจ้าง ตัด active/remote), ลบปุ่ม "ยกเลิกที่นั่ง"
- **ตัวกรองบริษัท** ตัดตัวย่อ T·/E·/O· ออก, **ปุ่มมุมมอง** = "มุมมอง" เสมอ + ไอคอน folder→eye
- เน้น **"ห้อง" แทน "ที่นั่ง"** ทั้งระบบ (การจัดห้อง/เข้าห้องแล้ว/ยังไม่เข้าห้อง)
- **Responsive จอแคบ/แบ่งจอ**: topbar แถวเดียวชิดซ้าย, ปุ่มขวาชิดขวา, undo/redo ติดกลุ่มปุ่มขวา (gap 20px)
- แก้ dropdown ในฟอร์ม: ลบขีด "—", ลูกศร custom สมดุล
- Blueprint popup: ปุ่ม "แก้ไข" คำเดียว + ✕ จัดกลุ่มมุมขวาบน

### กลุ่ม Sidebar / Roster / Tray
- ถาด "รอจัดที่" → **roster สไตล์หน้าโปรดักส์** (ทุกคน + ช่องค้นหา + ชิป, คนถูกจัดแล้วสีจาง)
- กล่อง roster **สูงเท่าโซนห้อง** (syncTrayHeight cap = main height, scroll ในกล่อง)
- คำอธิบาย: **"เลือกรายชื่อที่ต้องการ แล้วลากเพื่อวางตำแหน่ง"** (ใช้ทุกกล่อง)
- หน้าโปรดักส์: คนที่อยู่ในโปรดักส์นั้นแล้ว **จางลง กันเลือกซ้ำ** (per-product) + ปุ่ม "จัดการเลน" + ลากสลับลำดับในเลนได้

### กลุ่ม WFH / โซนออนไลน์
- เพิ่ม **โซน "ทำงานออนไลน์ (WFH)"** บนผังที่นั่ง (กล่องเหมือนห้อง, เส้นขอบเดียวกัน, ปุ่ม + = EMPTY)
- ลากคนเข้า = ตั้ง remote+เอาออกจากที่นั่ง · badge "WFH" ใต้ชื่อ · **ลากสลับลำดับในกล่องได้**
- คน WFH ในตารางจัดการ คอลัมน์ห้องขึ้น **"WFH"**

### กลุ่มผังองค์กร
- แก้คำในฟอร์ม "เพิ่มบุคคล" ไม่มีขีด/ไม่งง, เปลี่ยน `<select>` เป็น **picker สวย + ค้นหา** (เหมือนหน้าโปรดักส์)
- เพิ่มคนในผัง → **ไปโผล่หน้าจัดการอัตโนมัติ** (Manage = master)
- ปุ่ม "จัดการเลน" (เพิ่ม/แก้/สลับ/สี), การ์ดแก้ไขได้ทุกบริษัท
- Card: ถอด badge ตัวย่อบริษัท + **วงแหวนเขียว (สีค้าง)**, เพิ่มสถานะ WFH/ที่ออฟฟิศ+ห้อง (pill กระชับ)
- ปุ่ม "ผูกอัตโนมัติ" — เพิ่มแล้ว**ถอดออกทีหลัง** (ผู้ใช้ไม่เอา)
