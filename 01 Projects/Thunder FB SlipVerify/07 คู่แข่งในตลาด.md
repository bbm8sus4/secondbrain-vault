# 🥊 คู่แข่งในตลาด

> กลับไป [[00 แผนที่ความรู้ (MOC)]]

## กลุ่ม A — แพลตฟอร์มแชทที่ตรวจสลิปบน Facebook **ได้แล้ว** (คู่แข่งตรงของ pivot นี้)
| เจ้า | ทำอะไรบน FB | ราคา/จุดเด่น | จุดอ่อน |
|------|------------|-------------|---------|
| **ZWIZ.AI** | บอท FB+LINE มีตรวจสลิปในตัว | Premium แถม 100 สลิป/เดือน | ตรวจสลิปเป็นฟีเจอร์เสริม ไม่ใช่แกน |
| **Chatcone** | รวมแชท FB/LINE + SlipCheck 1 วิ | ต่อ bank API ตรง, กันสลิปซ้ำ | ราคาไม่เปิดชัด, ตรวจสลิปเป็นส่วนเสริม |
| **Page365** | ระบบหลังร้าน + ตรวจสลิป | อัป 3 สลิปพร้อมกัน, auto เปลี่ยนสถานะบิล | เน้นจัดการบิล ช่องทางแชทรอง |
| **Commerzy** | ตรวจสลิปผ่าน KBank API | เริ่ม ~500 บาท | ผูกกับ KBank API เจ้าเดียว |

## กลุ่ม B — ผู้ให้บริการ API ตรวจสลิปโดยตรง (คู่แข่งของ EasySlip/Thunder API)
ตลาดแบ่ง 3 ชั้น: **ธนาคารตรง** (KBank/SCB) · **ตัวกลาง** (RDCW, SlipOK, Thunder/EasySlip) · **e-wallet** (TrueMoney)

| เจ้า | ราคา (ยืนยัน 9 ก.ค. 2026) | หมายเหตุ |
|------|------|----------|
| **SlipOK** | ฟรี OK BASIC 100/เดือน · ฿350/500 · ฿600/1,000 · ฿1,000/2,000 · ฿2,580/6,000 | **คู่แข่งตรงสุด** มีฟรีทีเยอร์ ยิ่งซื้อยิ่งถูก/สลิป |
| **RDCW** | endpoint `suba.rdcw.co.th/v2/inquiry` | นิติบุคคล, มี error code "Bank API Error" ชัด |
| **Slip2Go / Check-slip / Bentoweb** | ขอใบเสนอราคา | รายเล็ก |
| **ธนาคารตรง (KBank/SCB)** | ต่อรองกับธนาคาร | ⚠️ **KBank ห้ามร้านบน FB/Line + รับเฉพาะนิติบุคคล** → ต้องผ่านตัวกลาง |

## 💡 ข้อสังเกตเชิงกลยุทธ์
- **ไม่มีใครทำ "ตรวจสลิปบน Facebook" เป็นแกนหลักจริงจัง** — ทุกเจ้าทำเป็นฟีเจอร์เสริมของแพลตฟอร์มแชทรวม
- **ช่องว่าง** = บอทตรวจสลิป Facebook ที่ **โฟกัส/แม่น/เร็ว** โดยมี Thunder API เป็นแกน
- SlipOK มีฟรีทีเยอร์ + ถูก → Thunder ต้องสู้ด้วย **"ความแม่น + enterprise-grade + ครบวงจรบน FB"** ไม่ใช่สู้ราคา
