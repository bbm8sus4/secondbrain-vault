# ⚖️ กฎหมาย ความเสี่ยง และคำถามเปิด

> กลับไป [[00 แผนที่ความรู้ (MOC)]]
> 🔴 โน้ตนี้คือ **ประเด็นสำคัญสุดที่ต้องเคลียร์ก่อนสเกล**

## 🔴 ประเด็นชี้ขาด: กฎธนาคารเรื่อง Social Media
**KBank Slip Verification API ทางตรง "ห้าม" ร้านค้าบนแพลตฟอร์ม Social Media (Facebook/Line)** ขึ้นทะเบียนใช้บริการ — อยู่ในลิสต์ "ธุรกิจที่ธนาคารไม่อนุญาต" + รับเฉพาะนิติบุคคล

**นัย:**
- นี่คือเหตุผลที่ **ตัวกลางอย่าง EasySlip/SlipOK มีอยู่** (ไปต่อ ITMX/ธนาคารแทนร้าน)
- เราต้องต่อผ่าน **EasySlip/Thunder API เท่านั้น — ห้ามให้ร้านไปต่อ bank API ตรง**
- ต้องเช็คให้ชัดว่า **เงื่อนไขบริการ (ToS) ของ EasySlip เองรองรับ use case "บน Facebook"** ก่อนสเกล
- _เคยมีดีล KBank×Meta ปี 2566 เช็คสลิปใน Messenger — แต่เป็น integration ตรงของ Meta+KBank คนละเรื่องกับร้านสมัคร API เอง_

## PDPA + ข้อมูลการเงิน
- ข้อมูลสลิป = ข้อมูลส่วนบุคคล/การเงิน → เข้าข่าย **PDPA ไทย** ต้องมีฐานการประมวลผล + Privacy Policy + เก็บเท่าที่จำเป็น
- สอดคล้องกฎภายใน: ข้อมูลสลิป/การเงินเป็นความลับ
- Meta Platform Terms: ต้องลบรูปสลิป "ทันทีที่ทำได้", ประมวลผล "ในนามร้านค้าเท่านั้น", แยกข้อมูลรายร้าน → วางโมเดลเป็น **"ผู้ประมวลผลแทนร้านค้า"** (ตรง positioning B2B ของ Thunder)

## Risk Register (ความเสี่ยงที่ต้องบริหาร)
| ความเสี่ยง | ระดับ | การรับมือ |
|-----------|:----:|-----------|
| นโยบายธนาคาร social media | 🔴 สูงสุด | ต่อผ่านตัวกลางเท่านั้น + เคลียร์ ToS/ฐานกฎหมายก่อนสเกล |
| App Review/Business Verification เป็นคอขวด | ⚠️ | เริ่มยื่นเนิ่นๆ เผื่อเวลาแก้หลายรอบ |
| Meta เปลี่ยนกฎบ่อย (เช่น message tag ตาย เม.ย. 2026) | ⚠️ | ออกแบบให้เปลี่ยน channel ได้ |
| ลิงก์รูป IG/คอมเมนต์หมดอายุ + IG ห้ามเก็บ | ⚠️ | flow แบบ pass-through |
| PDPA — ข้อมูลสลิปเป็นข้อมูลส่วนบุคคล | ⚠️ | นโยบาย + เก็บเท่าที่จำเป็น |
| พึ่ง ITMX/bank API (ล่ม/เปลี่ยน) | ⚠️ | retry + แจ้งผู้ใช้ให้ดี |
| CDN ย่อรูปคอมเมนต์ | 🟡 | เทสต์/ดึงไฟล์ต้นฉบับ (Stage 3) |

## ❓ คำถามที่ยังเปิดอยู่ (เคลียร์ก่อนลงทุนจริง)
1. 🔴 **ฐานทางกฎหมาย/ธปท.:** EasySlip/SlipOK ให้บริการร้านบน Facebook/LINE บนสัญญาแบบไหน (aggregator/ITMX agreement?) และขยายมาเป็น product "ใน" Facebook ได้ถูกกฎหมายไหม — **ต้องเช็คกับฝ่ายกฎหมาย**
2. **PDPA + Meta Payments policy** ผูกมัดอะไรเพิ่มกับข้อมูลสลิป (ชื่อผู้จ่าย/รับ, เลขบัญชี, ยอด)
3. **rate limit + อายุลิงก์ CDN จริง** ที่ระดับ scale — ต้องทดสอบ
4. **accuracy/latency จริง** ของ EasySlip (เว็บเคลม 99.98% แต่ยังไม่มีการวัดอิสระ)
