## โมเดลธุรกิจ
- รายได้ 2 ทาง: Slip Verification API (คิดต่อสลิป) + LINE chatbot (subscription)
- เก็บเงินแบบ **pre-paid** (เติมล่วงหน้า) → margin ต้นเดือนดู "ติดลบ" เป็นเรื่องปกติ
- ต้นน้ำ: ตรวจสลิปข้ามธนาคารผ่าน **ITMX / API ธนาคาร** (ดู [[06 EasySlip และ Thunder API]])

## จุดยืน CS / โทนแบรนด์
- โทนสุภาพ ลงท้าย "ค่ะ/นะคะ", เปิดหัว ⚠️/✅/📢, ปิดเคส ✅ + เบอร์ติดต่อ
- ประกาศผ่าน Facebook Page + LINE OA broadcast
- เคสบ่อย: ธนาคารส่ง QR ผิด, สลิปกสิกร "ไม่พบข้อมูล", peak ธนาคารช้า, LINE OA/Cloudflare ล่ม
- 📌 นัย: ไป Facebook ต้องเตรียม flow CS/ประกาศบนช่องทางใหม่ด้วย

## SWOT ของการ pivot ไป Facebook
- **จุดแข็ง:** API ตรวจสลิปแม่น + ฐานลูกค้า + แบรนด์ในวงการ, bot engine พร้อม
- **จุดอ่อน:** ยังไม่มีประสบการณ์ Facebook Platform, ต้องผ่าน App Review, ทีมเล็ก
- **โอกาส:** social commerce ~38%, ยังไม่มีเจ้าไหนทำตรวจสลิป FB เป็นแกน, พ.ร.ก.ไซเบอร์ดัน demand
- **ความเสี่ยง:** Meta เปลี่ยนกฎบ่อย, คู่แข่งขยับก่อน, พึ่ง ITMX/ธนาคาร, สงครามราคาจาก SlipOK
