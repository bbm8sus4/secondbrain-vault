# 🎯 ปัจจัยสำเร็จและกับดัก

> กลับไป [[00 แผนที่ความรู้ (MOC)]]

## "สำเร็จ" หมายความว่าอะไร (3 ระดับ)
- **Pilot สำเร็จ:** flow ทำงานจริงบนเพจเราเอง ตรวจแม่นทุกเคสทดสอบ
- **Product สำเร็จ:** ร้านค้าจริงเชื่อมเพจตัวเองแล้วใช้งานประจำ (retention)
- **ธุรกิจสำเร็จ:** ลูกค้า Facebook เติบโตจนเป็นช่องทางรายได้ใหม่ของ Thunder

## ✅ ปัจจัยที่ทำให้สำเร็จ
1. **ความแม่น + ครบเคส** — จับปลอม/ซ้ำ/ยอดผิดได้จริง (ไม่ใช่แค่ happy path) → ร้านถึงกล้าเลิกเช็คมือ
2. **เร็ว** — ตอบใน 2–5 วิ (EasySlip เร็วพออยู่แล้ว, อยู่ที่ pipeline เรา)
3. **เริ่มเล็ก พิสูจน์ก่อน** — pilot เพจเดียวให้เห็นผลจริงก่อนลงทุนทำ product เต็ม
4. **ยื่น Meta review เนิ่นๆ** — เป็นคอขวด อย่ารอจนท้าย
5. **ต่อยอดจากของที่มี** — reuse EasySlip API + billing + ฐานลูกค้า Thunder
6. **ประสบการณ์ในแชทลื่น** — ข้อความตอบเป็นภาษาคน ชัด ไม่กำกวม

## 🔴 กับดักที่ทำให้ล้ม (ต้องเลี่ยง)
- **ทำ product เต็มก่อนพิสูจน์ flow** → ลงทุนเยอะแล้วพบว่าเคสจริงไม่เวิร์ค → pilot ก่อนเสมอ
- **webhook ตอบช้าเกิน 5 วิ** → Meta ตัด subscription → ต้องใช้ queue แยกงานหนัก
- **ไม่เคลียร์กฎธนาคาร** ([[09 กฎหมาย ความเสี่ยง และคำถามเปิด]]) → โดนบล็อกตอนโตแล้ว → เคลียร์ตั้งแต่ต้น
- **เก็บรูปสลิปถาวร** → ผิด Meta Platform Terms + PDPA → pass-through เท่านั้น
- **ไม่ผ่าน App Review** เพราะเตรียมเอกสาร/วิดีโอไม่ครบ → ยื่นเนิ่นๆ เผื่อแก้หลายรอบ
- **CDN ย่อรูปคอมเมนต์** (Stage 3) → อ่าน QR ไม่ออก → เทสต์/ดึงไฟล์ต้นฉบับ

## ตัวชี้วัดว่าสำเร็จ (ดูตอนมีผู้ใช้จริง)
- อัตราตรวจสำเร็จ/ตรวจไม่ออก · เวลาตอบเฉลี่ย (เป้า < 5 วิ)
- จำนวนสลิปปลอม/ซ้ำที่จับได้ (มูลค่าที่ป้องกันการโกง)
- ร้านที่ใช้ต่อเนื่อง (retention) + จำนวนสลิป/เดือน
