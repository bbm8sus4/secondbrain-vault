# 📋 คุยกับทีม: ตรวจสลิปใน Facebook

## เราจะทำอะไร
บอท **ตรวจสลิปโอนเงินอัตโนมัติในแชท Facebook** — ลูกค้าส่งรูปสลิปในแชทเพจร้าน บอทตรวจว่าจริง/ปลอม/ซ้ำ/ยอดตรงไหม แล้วตอบใน 2-5 วินาที

**ต่อยอดจากของที่มี:** ใช้ EasySlip/Thunder Slip Verification API เดิม — แค่ขยายช่องทางจาก LINE → Facebook

## ทำไมตอนนี้
- Thunder แข็งบน LINE แต่ **ยังไม่มีของบน Facebook เลย** ขณะที่คู่แข่ง (ZWIZ, Chatcone, Page365) เริ่มทำแล้ว
- Facebook/social commerce = ~38% ของ e-commerce ไทย · chat commerce โตต่อเนื่อง
- ตรงโจทย์ขยายฐานลูกค้าใหม่ให้ Thunder

## จะจัดการยังไง (3 สเตจ — เริ่มเล็ก พิสูจน์ก่อน)
| สเตจ | ทำอะไร | ต้องรอ Meta? |
|------|--------|:-----------:|
| **1. Pilot เพจเดียว** | พิสูจน์ flow บนเพจเราเอง: รับสลิปในแชท→ตรวจ→ตอบ | ❌ เริ่มได้เลย |
| **2. Product หลายร้าน** | ให้ร้านเชื่อมเพจตัวเอง + dashboard + billing | ✅ App Review |
| **3. คอมเมนต์/ไลฟ์/IG** | ตรวจสลิปใต้คอมเมนต์/ไลฟ์ + Instagram | ✅ |

## ต้องเตรียมอะไร (แบ่งงานได้เลย)
**คน**
- Dev backend — ทำ webhook + ต่อ EasySlip + logic ตอบ
- คนดูแลฝั่ง Meta/กฎหมาย — เอกสารยืนยันธุรกิจ + App Review + PDPA

**ของ/บัญชี**
- Facebook App + Business Manager + เพจสำหรับทดสอบ
- Server ที่มี HTTPS จริง (self-signed ใช้ไม่ได้)
- EasySlip API key + โควตาสำหรับ dev

**เอกสาร (ใช้ตอนขึ้น Stage 2)**
- หนังสือรับรองบริษัท Thunder (Business Verification)
- Privacy Policy รองรับข้อมูลสลิป (PDPA)
- วิดีโอ demo use-case (สำหรับ App Review)

## เรื่องที่ต้องตัดสินใจ/เคลียร์ก่อน
1. **ใครเป็นเจ้าภาพ dev** + timeline คร่าวๆ
2. **ใช้เพจไหนทำ pilot** (เพจ Thunder / EasySlip / เพจร้านตัวอย่าง)
3. 🔴 **เคลียร์ฐานกฎหมาย** — KBank API ตรงห้ามร้านบน Facebook/Line ขึ้นทะเบียน (เราเป็นตัวกลางผ่าน EasySlip/ITMX จึงทำได้ แต่ต้องยืนยันว่าเงื่อนไขบริการรองรับ use case บน Facebook ก่อนสเกลใหญ่)

## ปัจจัยสำเร็จ vs กับดัก
**✅ สำเร็จเมื่อ:** ตรวจแม่น+ครบเคส (จับปลอม/ซ้ำ/ยอดผิดได้จริง), เร็ว, เริ่มเล็กพิสูจน์ก่อน, ยื่น Meta review เนิ่นๆ, reuse ของเดิม
**🔴 ล้มเพราะ:** ทำ product เต็มก่อนพิสูจน์ flow · webhook ช้าเกิน 5 วิ · ไม่เคลียร์กฎธนาคาร · เก็บรูปสลิปถาวร (ผิด PDPA)

## Next step ที่เสนอ
1. ประชุมทีมเคาะ 3 เรื่องด้านบน (เจ้าภาพ / เพจ pilot / กฎหมาย)
2. เริ่ม Pilot Stage 1 (dev ทำ core บนเพจเราเอง — ไม่ต้องรอใคร)
3. ขนานกัน: เริ่มเตรียมเอกสาร Business Verification (เพราะรอ Meta นาน)
