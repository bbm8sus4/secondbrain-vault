# Course Intake Kit (Master) — ชุดแบบฟอร์มกลางสำหรับลูกค้าคอร์สอบรม AI

**ที่มา:** ถอดแบบจากงาน บ.ขอนแก่นอิเล็คทริค (ก.ค. 2026) → ทำเป็นเทมเพลตกลาง ใช้ซ้ำได้ทุกลูกค้า
**หลักการ:** ทุกอย่างที่เป็นของลูกค้ารายใดรายหนึ่ง ถูกแทนด้วย `[placeholder]` — ก่อนใช้ให้ copy ทั้งโฟลเดอร์ไปไว้ใน `01 Projects/<ชื่องาน>/` แล้วเติมค่า

---

## ชุดฟอร์มมี 4 ตัว (ครบวงจร ก่อน → หลัง → ติดตาม)

| # | ฟอร์ม | ผู้ตอบ | ตอบเมื่อ | วัดอะไร |
|---|-------|--------|----------|---------|
| 1 | แบบสอบถามผู้ว่าจ้าง (Needs Analysis) | เจ้าของ/ผู้ตัดสินใจ | ก่อนเสนอราคา/ออกแบบคอร์ส | เป้าหมาย · ขอบเขต · งบ · ข้อจำกัด |
| 2 | แบบสำรวจผู้เรียน (Pre-training) | ผู้เรียนทุกคน | ≥1 สัปดาห์ก่อนอบรม | พื้นฐาน · ความพร้อมเทคนิค · baseline ความมั่นใจ |
| 3 | แบบประเมินหลังอบรม (Post-training) | ผู้เรียนทุกคน | ทันทีหลังจบ | Kirkpatrick L1+L2 · Δ ความมั่นใจ · NPS |
| 4 | แบบติดตามผล 30–60 วัน (Follow-up) | ผู้เรียนทุกคน | 30–60 วันหลังจบ | Kirkpatrick L3 — ใช้จริงไหม · ประหยัดเวลาเท่าไหร่ |

**โมดูลเสริม (ใช้เฉพาะงาน in-house ที่เราจัดอาหาร):** ฟอร์มเลือกเมนูอาหาร/เบรค — แยกออกจากฟอร์มผู้เรียนแล้ว อย่ารวมกลับ (ของขอนแก่นเดิมรวมอยู่ ทำให้ฟอร์มยาวเกิน)

---

## Placeholder ที่ต้องเติมทุกครั้ง

- `[ชื่อบริษัทลูกค้า]` · `[จำนวนผู้เรียน]` · `[รายชื่อแผนก]`
- `[รายการหัวข้อคอร์ส]` — ดึงจากเมนูหัวข้อในฟอร์ม 1 ส่วน B หลังลูกค้าเลือกแล้ว ค่อยล็อกลงฟอร์ม 2–4
- `[วันอบรม]` · `[ชื่อผู้ประสานงานฝั่งเรา]`

## คำถามกุญแจ — ห้ามตัดออก

- **ฟอร์ม 1 · B2 (ระดับผลลัพธ์ต่อหัวข้อ) × C2 (ใครต้องทำได้)** → กำหนดความลึก + การแบ่งกลุ่ม ถ้าเป้ากับพื้นฐานผู้เรียนไม่ตรงกัน ต้องเจรจา scope/เพิ่มวันก่อนรับงาน
- **ฟอร์ม 1 · A3–A5** → ตั้งเป็น KPI วัดผล (Kirkpatrick L3–L4) ใส่ในใบเสนอราคา
- **ฟอร์ม 1 · G2 (ยอมรับผลลัพธ์แบบ tiered)** → กันดราม่า "ทำไมบางคนทำไม่ได้" ตั้งแต่ก่อนเริ่ม
- **ฟอร์ม 2 · ส่วนความพร้อมเทคนิค** → คัดระดับ/แบ่งกลุ่มผู้เรียน
- **ฟอร์ม 2 ข้อความมั่นใจ (baseline) ↔ ฟอร์ม 3 ข้อความมั่นใจ (หลังเรียน)** → Δ = หลักฐาน Learning ใช้ปิดการขายรุ่นถัดไป
- **ฟอร์ม 3 · NPS** → ตัวเลขรีวิว/ขายต่อ

## หลักการออกแบบ (จากบทเรียนขอนแก่น)

- ลด text input เหลือเฉพาะชื่อ/อีเมล/ตำแหน่ง — ที่เหลือเป็น choice ให้หมด (ตอบเร็ว วิเคราะห์ง่าย)
- ทุก scale มี help text อธิบายค่า 1/3/5
- checkbox ที่มี 5+ ตัวเลือก เปิด "อื่นๆ" เสมอ
- ⚑ = คำถามบังคับตอบ
- Google Forms MASTER ทั้ง 4 ตัว: สร้างด้วย `google-form-deploy/Code.gs` (clasp) — ดูขั้นตอน + วิธีทำสำเนาต่อลูกค้าที่ `google-form-deploy/DEPLOY_NOTES.md`
- ⚠️ **ห้าม clearAllItems ทับฟอร์มที่มี response แล้ว** — response จะ orphan ทันที เช็ค `getResponses().length` ก่อนแก้เสมอ

## Workflow ต่อลูกค้า 1 ราย

1. Copy โฟลเดอร์นี้ → `01 Projects/<ชื่องาน>/Forms/`
2. ส่งฟอร์ม 1 ให้ผู้ว่าจ้าง (หรือใช้เป็นบทสัมภาษณ์ 20–30 นาที)
3. อ่านคำตอบ → ล็อกหัวข้อ + จำนวนวัน + เงื่อนไข → เติม placeholder ฟอร์ม 2–4
4. ส่งฟอร์ม 2 ให้ผู้เรียน ≥1 สัปดาห์ก่อนวันงาน → ใช้ผลแบ่งกลุ่ม/ปรับความลึก
5. วันจบงาน: ฟอร์ม 3 (ให้ตอบก่อนแยกย้าย — response rate สูงสุด)
6. ตั้ง reminder 30–60 วัน: ส่งฟอร์ม 4 → สรุปรายงานผลให้ผู้ว่าจ้าง (อ้าง KPI จาก A3–A5)
