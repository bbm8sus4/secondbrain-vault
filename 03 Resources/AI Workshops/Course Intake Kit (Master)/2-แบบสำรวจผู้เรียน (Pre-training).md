# ฟอร์ม 2 — แบบสำรวจผู้เรียนก่อนอบรม (Pre-training) · เทมเพลตกลาง

**ผู้ตอบ:** ผู้เรียนทุกคน `[จำนวนผู้เรียน]` ท่าน
**วัตถุประสงค์:** ประเมินพื้นฐาน + ความพร้อมเทคนิค เพื่อแบ่งกลุ่ม/ปรับเนื้อหา + เก็บ baseline
**ส่งเมื่อ:** ≥1 สัปดาห์ก่อนวันอบรม · **เวลาตอบ:** ~5 นาที

> หัวฟอร์ม: *"แบบสอบถามนี้ใช้เพื่อเตรียมหลักสูตรให้ตรงกับงานและพื้นฐานของทุกท่าน รบกวนตอบตามจริงครับ ไม่มีถูกผิด — ใช้เพื่อออกแบบการอบรมเท่านั้น"*

---

## ส่วนที่ 1 — ข้อมูลทั่วไป

**1.** ชื่อ–นามสกุล · `[ข้อความสั้น]` · ⚑ *(ใช้จับคู่กับแบบหลังเรียน — ดูพัฒนาการรายคน)*
**2.** แผนก · `[ตัวเลือกเดียว: [รายชื่อแผนกของลูกค้า] · อื่นๆ]` · ⚑
**3.** ตำแหน่ง / หน้าที่หลักโดยย่อ · `[ข้อความสั้น]` · ⚑
**4.** อีเมล (สำหรับส่งสื่อ/เกียรติบัตร/ติดตามผล) · `[ข้อความสั้น]`

## ส่วนที่ 2 — พื้นฐานทั่วไป

**5.** ความคล่องในการใช้คอมพิวเตอร์ · `[สเกล 1–5: เริ่มต้น → คล่องมาก]` · ⚑
**6.** ระดับการอ่านภาษาอังกฤษ · `[ตัวเลือกเดียว: น้อย · พอได้ · ดี]` · ⚑

## ส่วนที่ 3 — ความพร้อมด้านเทคนิค ⭐ *(ใช้เฉพาะเมื่อคอร์สมีหัวข้อเทคนิค — ตัดออกได้ถ้าคอร์สพื้นฐานล้วน)*

**7.** ระบบปฏิบัติการเครื่องที่จะใช้ในวันอบรม · `[ตัวเลือกเดียว: Windows · macOS · Linux · ไม่แน่ใจ]` · ⚑
**8.** สิทธิ์ติดตั้งโปรแกรมบนเครื่อง · `[ตัวเลือกเดียว: ติดตั้งเองได้ · ต้องให้ IT ลงให้ · ไม่แน่ใจ]` · ⚑
**9.** ประสบการณ์เขียนโค้ด / ทำเว็บ · `[ตัวเลือกเดียว: ไม่เคยเลย · เคยเห็น HTML/CSS นิดหน่อย · เขียนโปรแกรม/ทำเว็บได้]` · ⚑
**10.** เคยใช้เครื่องมือเหล่านี้ไหม · `[เลือกได้หลายข้อ: GitHub/Git · Command line · ฐานข้อมูล SQL · ฐานข้อมูล NoSQL · ยังไม่เคยเลย]`
**11.** มีบัญชีเหล่านี้แล้วหรือยัง · `[เลือกได้หลายข้อ: [รายการบริการที่คอร์สใช้ เช่น GitHub · Vercel · Supabase] · ยังไม่มีเลย]`
**12.** เคยใช้ทักษะเหล่านี้ไหม (proxy วัด logic/automation) · `[เลือกได้หลายข้อ]`
- สูตร Excel ซับซ้อน (VLOOKUP/Pivot) · Macro/Apps Script · ตั้งค่า automation (Zapier/Make ฯลฯ) · เขียน SQL query · ยังไม่เคยเลย

**13.** ยินดีทำ pre-task สั้นๆ ก่อนวันงานไหม (เช่น สมัครบัญชี ติดตั้งโปรแกรมล่วงหน้า) · `[ตัวเลือกเดียว: ยินดี · ขอทำหน้างาน · ไม่แน่ใจ]`
**14.** ความสบายใจกับเครื่องมือสายเทคนิค · `[สเกล 1–5: อึดอัดมาก → สบายมาก]` · ⚑

## ส่วนที่ 6 — อื่นๆ

**23.** มีข้อกังวล/อุปสรรคเรื่องการใช้ AI ไหม · `[เลือกได้หลายข้อ + อื่นๆ]`

### 🔑 หมายเหตุผู้จัด
- **ข้อ 7–14 คือกุญแจแบ่งกลุ่ม** — ถ้าคนส่วนใหญ่ตอบ "ไม่เคยเขียนโค้ด / ต้องให้ IT" → แบ่งกลุ่มหรือปรับความลึกหัวข้อเทคนิค แจ้งผู้ว่าจ้างก่อนวันงาน
- **ข้อ 8 = action item ทันที** — ใครตอบ "ต้องให้ IT" ให้ประสาน IT ผ่านผู้ประสานงาน (ฟอร์ม 1 · E7) ล่วงหน้า
- **ข้อ 18 = baseline** — เก็บไว้เทียบกับฟอร์ม 3
- **ข้อ 19–20** เอาไปทำ use case ตัวอย่างในคลาสให้ตรงงานจริงผู้เรียน
