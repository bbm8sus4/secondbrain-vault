คู่มือคำสั่ง Claude Code — ฉบับสอน

คำสั่ง built-in ที่มากับทุกเครื่อง (ยังไม่ต้องเซตอะไรเพิ่ม) · แตกเป็นหลายมิติ: ทำอะไร → ใช้เมื่อไร → บริบทจริง · รวมโหมด permission (กด shift + tab สลับ)

อ่านตารางยังไง: ช่อง ใช้เมื่อไร–เป้าหมาย คือ "อยากทำอะไรถึงใช้คำสั่งนี้" · ช่อง ตัวอย่าง คือสถานการณ์จริงที่เจอบ่อย เอาไปเล่าให้ผู้เรียนเห็นภาพได้เลย

/clear= /new /resetล้างแชท เริ่มใหม่หมดเปลี่ยนเรื่องคุย ไม่อยากให้เรื่องเก่ามากวน/เปลืองโทเคนคุยงาน A เสร็จ จะเริ่มงาน B /clear ก่อน กันมันจำเรื่องเก่าผิดๆ

/contextดูว่าใช้ context ไปเท่าไหร่แล้วอยากรู้ว่าเหลือที่ให้คุยอีกแค่ไหนรู้สึกมันเริ่มลืม เช็คว่าเต็มหรือยัง จะได้ /compact ทัน
/resume= /continueกลับไปทำงานที่คุยค้างไว้ปิดไปแล้ว อยากสานต่อของเดิมเมื่อวานแก้บั๊กค้าง /resume เลือกแชทเดิมมาทำต่อ
/rewind= /undo /checkpointย้อนกลับไปจุดก่อนหน้ามันทำพลาด อยากถอยกลับก่อนพังClaude แก้ผิดไฟล์ /rewind ย้อนกลับจุดที่ยังดีอยู่
/exportบันทึกบทสนทนาเป็นไฟล์อยากเก็บบันทึก/ส่งต่อให้คนอื่นดูคุยได้คำตอบดีๆ /export เก็บไว้เป็นเอกสาร

โหมด Permission — กด shift + tab วนสลับ

คุมว่ามันต้องขออนุญาตแค่ไหน
⏸ Plan mode

อ่าน/วิเคราะห์อย่างเดียว — ไม่แก้ไฟล์ ไม่รันอะไรที่เปลี่ยนแปลง จะเสนอแผนมาให้ดูก่อน รออนุมัติค่อยลงมือ

ใช้เมื่อ: งานใหญ่/สำคัญ/production · อยากให้คิดก่อนทำ กันพลาด · เหมาะกับมือใหม่ เสี่ยงต่ำสุด

Default

โหมดปกติ — ถามทุกครั้งก่อนแก้ไฟล์หรือรันคำสั่ง คุณกด yes/no เอง

ใช้เมื่อ: อยากคุมทุกก้าว · งานทั่วไปที่ยังไม่ไว้ใจ เสี่ยงต่ำ

accept edits

อนุมัติการแก้ไฟล์ให้อัตโนมัติ (ไม่ต้องกด yes ทุกครั้ง) — แต่ยังถามก่อนรันคำสั่ง shell ที่เสี่ยง

ใช้เมื่อ: งานแก้โค้ดเยอะๆ ที่ไว้ใจได้ อยากให้ไหลลื่น แต่ยังคุมคำสั่งอันตราย เสี่ยงกลาง

auto mode

อนุมัติเกือบทุกอย่างเอง รวมถึงรันคำสั่งด้วย — ทำงานต่อเนื่อง ไม่ค่อยหยุดถาม

ใช้เมื่อ: งานที่ไหลยาว อยากให้เดินเองจนจบ · เชื่อมือแล้ว เสี่ยงสูง

bypass permissions

ไม่ถามอะไรเลย อนุมัติทุกอย่าง รวมคำสั่งลบไฟล์/อันตราย — เร็วสุด แต่มันรันอะไรก็ได้โดยไม่เตือน

ใช้เมื่อ: มั่นใจจริงๆ หรืออยู่ใน sandbox/เครื่องแยกเท่านั้น · ไม่แนะนำกับมือใหม่ หรือเครื่องที่มีของสำคัญ เสี่ยงสูงสุด

โหมดแก้ไฟล์เอง?รันคำสั่งเอง?ความเสี่ยง
Planไม่ไม่ต่ำสุด
Defaultถามก่อนถามก่อนต่ำ
accept editsใช่ถามก่อนกลาง
autoใช่ใช่ (เกือบหมด)สูง
bypassใช่ใช่ (ทั้งหมด)สูงสุด
จำง่ายๆ: สีเขียว = ระวังสุด · สีแดง = ปล่อยสุด — กด shift + tab วนไปเรื่อยๆ จนเจอโหมดที่อยากใช้
