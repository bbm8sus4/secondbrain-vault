SLIDE 05 · WHY WE STAY OPTIMISTIC
5:00 — 6:30 · 1.5 นาที

ของเราดี — แค่ต้องดันให้ถึง

เหตุผล 4 ข้อที่ผมยังเชื่อว่าเราเดินถูกทาง

  • 1. หน่วยเศรษฐศาสตร์เก่งระดับโลก — Margin ต่อ 1 สลิป 82.4%. เครื่องเราเดิน, ของเราขายได้, ต้นทุนเราต่ำ. ปัญหาเดียวคือ ดัน volume ให้คุ้ม fixed cost.
  • 2. ลูกค้าใหม่ปีนี้คุณภาพสูงขึ้น — จำนวนใหม่ +14%, แต่ ยอดเงิน +60% เทียบปีก่อน. แปลว่า "คนน้อยลง แต่จ่ายมากขึ้น" — สัญญาณ enterprise/serious customer มาแทน hobby user.
  • 3. EasySlip API คือพระเอก — ARPU ฿986/คน, ทำรายได้ ~60% ของ Group, MRR ~฿2.5M. ของชิ้นนี้ดันต่อได้อีกไกล.
  • 4. Run-rate ปีอยู่ที่ ~98% ของเป้า — แปลว่าถ้าเราทำเหมือนเดิม + เร่งอีกเล็กน้อย → จบปีตามเป้า.
💡 สรุปเฟรมงาน: "เครื่องเราดีอยู่แล้ว — งานของเดือนหน้าคือใช้เครื่องนี้ให้คุ้ม"

ARPU (รายได้ต่อ active user/เดือน) · เห็นชัดว่าใครคือพระเอก

EasySlip API ทำรายได้ ~8 เท่า ของ Bot ต่อ active user · เห็นทาง upgrade ที่ชัด

Q&A PREP · คำถามที่อาจโดนถาม
เตรียมคำตอบไว้ก่อน

คำถามอาจโดนถาม + คำตอบที่เตรียมไว้

7 คำถามที่ COO มักโดนถามในประชุมประจำเดือน — เตรียมคำตอบไว้ ตอบได้ทันที

Q1
"ทำไม Profit Margin ติดลบ −47% — บริษัทขาดทุนหรือเปล่า?"
A: "ไม่ใช่ขาดทุนถาวรครับ — เพราะธุรกิจเราเป็นแบบ เติมเงินล่วงหน้า ลูกค้าเทียบจ่ายหนักช่วงกลาง-ปลายเดือน. ค่าใช้จ่ายคงที่ของเราออกตั้งแต่ต้นเดือน. นี่เป็น cycle ปกติของธุรกิจแบบนี้. ดูทั้งปี เรากำไร — ดูเฉพาะ 8 วันแรกของเดือนเท่านั้นที่จะเห็นติดลบ. และที่สำคัญ: กำไรต่อสลิป margin 82% — โครงสร้างต้นทุนของเราเก่งระดับโลก."
Q2
"ทำไม Bot รายได้ลด — ลูกค้าหายไปไหน?"
A: "Bot ลดเพราะ ARPU ของ Bot ต่ำ (~฿120/active user) เทียบ EasySlip API (~฿986/คน) ห่างกัน 8 เท่า. ลูกค้า Bot จำนวนมากกำลัง upgrade ไปใช้ API ที่ตอบโจทย์มากกว่า. เราต้องช่วยให้ path การ upgrade ราบรื่น — นี่เป็นหนึ่งในสิ่งที่เราจะทำในเดือนหน้า."
Q3
"ลูกค้าใหม่ลด −32% — เป็นเพราะ Marketing ไม่ดีเหรอ?"
A: "ผมยังไม่ฟันธงครับ — เรากำลังตรวจดูว่าเป็นเพราะ seasonality (ต้นเดือน), channel (โฆษณาช่อง X ลด), หรือ คุณภาพ traffic เปลี่ยน. ที่น่าสนใจคือ ลูกค้าใหม่ปีนี้จ่ายเงินมากขึ้น +60% YoY — แปลว่าคนน้อยลงแต่คุณภาพสูงขึ้น. ทีม Marketing & Growth จะรายงานผลตรวจสอบในสัปดาห์หน้า."
Q4
"56,000 inactive — เราจะปลุกอย่างไร?"
A: "หลายชั้นพร้อมกัน: (1) Email/LINE campaign ติดต่อกลับพร้อม offer พิเศษ (2) สำรวจว่า feature ไหนหายทำให้คนเลิกใช้ (3) ออกฟีเจอร์ใหม่ที่ trigger คนกลับมาเช็ก. ทีม CRM/Marketing/Product ทำงานร่วมกัน. ถ้าปลุก 5% ของ 56K = ~2,800 คน → เพิ่มรายได้หลายล้านบาท/เดือน. Lever ที่ใหญ่สุด ใช้ resource น้อยที่สุด."
Q5
"Run-rate ปีต่ำกว่าเป้า ฿2.35M — เราทำไม่ทันแล้วเหรอ?"
A: "1.0x ครับ — แปลว่าเราต้องเร่งแค่นิดเดียว. เฉลี่ย/วันตอนนี้ ~฿336K vs ที่ต้องการ ~฿346K — ขาดอีก ประมาณ 3%. ถ้า 3 priorities เดือนหน้าสำเร็จแม้แค่ครึ่งเดียว เราจะปิด gap นี้ได้สบาย. ผมไม่กังวล — แต่จะไม่ปล่อย."
Q6
"คู่แข่งล่ะ? มีใครมาแข่งหนักไหม?"
A: "ตลาด slip verification เริ่มมีหลายเจ้าเข้ามาเล่น แต่เราได้เปรียบ 3 อย่าง: (1) Volume scale — เราตรวจ ~2 ล้านสลิป/วัน ทำให้ต้นทุนต่อหน่วยเรา 3 สต. (2) Margin/สลิป 82% — ราคาเราสู้ได้ทุกระดับ (3) เครือข่ายลูกค้า 67K + brand ทั้ง Thunder/EasySlip ที่ตลาดเข้าใจ. ที่เราต้องระวัง: v2 API adoption ต้องเร่ง เพื่อให้ feature gap ไม่กว้างเกิน."
Q7
"แล้วผม/ทีมผม จะช่วยอะไรได้บ้าง?"
A: "ทุกคนมีบทบาทใน 3 priority นี้: Tech/Product → ออก feature ที่ดึง inactive กลับ + ลด churn EasySlip. Sales/Marketing → เจาะ top customers, แก้ acquisition channel, ทำ campaign ปลุก. Support/CS → improve onboarding, follow-up first-pay. Ops/Finance → monitor margin, alert ถ้าเห็นอะไรผิดปกติ. หาเข้าหาผมหรือ direct manager ได้เลยถ้าอยากคุยรายละเอียด."

Top Risks · 30 วันข้างหน้า

ความเสี่ยงที่ผมเฝ้าติดตามเอง · มี mitigation พร้อม · ไม่ใส่ใน slide สาธารณะ

R1

Margin -47% ลากยาวเกิน 30 วัน

CRITICAL

What: ถ้าปลายเดือน margin ยังติดลบ → cash burn เร็วกว่าที่ projection ถือ. fixed cost ปัจจุบัน ~฿4.5M/เดือน vs revenue MTD ~฿3M (8 วัน)

Why now: 8 วันแรกของเดือน อยู่ในช่วง pre-paid cycle, ปลายเดือนต้องเก็บเกินค่าใช้จ่ายให้ได้

Mitigation: (1) เร่ง upsell EasySlip API ในกลุ่ม top 50 (2) ลูกค้าใหม่ที่จ่ายแล้ว ส่ง follow-up เติมเงินเพิ่ม (3) audit fixed cost — มีอะไรลดได้?

Trigger to escalate: ถ้าวันที่ 25 พ.ค. margin ยังติดลบ > 30% → call emergency review

R2

EasySlip API churn จากคู่แข่งใหม่

HIGH

What: EasySlip API ทำเงิน 60% ของ Group · ARPU ฿986 · ถ้าลูกค้าใหญ่ย้ายไปคู่แข่ง → impact หนัก

Why now: ตลาด slip verification เริ่มมีหลายเจ้า · feature parity กำลังลด

Mitigation: (1) Identify top 10 lighest-revenue customer · ติดต่อสำรวจ satisfaction (2) ออก feature exclusive ที่คู่แข่งยังไม่มี (3) lock-in contract ระยะยาว 6-12 เดือน + ส่วนลด

Trigger to escalate: ถ้ามีลูกค้า top-10 cancel 1 ราย → escalate ในสัปดาห์

R3

v1 API deprecation ดึงไม่ขึ้น

MEDIUM

What: v1: 7,461 calls/วัน vs v2: 37 calls/วัน (~0.5%) · ถ้า v1 ค้างนาน → maintenance burden + feature gap ใหญ่ขึ้น

Why now: ลูกค้าที่ใช้ v1 ยังเยอะ · migration พา effort

Mitigation: (1) DevRel ทำ migration guide + sample code (2) Sunset timeline ชัด (Q4) (3) Incentive ลูกค้า migrate (credit เพิ่ม)

Trigger to escalate: ถ้าผ่านไป 2 เดือนแล้ว v2 ยังต่ำกว่า 10% → re-strategize

R4

Token expiry incidents ค้าง

LOW

What: 2 token alerts active · ถ้าหมดอายุไม่ทัน → service disruption ลูกค้า

Mitigation: SRE rotate ภายใน 24 ชม. + setup auto-renewal คอนทรัก

⚠️ เก็บ risk register นี้ไว้ส่วนตัว — ใช้สำหรับ Board/CEO sync, ห้ามแชร์ในประชุม All-Hands
