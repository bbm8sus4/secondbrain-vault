# 📝 Q&A Prep · 7 คำถามที่อาจโดนถาม

### Q1: ทำไม Profit Margin ติดลบ −47% — บริษัทขาดทุนเหรอ?
**A:** ไม่ใช่ขาดทุนถาวรครับ — เพราะธุรกิจเราเป็นแบบ **เติมเงินล่วงหน้า** ลูกค้าเทียบจ่ายหนักช่วงกลาง-ปลายเดือน ค่าใช้จ่ายคงที่ของเราออกตั้งแต่ต้นเดือน นี่เป็น cycle ปกติของธุรกิจแบบนี้ ดูทั้งปีเรากำไร ดูเฉพาะ 8 วันแรกของเดือนเท่านั้นที่จะเห็นติดลบ และที่สำคัญ: **กำไรต่อสลิป margin 82%** — โครงสร้างต้นทุนของเราเก่งระดับโลก

### Q2: ทำไม Bot รายได้ลด — ลูกค้าหายไปไหน?
**A:** Bot ลดเพราะ ARPU ของ Bot ต่ำ (~฿120/active user) เทียบ EasySlip API (~฿986/คน) ห่างกัน 8 เท่า ลูกค้า Bot จำนวนมากกำลัง **upgrade** ไปใช้ API ที่ตอบโจทย์มากกว่า เราต้องช่วยให้ path การ upgrade ราบรื่น — นี่เป็นหนึ่งในสิ่งที่เราจะทำในเดือนหน้า

### Q3: ลูกค้าใหม่ลด −32% — เป็นเพราะ Marketing ไม่ดีเหรอ?
**A:** ผมยังไม่ฟันธงครับ — เรากำลังตรวจดูว่าเป็นเพราะ **seasonality** (ต้นเดือน), **channel** (โฆษณาช่อง X ลด), หรือ **คุณภาพ traffic** เปลี่ยน ที่น่าสนใจคือ **ลูกค้าใหม่ปีนี้จ่ายเงินมากขึ้น +60% YoY** — แปลว่าคนน้อยลงแต่คุณภาพสูงขึ้น ทีม Marketing & Growth จะรายงานผลตรวจสอบในสัปดาห์หน้า

### Q4: 56,000 inactive — เราจะปลุกอย่างไร?
**A:** หลายชั้นพร้อมกัน: **(1)** Email/LINE campaign ติดต่อกลับพร้อม offer พิเศษ **(2)** สำรวจว่า feature ไหนหายทำให้คนเลิกใช้ **(3)** ออกฟีเจอร์ใหม่ที่ trigger คนกลับมาเช็ก ทีม CRM/Marketing/Product ทำงานร่วมกัน ถ้าปลุก 5% ของ 56K = ~2,800 คน → เพิ่มรายได้หลายล้านบาท/เดือน **Lever ใหญ่สุด ใช้ resource น้อยที่สุด**

### Q5: Run-rate ปีต่ำกว่าเป้า ฿2.4M — เราทำไม่ทันแล้วเหรอ?
**A:** **1.0x** ครับ — แปลว่าเราต้องเร่งแค่นิดเดียว เฉลี่ย/วันตอนนี้ ~฿336K vs ที่ต้องการ ~฿346K — ขาดอีก **ประมาณ 3%** ถ้า 3 priorities เดือนหน้าสำเร็จแม้แค่ครึ่งเดียว เราจะปิด gap นี้ได้สบาย ผมไม่กังวล — แต่จะไม่ปล่อย

### Q6: คู่แข่งล่ะ? มีใครมาแข่งหนักไหม?
**A:** ตลาด slip verification เริ่มมีหลายเจ้าเข้ามาเล่น แต่เราได้เปรียบ **3 อย่าง**: **(1)** Volume scale — เราตรวจ ~2 ล้านสลิป/วัน ทำให้ต้นทุนต่อหน่วยเรา 3 สต. **(2)** Margin/สลิป 82% — ราคาเราสู้ได้ทุกระดับ **(3)** เครือข่ายลูกค้า 67K + brand ทั้ง Thunder/EasySlip ที่ตลาดเข้าใจ ที่เราต้องระวัง: **v2 API adoption** ต้องเร่ง เพื่อให้ feature gap ไม่กว้างเกิน

### Q7: แล้วผม/ทีมผม จะช่วยอะไรได้บ้าง?
**A:** ทุกคนมีบทบาทใน 3 priority นี้:
- **Tech/Product** → ออก feature ที่ดึง inactive กลับ + ลด churn EasySlip
- **Sales/Marketing** → เจาะ top customers, แก้ acquisition channel, ทำ campaign ปลุก
- **Support/CS** → improve onboarding, follow-up first-pay
- **Ops/Finance** → monitor margin, alert ถ้าเห็นอะไรผิดปกติ

หาเข้าหาผมหรือ direct manager ได้เลยถ้าอยากคุยรายละเอียด
