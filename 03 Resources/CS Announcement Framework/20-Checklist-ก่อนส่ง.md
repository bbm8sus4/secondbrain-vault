# 20 · Checklist ก่อนส่ง — ไล่ให้ครบทุกข้อ ทุกครั้ง

> คัดลอกทั้งบล็อกไปแปะในแชทกลุ่มทีมตอนขออนุมัติ แล้วติ๊กให้ครบ

```
ประกาศ: [ชื่อเรื่อง]  ·  แบรนด์: [....]  ·  Tier: [T1/T2/T3]

เนื้อหา
[ ] เติมช่อง {{...}} ครบทุกจุด ไม่มีเหลือค้างในข้อความ
[ ] วัน/เดือน/ปี + เวลา ระบุแบบเต็ม (ไม่ใช้ "พรุ่งนี้/คืนนี้")
[ ] อ่านออกเสียง 1 รอบ — สะกดถูก ไม่มีคำแปลก โทนสุภาพลงท้าย "ค่ะ"
[ ] ลิงก์ทุกลิงก์กดแล้วเปิดได้จริง (ทดสอบจากมือถือ)
[ ] มีช่องทางติดต่อกลับของแบรนด์นั้นท้ายข้อความ
[ ] ไม่มีข้อมูลภายใน (ชื่อพนักงาน / vendor / สาเหตุที่ยังไม่ยืนยัน)

กลุ่มเป้าหมาย + ช่องทาง
[ ] ระบุกลุ่มเป้าหมายชัด + จำนวนโดยประมาณ: [กลุ่ม.... / ~.... ราย]
[ ] ช่องทางตรงตาม 01-Channel-Matrix สำหรับประเภทนี้
[ ] LINE OA: เช็คโควตา broadcast คงเหลือพอ
[ ] SMS: นับตัวอักษร ≤ 70 (ไทย) = 1 เครดิต · เกิน = เช็คงบก่อน
[ ] SMS การตลาด (T3): มีข้อความ "ยกเลิกพิมพ์ STOP" แล้ว
[ ] เวลาส่งเหมาะสม (หลีกเลี่ยงก่อน 09:00 / หลัง 20:00 ยกเว้น T1 ด่วน)

อนุมัติ
[ ] C อนุมัติเป็นลายลักษณ์อักษรแล้ว (แคปไว้)  — ยกเว้น T1 จังหวะ "รับทราบปัญหา"
[ ] ถ้ามีชดเชย/ของแถม: C อนุมัติรายละเอียดการชดเชยแล้วโดยเฉพาะ
[ ] ถ้ามีภาพ: artwork เวอร์ชันสุดท้ายผ่านการตรวจแล้ว (ตัวเลข/วันที่บนภาพตรงกับข้อความ)

หลังส่ง
[ ] ลงบันทึกใน 21-Log-ประกาศ ทันที
[ ] เฝ้า inbox/กลุ่มลูกค้า 1 ชม.แรก — คำถามซ้ำให้ทำคำตอบกลางแจกทีม
```

## จุดพลาดบ่อย (จากเคสจริง)

- **ตัวเลขบนภาพไม่ตรงกับข้อความ** — แก้ข้อความแล้วลืมแก้ภาพ ตรวจคู่กันเสมอ
- **ส่งซ้ำกลุ่มเดิม** — ก่อน broadcast แบบ per-chat ให้สแกนรายการที่ส่งไปแล้วก่อนทุกครั้ง (เคย EasyBot Affiliate ส่งค้าง 250/1245 แล้วต้อง resume)
- **ลืมแจ้งจบงาน** — ลูกค้าจำได้ว่าประกาศปิดปรับปรุง แต่ไม่รู้ว่าเสร็จหรือยัง → จังหวะ "เสร็จแล้ว" ห้ามข้าม
