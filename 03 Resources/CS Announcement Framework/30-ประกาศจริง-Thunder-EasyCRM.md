# 30 · ประกาศจริง (Thunder / EasyCRM) — ที่หนิงใช้จริงในสนาม

> คลังประกาศต้นฉบับจากทีม CS (หนิง) — ใช้ **โทน / คำ / โครง** จากไฟล์นี้เป็น "เสียงทีม" · เทมเพลตในไฟล์ 10–13 = โครงมาตรฐาน · ไฟล์นี้ = ตัวอย่างจริงตามเคส

## Flow รับปัญหา (จากต้นฉบับ)

### ปัญหาบอทไม่ตรวจสอบ — ขั้นตอนเบื้องต้น
1. ขอบริการที่มีปัญหา → ตรวจการเชื่อมต่อ
2. ช่องทางการเชื่อมต่อ + ขออนุญาตลูกค้าเข้าไปทดสอบสลิป
   - LineOA: เพิ่มเพื่อน
   - Line กลุ่ม: ขอเข้ากลุ่ม

### ตรวจแล้วไม่ทราบสาเหตุ → แนบให้ครบ
- รูปสลิปทั้งหมด (ถ้ามีประวัติสลิป ไม่ต้องขอลูกค้า)
- ช่วงเวลาที่เกิดปัญหา
- รายละเอียดปัญหา

**ปัญหาการเชื่อมต่อ:** แก้ได้เอง → แจ้งลูกค้ากลับได้เลย

### ข้อมูลที่ต้องขอลูกค้า (แยกตามสินค้า)

**Line Chat BOT** — ยูสเซอร์ · เซอร์วิส/สาขา · แชทลูกค้า · การเชื่อมต่อ (LineOA/Line กลุ่ม)

**ThunderAPI** — ข้อความมาตรฐาน:
> สวัสดีค่ะ รบกวนขอข้อมูลให้แอดมิน ดังนี้
> - ขอ Email ที่ลงทะเบียนใช้งาน
> - Status Error
> - Message Error
> - ภาพประกอบ
>
> หลังจากแจ้งข้อมูลเรียบร้อย แอดมินประสานงานเจ้าหน้าฝ่ายเทคนิกตรวจสอบให้ค่ะ

### ปัญหาหลัก 3 กลุ่ม
1. ระบบล่ม
2. ธนาคาร
3. ช่วงรับสลิปเยอะ

### ประสานงาน
- กลุ่ม **Dav X Support** · @ หนิง / โด้ / พี่แบงค์
- โด้ = ช่วยดูแลลูกค้าเวลาแอดมินรับมือไม่ไหว
- **ประกาศเฉพาะเมื่อลูกค้าแจ้งเข้ามาเยอะ** — ช่องทาง: Facebook Page + LINE OA broadcast

## Thunder Solution — เคสจริง

### 1. KTB (กรุงไทย) — ข้อมูลใน QR สลิปผิดพลาด

> เรียน คุณลูกค้าที่เคารพ
>
> ขออภัยในความไม่สะดวกที่เกิดขึ้นค่ะ 🙏
>
> จากการตรวจสอบพบว่า ปัญหาดังกล่าวเกิดจาก **ธนาคารกรุงไทย (KTB)** ในขั้นตอนการส่งข้อมูลธุรกรรมไปยัง **ระบบกลาง ITMX** (ระบบกลางของธนาคารแห่งประเทศไทยที่ใช้ในการเชื่อมโยงธุรกรรมระหว่างธนาคารต่าง ๆ)
>
> 📌 **กรณีที่ยอดเงินใน QR Code ในสลิปแสดงข้อมูลผิดพลาด**
>
> Thunder Solution:
> - ระบบของเราตรวจสอบข้อมูลผ่าน QR Code ที่ธนาคารต้นทาง (ในกรณีนี้คือ KTB) อัปเดตมาในสลิป
> - หากธนาคารกรุงไทยส่งข้อมูลที่คลาดเคลื่อน บอทตรวจสอบสลิปของเราจะแสดงผลตามข้อมูลที่ได้รับจากสลิปนั้น
>
> **แนวทางการแก้ไข**
> ขณะนี้ทางเราได้ดำเนินการส่งเรื่องไปยัง ระบบกลาง ITMX เพื่อให้ทำการตรวจสอบและประสานงานกับธนาคารกรุงไทย เพื่อแก้ไขข้อมูลให้ถูกต้องเรียบร้อย
>
> ✅ หากมีความคืบหน้าจากการตรวจสอบของ ITMX ทางเราจะแจ้งให้ลูกค้าทราบทันที
>
> ขอขอบคุณลูกค้าทุกท่านที่เข้าใจและให้ความร่วมมือเสมอมาค่ะ
> หากมีข้อสงสัยเพิ่มเติมสามารถติดต่อฝ่ายบริการลูกค้าได้ตลอดเวลานะคะ
>
> ด้วยความเคารพ
> **ฝ่ายบริการลูกค้า Thunder Solution 🙏**

### 7. Cloudflare มีปัญหา

**แจ้ง:**
> ⚠️ **ระบบ Cloudflare มีปัญหา** ⚠️
>
> Cloudflare ซึ่งเป็นบริการเครือข่ายระดับโลกที่ทำหน้าที่เป็นตัวกลางระหว่างผู้ใช้และเว็บไซต์ เกิดความผิดปกติ ส่งผลให้ระบบบางส่วนของ Thunder Solution ไม่สามารถให้บริการได้ชั่วคราว
>
> ขณะนี้ทีมงานของเรากำลังประสานงานกับทาง Cloudflare อย่างเร่งด่วน เพื่อแก้ไขปัญหาให้กลับมาทำงานปกติโดยเร็วที่สุด
> เราจะอัปเดตความคืบหน้าให้ทุกท่านทราบทันทีเมื่อมีความเปลี่ยนแปลง
>
> ขออภัยในความไม่สะดวก
> ทีมงาน Thunder Solution 🙏

**ปิดเคส:**
> ✅ ระบบของ Thunder Solution และหน้าเว็บไซต์กลับมาให้บริการได้ตามปกติแล้ว
>
> ขณะนี้ระบบทุกส่วนของ Thunder Solution ทั้งการใช้งาน API, Dashboard และการเข้าสู่เว็บไซต์ สามารถใช้งานได้ตามปกติ
>
> ⚠️ จากที่ตรวจสอบปัญหาเกิดขึ้นจากเครือข่ายของ Cloudflare เบื้องต้นได้รับการแก้ไขแล้วเป็นที่เรียบร้อย
> ทีมงานยังคงเฝ้าติดตามการทำงานของระบบอย่างต่อเนื่อง เพื่อให้การใช้งานเป็นไปอย่างราบรื่นที่สุด

### 9. LINE OA ล่ม

**แจ้งสั้น:**
> ขณะนี้ระบบของ Line Official ขัดข้อง
> ไม่สามารถให้บริการลูกค้าได้ชั่วคราว
>
> หากระบบสามารถกลับมาใช้งานได้แล้ว ทางเราจะรีบให้บริการลูกค้าโดยเร็วที่สุด
> ต้องขออภัยในความไม่สะดวกด้วยนะคะ

**แจ้งแบบละเอียด (สลิปแสดงผลผิดปกติ):**
> ⚠️ **การแสดงผลตรวจสอบสลิปผิดปกติชั่วคราว**
>
> ขณะนี้ บอทตรวจสอบสลิปบน LINE Official Account (LINE OA) มีปัญหาแสดงผลชั่วคราว
> เนื่องจาก LINE OA มีปัญหาทางเทคนิค ส่งผลให้ข้อมูลสลิปบางรายการไม่สามารถแสดงผลได้อย่างถูกต้อง
>
> เพื่อป้องกันความคลาดเคลื่อนในการยืนยันรายการ ทีมงานอยู่ระหว่างติดตามอย่างใกล้ชิด จะแจ้งให้ทราบทันทีเมื่อระบบกลับมาใช้งานได้ตามปกติ
>
> ระหว่างนี้ ท่านกรุณาตรวจสอบยอดเงินผ่าน **SMS จากธนาคาร** หรือ **ประวัติธุรกรรมในแอปธนาคาร**
>
> ขออภัยในความไม่สะดวก
> ทีมงาน Thunder Solution

## สังเกตโทน (Voice signature ของทีมนี้)

- ลงท้าย **"ค่ะ / นะคะ"** เกือบทุกข้อความ (โทน CS ผู้หญิง)
- ใช้ **🙏 emoji** ท้ายแบรนด์บ่อยมาก (ประจำ Thunder)
- ⚠️ / ✅ / 📢 / 📌 / ⏰ เป็นตัวเปิดหัวเรื่องประจำ
- **ยืนยัน "ไม่ใช่ปัญหาที่ระบบเรา"** เมื่อเป็นปัญหาธนาคาร/LINE OA/Cloudflare — บ่อยและตรงไปตรงมา
- แนะนำ **"ตรวจ SMS หรือประวัติธุรกรรมในแอปธนาคาร"** เป็นทางเลี่ยงระหว่างล่ม
- ตอนปิดเคส: ✅ + สรุปว่าใช้งานได้ปกติ + เบอร์ติดต่อ
- Line 1 ปิดท้าย: **"ขออภัยในความไม่สะดวกมา ณ ที่นี้ 🙏"** เป็นวลีประจำ
