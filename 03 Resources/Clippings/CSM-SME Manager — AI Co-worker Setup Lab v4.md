## สร้าง AI ผู้จัดการการตลาดส่วนตัวของคุณ ทีละ Prompt

คู่มือ **step-by-step** เดิน 6 Prompt เพื่อปั้น Claude ให้เป็น **Marketing Manager AI** — ตอนนี้อยู่โหมด **🏢 SME ทั่วไป** ใช้ได้กับธุรกิจ SME ทุกแบบ (Campaign → Leads → CAC → Conversion) กดสลับ 🏍/🏢 หรือเปลี่ยน **บทบาท** ด้านบนได้เลย แล้วเนื้อหา Prompt จะเปลี่ยนตาม — กดติ๊กกลางการ์ดเพื่อเก็บความคืบหน้า (แยกตามบทบาท)

### 🎯 ทำไมต้องเดินทีละ Prompt?

## ภาพรวมตำแหน่ง + KPI ที่ต้องดู

อ่านให้เข้าใจก่อนเริ่ม — Prompt ทั้งหมดจะอ้างอิงหน้าที่และ KPI ชุดนี้

หน้าที่หลักของ Marketing Manager

เวอร์ชันกลางสำหรับ SME ทุกอุตสาหกรรม — วางแผนแคมเปญ, วัด ROI ช่องทาง, สร้าง lead และดูแลแบรนด์

📊

**KPI ที่ต้องดูแล** — ชุดนี้คือหัวใจของ Dashboard ที่จะสร้างใน Prompt 5 และวิเคราะห์ใน Prompt 6

| KPI | นิยาม | Target |
| --- | --- | --- |
| Marketing ROI / ROAS | รายได้ที่เกิด ÷ งบที่ใช้ | **≥ ตามเป้า** |
| Cost per Lead (CPL) | งบ ÷ จำนวน lead | **ลดลง** |
| Leads / MQL | จำนวน lead ที่สร้างได้ | **≥ เป้า** |
| CAC | ต้นทุนได้ลูกค้า 1 ราย | **< LTV/3** |
| Conversion Rate | Lead → Customer | **เพิ่มขึ้น** |
| Channel Mix | สัดส่วนยอด/lead ตามช่องทาง | **สมดุล** |
| Engagement / Reach | ยอดมีส่วนร่วม/การเข้าถึง | **โตต่อเนื่อง** |
| Spend vs Budget | % งบที่ใช้จริง vs แผน | **± 5%** |

## Prompt 1 — สัมภาษณ์ (Interview)

วางในแชทธรรมดา (Default Sonnet, ไม่ต้องใน Project) → Claude ถามทีละข้อ → ตอบครบแล้วไป Prompt 2

Copy บล็อกด้านล่าง → วางในแชท → ตอบทีละข้อ (พิมพ์แค่ A/B/0/E) จนครบ 4 หมวด

PROMPT 1 — Interview Marketing · SME ทั่วไป

```
ผมกำลังจะให้คุณเป็น Marketing Manager ส่วนตัวของผม ก่อนเริ่ม ช่วยสัมภาษณ์ผมเพื่อเก็บข้อมูลครบทุกอย่างที่ต้องใช้สร้าง SKILL.md

กฎการถาม (สำคัญมาก):
• ถามทีละคำถามเท่านั้น รอผมตอบก่อนแล้วค่อยถามต่อ
• แต่ละตัวเลือกต้องอยู่คนละบรรทัด ห้ามอยู่บรรทัดเดียวกัน
• รูปแบบที่ต้องใช้ทุกครั้ง:

คำถามที่ X — [หัวข้อ]
[คำถาม]

A — [ตัวเลือก]
B — [ตัวเลือก]
C — [ตัวเลือก]
D — [ตัวเลือก]
0 — ยังไม่รู้ / ไม่แน่ใจ
E — ไม่มีในนี้

• ทุกคำถามต้องมีตัวเลือก 0 เสมอ
• ถ้าผมตอบ 0 ให้จดเรื่องนั้นไว้เป็น gap ที่ต้องหาข้อมูลเพิ่ม
• ถ้าผมตอบ E ให้เสนอตัวเลือกใหม่สำหรับคำถามนั้นเท่านั้น
• ผมจะตอบแค่ตัวอักษรเดียว เช่น A หรือ 0 หรือ E

ถามให้ครบทั้ง 4 หมวดนี้ (แต่ละหมวดถามทีละข้อ):

หมวด 1 — บริบทการตลาดปัจจุบัน: ช่องทางที่ใช้, งบต่อเดือน, ขนาดทีม, วิธีเก็บ lead, ระบบ/เครื่องมือที่ใช้ (CRM, ads manager, analytics)

หมวด 2 — งานที่อยากให้ Marketing AI ช่วยมากที่สุด: เช่น campaign planning / content calendar / วิเคราะห์ ROI ช่องทาง / สรุปผลแอด / competitor watch / email/CRM marketing

หมวด 3 — สไตล์การตอบที่ชอบ: เช่น Action-oriented / number-driven / มี framework (AARRR, funnel) / เสนอไอเดียคอนเทนต์เป็นชุด

หมวด 4 — KPI / ตัวเลขที่อยากเน้นดูเป็นพิเศษ: เช่น ROAS / CPL / CAC / Conversion rate / MQL→SQL / LTV:CAC

พอตอบครบทุกหมวดแล้ว ก่อนจบให้สรุปทุกข้อที่ผมตอบ 0 พร้อมแบ่งเป็น 2 กลุ่ม:
• "ต้องหาก่อนสร้างระบบ" — ถ้าไม่รู้จะทำ Dashboard ถูกต้องไม่ได้ พร้อมบอกว่าหาได้จากไหน
• "รอได้" — ระบบทำงานได้แม้ยังไม่มีข้อมูลนี้ ค่อยเติมทีหลังได้

เริ่มคำถามแรกได้เลยครับ
```

⚠️

**อย่าใจร้อน:** ตอบทีละข้อให้ครบทั้ง 4 หมวดก่อน อย่าข้ามไป Prompt 2 จนกว่า Claude จะสรุป gap ("ต้องหาก่อน" / "รอได้") ให้ครบ

## Prompt 2 — สร้าง SKILL.md

Universal Template · ใช้ต่อในแชทเดิมหลังตอบ Prompt 1 ครบ → ได้ไฟล์ SKILL.md กด Save Skill ได้ทันที

ต่อในแชทเดียวกับ Prompt 1 — Claude จะสรุปทุกอย่างเป็นไฟล์ skill ที่ Save ได้

```
จากทุกอย่างที่เราคุยกันมา ช่วยเขียน SKILL.md ให้ผมเลยครับ
ต้องมี 5 ส่วนนี้:

1. ข้อมูลธุรกิจ + บทบาทของ ผู้จัดการการตลาด (สรุปจากที่คุยกันมา ให้ครบและถูกต้อง)
2. หน้าที่ของ ผู้จัดการการตลาด AI (ต้องช่วยผมทำอะไรบ้าง)
3. วิธีตอบ (ตรง ชัด ใช้ตาราง ชี้ปัญหาก่อนเสมอ)
4. KPI / ตัวเลขที่อยากเน้นดูเป็นพิเศษ
5. สิ่งที่ต้องช่วยผมหาก่อน (จากกลุ่ม "ต้องหาก่อนสร้างระบบ" ที่สรุปไว้)

ระบุว่าแต่ละอันสำคัญยังไง หาได้จากไหน และควรหาก่อนหรือรอได้

เขียนให้ครบ พร้อมใช้งานได้เลยครับ

⚠️ ไฟล์ต้องเริ่มต้นด้วย YAML frontmatter นี้เสมอ (ถ้าไม่มี Save ไม่ได้):

---
name: marketing
description: Marketing Manager ส่วนตัวที่รู้จักธุรกิจของผมโดยเฉพาะ
version: 1.0
---

⚠️ name ต้องเป็นตัวพิมพ์เล็กล้วน (marketing ไม่ใช่ Marketing) เพราะคำสั่งเรียก skill เป็น case-sensitive — ถ้าตั้งตัวใหญ่จะพิมพ์ /marketing ไม่ติด

แล้วส่งเป็นไฟล์ SKILL.md ผมจะได้กด Save skill ได้ทันทีในแชทนี้
```

💡

**name ต้องเป็นตัวพิมพ์เล็กล้วน เป็นภาษาอังกฤษ ห้ามมี space/อักขระพิเศษ** — แต่ละบทบาทใช้ name ของตัวเอง (เช่น `marketing`) เพื่อให้พิมพ์ `/marketing` เรียกได้ (ถ้าตั้งตัวใหญ่จะขึ้น "Unknown command")

## Prompt 3 — ตั้ง Project

Universal Template · ใช้หลัง Save SKILL.md → ได้ Project Description + Instructions ไปวางใน Claude Project

ขั้นที่ 1 — สร้าง Claude Project

ตั้ง Project แยกไว้เป็นบ้านถาวรของบทบาทนี้ — แนบ Skill ที่ Save ไว้รอบที่แล้ว แล้วค่อยรัน Prompt สัมภาษณ์ในขั้นถัดไป

1. เปิด Claude → กด **\+ New Project** (เมนูซ้าย) → ตั้งชื่อ `Marketing Manager`
2. ใส่คำอธิบายสั้น ๆ ในช่อง "What are you trying to achieve?" เช่น "ผู้จัดการการตลาดของบริษัท"
3. แนบ **Skill / SKILL.md** ที่ Save ไว้จาก Prompt 2 เข้า Project (Project knowledge / Files) เพื่อให้ Claude รู้จักธุรกิจตั้งแต่ต้น
4. เปิดแชทแรกใน Project ที่เพิ่งสร้าง → พร้อมไปขั้นที่ 2
📌

ทำ Project ให้เสร็จ **ก่อน** วาง Prompt ข้างล่าง — Prompt สัมภาษณ์จะให้ผลดีที่สุดเมื่อรันอยู่ในแชทแรกของ Project ที่มี Skill แนบไว้แล้ว

ขั้นที่ 2 — สัมภาษณ์รอบ Project (วาง Prompt)

วาง Prompt นี้ในแชทแรกของ Project → ตอบ 4 หมวด → ได้ Project Description + Instructions ไปวางใน Project

PROMPT 3 — Project Setup Interview

```
ผมกำลังจะสร้าง Claude Project เพื่อให้คุณทำงานใน context ของบริษัทผมโดยเฉพาะ ในบทบาท ผู้จัดการการตลาด

คุณมี Skill ที่รู้จักธุรกิจผมอยู่แล้ว (ผมสร้างไว้รอบที่แล้ว) แต่ Project Instructions จะทำให้คุณรู้จักบริษัทผมในระดับที่ลึกกว่านั้น เช่น โครงสร้างองค์กร เป้าหมายปีนี้ ข้อมูลที่จะ upload เข้า Project และสิ่งที่อยากให้โฟกัสเป็นพิเศษ

ขั้นแรกสุด — ห้ามถามซ้ำ:
1. อ่าน Skill ที่แนบไว้ใน Project ก่อน แล้วสรุปสั้น ๆ ว่า "จาก Skill ผมรู้เรื่องนี้แล้ว: ..." (ธุรกิจ, บทบาท, KPI, ปัญหาหลัก ฯลฯ)
2. อะไรที่ Skill ตอบได้แล้ว ห้ามถามซ้ำ — ให้ดึงมาใช้เลย
3. ถามเฉพาะข้อมูล "ใหม่" ระดับ Project ที่ Skill ยังไม่มี หรือที่ต้อง update ให้เป็นปัจจุบันเท่านั้น
4. ถ้าหมวดไหน Skill ครอบคลุมครบแล้ว ให้ข้ามไปเลย แล้วบอกผมว่า "หมวดนี้ใช้ข้อมูลจาก Skill ได้เลย ไม่ต้องถาม"

กฎการถาม (สำคัญมาก):
• ถามทีละคำถาม รอผมตอบก่อนแล้วค่อยถามต่อ
• ถามเฉพาะสิ่งที่ Skill ยังไม่รู้ — ถ้ารู้แล้วให้ข้าม
• แต่ละตัวเลือกต้องอยู่คนละบรรทัด
• รูปแบบที่ต้องใช้ทุกครั้ง:

คำถามที่ X — [หัวข้อ]
[คำถาม]

A — [ตัวเลือก]
B — [ตัวเลือก]
C — [ตัวเลือก]
D — [ตัวเลือก]
0 — ยังไม่รู้ / ไม่แน่ใจ
E — ไม่มีในนี้

ถ้าผมตอบ E ให้เสนอตัวเลือกใหม่สำหรับคำถามนั้นเท่านั้น

ครอบคลุม 4 หมวดนี้ (ข้ามหมวด/คำถามที่ Skill ตอบได้แล้ว):

หมวด 1 — context บริษัทสำหรับงานการตลาด: เป้าหมายปีนี้, ปัญหาหลักที่กำลังเผชิญ, ทีม/คนที่เกี่ยวข้องกี่คน (ส่วนไหนอยู่ใน Skill แล้วให้ใช้เลย ถามเฉพาะที่ขาดหรือต้อง update)

หมวด 2 — ข้อมูลที่จะอยู่ใน Project: จะ upload ไฟล์อะไรเข้ามาบ้าง เช่น รายงานแคมเปญ / Ads report (Meta/Google/TikTok) / Lead export / สรุปยอด engagement

หมวด 3 — งานหลักที่อยากให้ ผู้จัดการการตลาด AI ทำใน Project นี้: เช่น วิเคราะห์ทุกเดือน / ช่วยตัดสินใจ / ดู trend ระยะยาว

หมวด 4 — ข้อตกลงการทำงาน: รูปแบบ output ที่ชอบ, เรื่องที่ห้ามตอบนอกขอบเขต (ถ้า Skill ระบุวิธีตอบไว้แล้ว ใช้ต่อได้เลย)

พอครบแล้ว ส่งมาให้ผม 2 อย่างนี้:

① Project Description (1–2 ประโยค) สำหรับใส่ในช่อง "What are you trying to achieve?" ของ Claude Project

② Project Instructions ฉบับเต็ม สำหรับใส่ในช่อง Instructions ของ Project
```

ขั้นที่ 3 — วาง Description + Instructions ลงใน Project

เอา 2 อย่างที่ Claude ส่งมาในขั้นที่ 2 ไปตั้งค่าใน Project จริง — เสร็จแล้ว AI จะจำ context นี้ทุกแชทใน Project

1. เปิด Project `Marketing Manager` → กด **⚙️ Settings / Edit** ของ Project
2. วาง **① Project Description** ลงในช่อง "What are you trying to achieve?"
3. วาง **② Project Instructions** ฉบับเต็มลงในช่อง **Instructions** ของ Project
4. กด **Save** → เปิดแชทใหม่ใน Project เพื่อทดสอบว่า Claude เข้าใจ context
📌

Instructions อยู่ระดับ Project — จะมีผลกับ **ทุกแชท** ใน Project นี้ ไม่ต้องวาง context ซ้ำทุกครั้ง ทำครั้งเดียวจบ

```
ช่วยสร้างโครงสร้างโฟลเดอร์สำหรับงาน ผู้จัดการการตลาด ให้ผมหน่อยครับ ขอให้เป็นระบบ ทำงานซ้ำได้ทุกเดือน และหาของเจอง่าย
```
