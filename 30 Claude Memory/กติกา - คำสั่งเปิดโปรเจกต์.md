When user says "เปิด <something> ทำงานต่อ", follow these 3 modes:

**Mode 1 — Direct path** (fastest, never miss):
- User gives explicit path like `~/Desktop/EasyBOT/finance-webapp`
- Just `cd` and start working. No questions.

**Mode 2 — Named alias** (user says it once, I remember forever):
- User gives a name like `easybot.finance` or `EasyBOT Finance`
- I search local disk first, find the candidate path
- **Ask user to confirm the path before any clone**
- Save the name→path mapping to memory so next time the same name resolves directly to Mode 1

**Mode 3 — Not found**:
- If I can't find it locally, I MUST ask before `git clone`
- Never auto-clone without explicit user approval. Every time.

**Why:** User wants speed when path is given, persistence when name is given (one-time setup), and zero surprise clones into their working directory.

**How to apply:** Trigger on any message starting with "เปิด" + project reference + "ทำงานต่อ" (or similar). Branch by which info user gave. When saving name→path mappings, store them as small `alias_<name>.md` memories or append to a single `project_aliases.md` index.

**Tool/skill aliases:**
- `cx` → เปิด Codex (invoke `codex:setup` skill)
