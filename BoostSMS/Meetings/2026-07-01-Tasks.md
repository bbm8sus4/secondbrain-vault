## Task Details

### 🔴 P0 — ก่อน AIS go-live (ก.ค. 2569)

#### #1 [บอม] วิเคราะห์เงื่อนไข AIS + จัดทำแบบฟอร์มกรอกข้อมูลลูกค้า (Lean)
- **Owner:** บอม
- นำเงื่อนไขของ AIS มาแยกขั้นตอน
- ออกแบบเอกสาร "แบบฟอร์มการกรอกข้อมูลสำหรับลูกค้า"
  - **Lean:** ไม่ถามซ้ำ ไม่มีฟิลด์เกินจำเป็น
  - **ใช้งานง่าย:** จบใน flow เดียว
- Deliverable: PDF/HTML แบบฟอร์ม + workflow diagram

#### #2 [Product] ออกแบบ LeanFlow Back-office + export CSV/Excel/PDF
- ออกแบบ workflow หลังบ้านให้สั้นที่สุด (LeanFlow)
- ฟังก์ชัน export เป็น CSV / Excel / PDF ส่งให้ AIS ตามรอบ
- **AIS ไม่เก็บข้อมูลลูกค้า** — data sovereignty อยู่ที่เรา 100%
- Deliverable: back-office spec + export module

#### #4 [Infra] เตรียมกลไก failover สลับท่อส่ง AIS ↔ Dtac
- Mechanism สลับ provider อัตโนมัติเมื่อท่อใดท่อหนึ่งมีปัญหา
- ต้องมี health check + fallback policy
- Deliverable: failover architecture + monitoring plan

#### #8 [Finance] ตรวจสอบต้นทุน AIS + margin
- ยืนยันต้นทุน: **0.29 บาท / SMS**
- เป้า **500,000 SMS = 155,150฿** (หัก 3% ตามเงื่อนไข)
- อัปเดต `10-Cost-Structure-BoostSMS.xlsx` + `11-Commission-System-BoostSMS.xlsx`
- Deliverable: pricing sheet ใหม่ + margin analysis

---

### 🟡 P1 — Critical infra

#### #3 [Frontend] เพิ่ม Progress Bar แสดงสถานะการส่งให้ลูกค้าเห็น
- UI Progress Bar ในหน้าลูกค้า
- Real-time: queued → sending → delivered / failed
- Deliverable: UI component + backend event stream

#### #5 [Infra] เพิ่มท่อส่งเป็น 2 เส้น เพื่อเร่งความเร็ว OTP
- Parallel dispatch ลด latency OTP
- Load balancing + monitoring per lane
- Deliverable: dual-lane dispatcher

#### #6 [Infra] Internal pipeline — บริหารจัดการท่อส่งเอง
- ย้ายมาจัดการท่อส่งเอง: queue, retry, routing, throttle
- ประสิทธิภาพสูงสุด แทนพึ่ง provider layer
- Deliverable: internal pipeline service

#### #7 [Report] ปรับ format รายงานให้สอดคล้องกับ AIS
- รีวิว report ทุกตัวในระบบ
- ปรับ column / field / รูปแบบ ให้ตรง spec AIS
- Deliverable: updated report templates + reconciliation doc

---

### 🟠 P2 — Compliance guardrail

#### #9 [Finance] ระบบคืนเครดิตจาก Callback ที่มีปัญหา + Log System
- คืนเครดิตอัตโนมัติเมื่อ callback ส่งไม่สำเร็จ
- Log ทุกรายการคืนเครดิต + รายการทางการเงิน
- Audit trail ตรวจสอบย้อนหลังได้
- Deliverable: refund automation + audit log module

#### #10 [Compliance] Refund policy สอดคล้องกับระเบียบ กสทช.
- รีวิว refund policy ให้รัดกุม
- ตรงตามระเบียบ กสทช.
- ประสานฝ่ายกฎหมายถ้าจำเป็น
- Deliverable: policy document + terms + approval workflow

---

### 🟢 P3 — Feature expansion (หลัง AIS นิ่ง)

#### #11 [Product] เพิ่มฟีเจอร์ CRM ดูแลลูกค้าใน BoostSMS
- เพิ่มระบบ CRM เข้าไปเป็นฟีเจอร์ใน BoostSMS
- **ประเด็นที่ต้องเคลียร์ก่อน:**
  1. Overlap กับ EasyCRM มั้ย — แยก product หรือ shared component?
  2. อยู่ใน package ไหน — Trial ให้ใช้มั้ย หรือ paid-only?
  3. Impact ต่อ `02-Packages-Pricing` และ `03-Sales-Commission`
- เริ่มจาก write spec + user story
- Deliverable: PRD + technical spec

#### #12 [Content] เขียน content เพิ่มสำหรับฟีเจอร์ CRM ใน BoostSMS
- Sale page copy
- Feature description
- Onboarding guide
- Sales pitch
- Positioning: BoostSMS ทำอะไรได้บ้างนอกเหนือจากส่ง SMS
- Deliverable: content pack (copy + assets)
