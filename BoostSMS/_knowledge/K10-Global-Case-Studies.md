# K10 · Global SMS Case Studies

> **TL;DR (EN):** Five world-class SMS programs to learn from — Domino's (order tracking), Walgreens (Rx + appointment), Starbucks (loyalty), Sephora (segmentation), and Amtrak (real-time alerts). Common patterns: SMS as a "moment of truth" channel for transactional/time-sensitive events, layered with light promotional touches at known engagement windows. None of these brands SPAM with SMS — they earn it.
>
> **สรุป (TH):** 5 case study SMS ระดับโลก — Domino's (tracking), Walgreens (สุขภาพ + นัด), Starbucks (loyalty), Sephora (segmentation), Amtrak (real-time alert) จุดร่วม: SMS = ช่องของ "moment ที่สำคัญ" + promo เบา ๆ ตามเวลาที่เหมาะ ทุกแบรนด์ **ไม่สเปม** — สร้าง trust ก่อน

## 1. 🍕 Domino's Pizza — SMS Order Tracking ที่กลายเป็น Brand Asset

### ภาพรวม
- ใช้ SMS เป็น "order tracker" หลัก
- ลูกค้า opt-in ตอนสั่ง → ได้ status update ตลอด

### Flow ตัวอย่าง
```
SMS 1: "Order confirmed - $24.50 - Cooking now! 🍕"
SMS 2: "Out for delivery - Driver: John - Track: bit.ly/x"
SMS 3: "Delivered! Enjoy! Rate at: bit.ly/r"
```

### Key Insight
- ลด support call 30%+ (ลูกค้าไม่ต้องโทรถามสถานะ)
- เพิ่ม return purchase 25%
- "Pizza Tracker" กลายเป็น cultural icon

### บทเรียนสำหรับ BoostSMS
- ✅ Transactional SMS = brand experience
- ✅ Real-time status update > generic promotion
- ✅ End with CTA (rate, reorder)

## 2. 💊 Walgreens — Rx + Appointment Reminders

### ภาพรวม
- Pharmacy + healthcare chain (US, 9,000+ store)
- ใช้ SMS heavy สำหรับ:
  - Rx ready for pickup
  - Refill reminder
  - Vaccine appointment
  - Health screening

### Flow ตัวอย่าง
```
SMS: "Walgreens: Your prescription is ready
Pickup by 9pm tonight or it will be returned
Address: 123 Main St"
```

### ตัวเลข
- 50%+ ของลูกค้า opt-in
- Refill SMS เพิ่ม compliance 30%
- ลด no-show ของ vaccine 40%

### บทเรียน
- ✅ SMS for healthcare = compliance driver
- ✅ Specific time/place ใน SMS
- ✅ Personalized > generic

## 3. ☕ Starbucks — SMS + App Loyalty Hybrid

### ภาพรวม
- SMS ใช้ส่งให้ member ที่ไม่มี app (older demographic)
- App-centric แต่ SMS เป็น backup channel

### Use Cases
- Double Star Days (specific date)
- Free birthday drink reminder
- Loyalty point expiry alert
- New product launch

### Flow ตัวอย่าง
```
SMS: "Starbucks: Double Star Day TOMORROW!
Earn 2x on every order 10am-9pm
Use mobile order: app.starbucks.com"
```

### บทเรียน
- ✅ SMS ใช้กับ tier ที่ engage ต่ำ (ไม่ใช่ everyone)
- ✅ Time-bound promo เน้น urgency
- ✅ Drive ลูกค้าไป primary channel (app)

## 4. 💄 Sephora — Hyper-Segmented SMS

### ภาพรวม
- Beauty Insider program (25M+ members)
- ส่ง SMS แค่ tier สูง (VIB, Rouge) ที่ opt-in
- Content เฉพาะคน

### Segmentation Strategy
- Skin type (oily/dry/combo)
- Past purchase categories (skincare/makeup/fragrance)
- Tier (Insider/VIB/Rouge)
- Birthday month
- Visit recency

### Flow ตัวอย่าง
```
SMS: "Hi [Name] - your favorite serum is back in stock!
Reserve for in-store pickup today only
Address: [nearest store]"
```

### ตัวเลข
- SMS opt-in rate ของ Rouge tier = 80%+
- Conversion rate of personalized SMS = 12%
- CLV ของ SMS-opted member สูงกว่า non-opted 40%

### บทเรียน
- ✅ Segment คือทุกอย่าง — personalize hard
- ✅ Top tier = consent ง่ายกว่า
- ✅ SMS = exclusive feeling

## 5. 🚂 Amtrak — Real-time Service Alerts

### ภาพรวม
- US rail service
- SMS สำคัญสำหรับ travel alerts

### Use Cases
- Departure delay
- Platform change
- Service disruption
- Booking confirmation
- Mobile ticket

### Flow ตัวอย่าง
```
SMS: "Amtrak Update: Train 175 to NYP
delayed 30 min. New departure 4:45 PM
Track: amtrak.com/track"
```

### บทเรียน
- ✅ SMS = trust channel for critical updates
- ✅ Brevity in alerts (no fluff)
- ✅ Link to detail page (don't cram into SMS)

## 7. Pattern ที่ทุกเจ้าทำ

| Pattern | ทำไม work |
|---|---|
| **Real-time status** | ลด support, สร้าง trust |
| **Time-bound urgency** | 24h promo, expiring alerts |
| **Personalization** | Name + history + location |
| **Brand-first identification** | "From: BrandName" ทันที |
| **Short URL** | Mobile click friction |
| **One CTA per SMS** | ลด decision paralysis |
| **Opt-out always** | Trust + compliance |

## 8. Sequence Strategy ที่ใช้กัน

### E-commerce Standard
1. Order placed (SMS)
2. Payment confirmed (SMS)
3. Shipped + tracking (SMS)
4. Out for delivery (SMS)
5. Delivered (SMS)
6. Review reminder (SMS, 3 วันหลัง)

### Subscription / SaaS
1. Welcome SMS (sign-up)
2. Day 3: Quick start tip
3. Day 7: Engagement check
4. Day 14: Premium upgrade offer
5. Day 30: Renewal reminder

### Healthcare
1. Appointment scheduled
2. 7 วันก่อน: Reminder
3. 24h ก่อน: Confirm/reschedule
4. 2h ก่อน: Location + arrival
5. Post-visit: Survey + follow-up

## 9. SMS ที่ FAIL — Case Studies

### ❌ T-Mobile (2020) — SMS Spam Lawsuit
- ส่ง marketing SMS ลูกค้าที่ opt-out
- โดน class action lawsuit
- ปรับ $5M+

### ❌ Spotify (2018) — Region Mismatch
- ส่ง SMS ภาษาผิด → ลูกค้า confuse
- High opt-out spike

### ❌ Indian Bank Scam SMS (ongoing)
- คนทั่วไปได้ "ธนาคารแจ้งบัญชีถูกระงับ"
- Sender ID ปลอม
- บทเรียน: ทำไม sender ID registration สำคัญ

### บทเรียนรวม
- ✅ Honor opt-out **ทันที**
- ✅ Localization สำคัญ
- ✅ Sender authentication ป้องกัน phishing impersonation
