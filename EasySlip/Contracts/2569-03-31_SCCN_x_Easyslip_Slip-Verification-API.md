# เนื้อหาบันทึกข้อตกลง

## ข้อ 1. ขอบเขตการให้บริการ

พันธมิตรด้านการเชื่อมต่อระบบตกลงใช้บริการ Slip Verification API และผู้ให้บริการ API ตกลงให้บริการ Slip Verification API ให้แก่พันธมิตรด้านการเชื่อมต่อระบบ เพื่อเชื่อมต่อระบบ Super rOS ของพันธมิตรด้านการเชื่อมต่อระบบ ตามรายละเอียดการใช้บริการช่องทางการเชื่อมต่อ Easy API for Partners ที่ระบุในเอกสารแนบท้ายหมายเลข 1 ที่แนบท้ายบันทึกข้อตกลงฉบับนี้ เข้ากับระบบของผู้ให้บริการ API เพื่อให้สามารถรองรับการตรวจสอบข้อมูลใน QR Code หรือ Barcode หรือรหัสอื่นใด ที่ระบบของผู้ให้บริการ API รองรับให้ตรวจสอบได้ (ซึ่งต่อไปนี้จะเรียกว่า "รหัสชำระเงิน") ที่ปรากฏบนหลักฐานการชำระเงิน (Slip) ของผู้ชำระค่าสินค้า/บริการ (ซึ่งต่อไปนี้จะเรียกว่า "ผู้ชำระเงิน") ครอบคลุมถึงการสาธิตคุณสมบัติ (Feature Demonstration) การทดสอบระบบ (System testing) การพัฒนาทางเทคนิค และการให้บริการเชิงพาณิชย์แก่ผู้ชำระเงิน ทั้งสองฝ่ายจึงตกลงกันโดยมีรายละเอียดดังนี้

### 1.1 พัฒนาและดำรงการเชื่อมระบบ
พันธมิตรด้านการเชื่อมต่อระบบตกลงพัฒนาและดำรงการเชื่อมระบบ Super rOS เข้ากับระบบของผู้ให้บริการ API ตามรูปแบบและวิธีที่ผู้ให้บริการ API กำหนด โดยพันธมิตรด้านการเชื่อมต่อระบบตกลงเป็นผู้รับผิดชอบค่าใช้จ่ายต่าง ๆ ที่เกิดจากการพัฒนาและเชื่อมต่อระบบดังกล่าว รวมถึงค่าใช้จ่ายในการบริหารจัดการระบบภายในของพันธมิตรด้านการเชื่อมต่อระบบเองทั้งสิ้น

### 1.2 รหัสประจำผู้ให้บริการ API
การใช้ Consumer ID หรือ API key หรือ Consumer secret หรือรหัสผ่าน หรือ access token หรือเครื่องมืออื่นใดที่พันธมิตรด้านการเชื่อมต่อระบบใช้เป็นเครื่องมือในการรับ-ส่งคำสั่ง/คำขอ/รายงาน/ผลการดำเนินการต่าง ๆ ผ่านบริการ Slip Verification API (ซึ่งต่อไปนี้ หากไม่เรียกชื่อใดชื่อหนึ่งจะเป็นการเฉพาะเจาะจง จะรวมเรียกว่า "รหัสประจำผู้ให้บริการ API") ผู้ให้บริการ API ตกลงยินยอมปฏิบัติตามเงื่อนไขดังต่อไปนี้

**1.2.1** หากการใช้บริการ Slip Verification API ตามบันทึกข้อตกลงฉบับนี้ ผู้ให้บริการ API ได้กำหนดให้ต้องใช้บริการผ่านรหัสประจำพันธมิตรด้านการเชื่อมต่อระบบแล้ว พันธมิตรด้านการเชื่อมต่อระบบจะต้องสร้าง/เปลี่ยนแปลงรหัสประจำพันธมิตรด้านการเชื่อมต่อระบบ ตามเงื่อนไขและขั้นตอนที่ผู้ให้บริการ API กำหนด และจะต้องระบุรหัสประจำพันธมิตรด้านการเชื่อมต่อระบบ ตามเงื่อนไขและขั้นตอนที่ผู้ให้บริการ API กำหนด

**1.2.2** พันธมิตรด้านการเชื่อมต่อระบบจะต้องเก็บรักษารหัสประจำพันธมิตรด้านการเชื่อมต่อระบบไว้ในที่ปลอดภัยและถือเป็นความลับเฉพาะตัวของพันธมิตรด้านการเชื่อมต่อระบบเท่านั้น การเปิดเผยรหัสประจำของพันธมิตรด้านการเชื่อมต่อระบบ ถือเป็นการปฏิบัติผิดบันทึกข้อตกลงฉบับนี้ หากเกิดความเสียหายประการใด จากการที่พันธมิตรด้านการเชื่อมต่อระบบเปิดเผยหรือกระทำการใดๆ จนทำให้บุคคลอื่นล่วงรู้ หรือได้รับรหัสประจำผู้ให้บริการ API หรือรหัสประจำผู้ให้บริการ API สูญหายหรือถูกกระทำการ ผู้ให้บริการ API จะต้องรับผิดชอบในความเสียหายที่เกิดขึ้นก่อนที่ผู้ให้บริการ API จะดำเนินการอายัด หรือระงับการใช้รหัสประจำพันธมิตรด้านการเชื่อมต่อระบบนั้น

**1.2.3** ในกรณีที่รหัสประจำพันธมิตรด้านการเชื่อมต่อระบบเกิดการสูญหาย/ถูกโจรกรรม/ถูกระงับ หรือพันธมิตรด้านการเชื่อมต่อระบบ ลืมรหัสประจำของพันธมิตรด้านการเชื่อมต่อระบบ พันธมิตรด้านการเชื่อมต่อระบบสามารถส่งรายละเอียดมาที่ Email Address: hello@easyslip.com ตลอด 24 ชั่วโมง เพื่อดำเนินการตามขั้นตอนที่ผู้ให้บริการ API กำหนด

**1.2.4** การกระทำการใดๆ ซึ่งรวมถึง (แต่ไม่จำกัดเพียง) การตกลงยอมรับ/การแก้ไข/การเปลี่ยนแปลง/การเพิ่มเติม ของเงื่อนไข/บริการ/อัตราค่าธรรมเนียม/บันทึกข้อตกลงฉบับนี้ ที่ผู้ให้บริการ API ได้แจ้งให้ทราบผ่านช่องทางต่างๆ ไม่ว่าจะเป็นการกระทำของผู้ให้บริการ API เองก็ดี หรือการกระทำของบุคคลอื่นใดก็ดี หากได้กระทำไปโดยผ่านการใช้รหัสประจำพันธมิตรด้านการเชื่อมต่อระบบแล้ว พันธมิตรด้านการเชื่อมต่อระบบตกลงให้ถือว่าถูกต้องสมบูรณ์ และมีผลผูกพันพันธมิตรด้านการเชื่อมต่อระบบ เสมือนหนึ่งได้กระทำโดยพันธมิตรด้านการเชื่อมต่อระบบเอง รวมทั้งให้ถือว่าการที่พันธมิตรด้านการเชื่อมต่อระบบได้ลงลายมือชื่ออิเล็กทรอนิกส์ไว้ให้แก่ผู้ให้บริการ API เพื่อเป็นหลักฐานในการทำธุรกรรมผ่านบริการในครั้งนั้นๆ โดยถูกต้องสมบูรณ์แล้ว นับตั้งแต่เวลาที่มีการยืนยันทำธุรกรรม และพันธมิตรด้านการเชื่อมต่อระบบ ตกลงให้ผู้ให้บริการ API สามารถใช้ข้อมูลการทำธุรกรรมดังกล่าวเป็นต้นฉบับเอกสารที่ใช้เป็นพยานหลักฐานในการพิสูจน์ว่าพันธมิตรด้านการเชื่อมต่อระบบได้ทำธุรกรรมนั้น และในการดำเนินการทางกฎหมายได้ทุกประการ ทั้งนี้พันธมิตรด้านการเชื่อมต่อระบบ ตกลงยอมรับและรับทราบถึงความเสี่ยงที่เกี่ยวกับการใช้บริการผ่านช่องทางอิเล็กทรอนิกส์ เนื่องจากพันธมิตรด้านการเชื่อมต่อระบบสามารถทำธุรกรรมด้วยตนเองได้โดยไม่จำต้องมีเอกสาร หรือหลักฐานใดยืนยันเพิ่มเติม และ/หรือมอบให้แก่ผู้ให้บริการ API อีกแต่ประการใด เว้นแต่รายการที่ผู้ให้บริการ API กำหนดให้ต้องทำเอกสาร หรือหลักฐานเพิ่มเติมเพื่อให้ถูกต้องตามวิธีการ และประเพณีของผู้ให้บริการ API ในเรื่องนั้น ๆ พันธมิตรด้านการเชื่อมต่อระบบจะต้องทำเอกสารหรือหลักฐาน (หากมี) ตามที่ผู้ให้บริการ API กำหนดด้วย

### 1.3 การเชื่อมต่อโปรแกรมประยุกต์ (API Integration)
พันธมิตรด้านการเชื่อมต่อระบบมีสิทธิการเชื่อมต่อระบบ Easy API for Partners เข้ากับระบบ Super rOS ของพันธมิตรด้านการเชื่อมต่อระบบ เพื่อสนับสนุนให้ร้านค้าภายใต้แพลตฟอร์มของพันธมิตรด้านการเชื่อมต่อระบบ สามารถดำเนินการตรวจสอบผลการโอนเงินผ่านหลักฐานการชำระเงิน (Slip Verification) ได้โดยอัตโนมัติ

**1.3.1 โครงสร้างบัญชีหลักและบัญชีย่อย (Master–Sub Branch Structure):** ผู้ให้บริการ API ตกลงจัดให้มีระบบบัญชีหลัก (Master Account) เพื่อให้พันธมิตรด้านการเชื่อมต่อระบบสามารถบริหารจัดการ และควบคุมดูแลบัญชีร้านค้าปลีกย่อย (Sub-branch) จำนวนหลายรายการพร้อมกันภายใต้บัญชีหลักเพียงบัญชีเดียวได้

**1.3.2 การบริหารจัดการโควตาการใช้งาน (Quota Management):** ผู้ให้บริการ API จัดให้มีระบบสนับสนุนการกำหนด จัดสรร และควบคุมปริมาณโควตาการใช้งาน API ในระดับรายสาขา (Branch Level) โดยแยกส่วนออกจากโควตาหลักของบริษัท ทั้งนี้ ระบบจะดำเนินการตัดโควตาแบบทันที (Real-time) และมีระบบแจ้งเตือนไปยังพันธมิตรด้านการเชื่อมต่อระบบเมื่อโควตาการใช้งานครบจำนวนตามที่กำหนด

**1.3.3 เข้าถึงข้อมูลการใช้งานผ่าน API (Usage Data via API):** ผู้ให้บริการ API ตกลงจัดให้มีระบบที่รองรับการแลกเปลี่ยนข้อมูลการใช้งานผ่าน API เพื่อแสดงผลข้อมูลดังกล่าวบนแผงควบคุม (Dashboard) ของพันธมิตรด้านการเชื่อมต่อระบบโดยตรง เพื่อประโยชน์ในการตรวจสอบและติดตามผล

**1.3.4 การบริหารจัดการรหัสผ่าน API (API Key Management):** พันธมิตรด้านการเชื่อมต่อระบบมีสิทธิในการสร้าง จัดเก็บ เรียกใช้งาน หรือบริหารจัดการรหัสผ่าน API (API Key) ของร้านค้าปลายทางภายใต้การดูแลของตนเองได้อย่างอิสระ ผ่านระบบที่ผู้ให้บริการ API จัดเตรียมไว้ให้

### 1.4 ความรับผิดเมื่อข้อมูลไม่ครบถ้วน
หากข้อมูลที่ผู้ให้บริการได้รับ หรือรายละเอียดของรหัสชำระเงินที่ผู้ชำระเงินได้รับจากพันธมิตรด้านการเชื่อมต่อระบบ (แล้วแต่กรณี) ไม่ครบถ้วนถูกต้อง หรือไม่สมบูรณ์ แตกต่างจากข้อมูลรหัสชำระเงินที่ผู้ชำระเงินได้ส่งให้พันธมิตรด้านการเชื่อมต่อระบบ และ/หรือผู้ชำระเงินส่งให้พันธมิตรด้านการเชื่อมต่อระบบ และ/หรือที่พันธมิตรด้านการเชื่อมต่อระบบส่งให้ผู้ให้บริการ API ซึ่งทำให้ผู้ให้บริการ API ตรวจสอบข้อมูลของรหัสชำระเงินที่ปรากฏบนหลักฐานการชำระเงิน (Slip) ผิดพลาด หรือแตกต่างจากผลการตรวจสอบข้อมูลรายละเอียดของรหัสชำระเงินที่ระบบของผู้ให้บริการ API ส่งให้พันธมิตรด้านการเชื่อมต่อระบบ (แล้วแต่กรณี) พันธมิตรด้านการเชื่อมต่อระบบจะต้องเป็นผู้รับผิดชอบในความผิดพลาดดังกล่าวแต่ผู้เดียว

### 1.5 คุณสมบัติและระบบจัดการ (System Features)

**1.5.1 แผงควบคุมสำหรับพันธมิตร (Customer Panel):** ผู้ให้บริการ API ตกลงจัดให้มีระบบบริหารจัดการสำหรับพันธมิตรด้านการเชื่อมต่อระบบ โดยมีฟังก์ชันการทำงานหลักดังนี้:
- **การบริหารจัดการสาขา (Branch Management):** ระบบสำหรับการสืบค้นและบริหารจัดการข้อมูลรายชื่อสาขาภายใต้บัญชีหลัก (Master Account) ของพันธมิตรด้านการเชื่อมต่อระบบ
- **การควบคุมโควตาการใช้งาน (Quota Control):** ระบบสำหรับการกำหนด จัดสรร เพิ่ม แก้ไข หรือระงับปริมาณโควตาการใช้งาน API ในระดับรายสาขา (Branch Level)
- **การสืบค้นข้อมูลบัญชี (Account Search):** ระบบสำหรับสืบค้นและตรวจสอบข้อมูลชื่อบัญชีธนาคารของผู้ใช้งานผ่านระบบของผู้ให้บริการ API

**1.5.2 การเชื่อมต่อข้อมูล API สำหรับภาคธุรกิจ (B2B API Integration):** ผู้ให้บริการ API ตกลงจัดเตรียมจุดเชื่อมต่อระบบ (Endpoint) เฉพาะสำหรับพันธมิตรด้านการเชื่อมต่อระบบ โดยดำเนินการแยกส่วนการทำงาน (Isolation) ออกจากจุดเชื่อมต่อสำหรับบุคคลทั่วไป พร้อมทั้งปรับปรุงรูปแบบข้อมูลตอบกลับ (Response Data) ให้รองรับมาตรฐานข้อมูลระดับองค์กร (Enterprise Data)

**1.5.3 มาตรฐานความปลอดภัยและการแยกบัญชี (API Security & Account Segregation):** ผู้ให้บริการ API จัดให้มีการบริหารจัดการรหัสประจำผู้ให้บริการ API (API Key) แบบ 2 ระดับ ได้แก่ รหัสเฉพาะสำหรับพันธมิตรด้านการเชื่อมต่อระบบ และรหัสสำหรับร้านค้าย่อยแต่ละราย ทั้งนี้ ร้านค้าย่อยแต่ละรายสามารถเพิ่มบัญชีธนาคารเพื่อตรวจสอบหลักฐานการชำระเงิน (Slip) ได้ครอบคลุม 18 ธนาคารพาณิชย์ รวมถึงระบบ PromptPay, KShop, แม่มณี, กรุงศรีมั่งมี และ True Wallet โดยระบบจะดำเนินการแยกฐานข้อมูลบัญชีธนาคารของแต่ละร้านค้าย่อยออกจากกันโดยเด็ดขาด

### 1.6 การใช้งานเชิงพาณิชย์ (Commercial Implementation)

**1.6.1 อัตราค่าธรรมเนียมและการชำระเงิน:** การใช้งาน API ในระบบใช้งานจริง (Production) ภายหลังสิ้นสุดช่วงสิทธิพิเศษ จะคิดอัตราค่าธรรมเนียม **0.125 บาทต่อรายการ (รวมภาษีมูลค่าเพิ่ม)** โดยผู้ให้บริการ API จะดำเนินการออกใบแจ้งหนี้รายเดือนตามรายละเอียดการใช้บริการจริง

**1.6.2 การแก้ไขอัตราค่าธรรมเนียม:** การแก้ไขเปลี่ยนแปลงอัตราค่าธรรมเนียมการใช้บริการ หรือเงื่อนไขการชำระเงิน คู่สัญญาจะต้องตกลงร่วมกันเป็นลายลักษณ์อักษร

**1.6.3 การใช้บริการของลูกค้าปลายทาง:** กรณีลูกค้าปลายทาง (End User) มีความประสงค์ใช้งาน API เชิงพาณิชย์นอกเหนือจากขอบเขตระบบ Super rOS ลูกค้าปลายทางดังกล่าวจะต้องดำเนินการลงนามขอใช้บริการและรับรหัสประจำผู้ให้บริการ API จากผู้ให้บริการ API โดยตรง

### 1.7 ห้ามให้ผู้อื่นมาร่วมใช้
พันธมิตรด้านการเชื่อมต่อระบบตกลงว่า จะมีให้ผู้อื่นมาร่วมใช้บริการภายใต้บันทึกข้อตกลงฉบับนี้ เว้นแต่จะได้รับอนุญาตเป็น ลายลักษณ์อักษรจากผู้ให้บริการ API ก่อน

### 1.8 ข้อมูลพันธมิตรและข้อจำกัด
พันธมิตรด้านการเชื่อมต่อระบบยอมรับว่า ข้อมูลพันธมิตรด้านการเชื่อมต่อระบบซึ่งรวมถึง (แต่ไม่จำกัดเพียง) ข้อมูล รูปภาพ สัญลักษณ์ เครื่องหมายการค้า เครื่องหมายบริการ หรือเครื่องหมายใดๆ (ซึ่งต่อไปนี้เรียกว่า "ข้อมูลพันธมิตรด้านการเชื่อมต่อระบบ") ที่พันธมิตรด้านการเชื่อมต่อระบบใช้ค้า และ/หรือนำเข้าสู่ระบบใดๆ ที่แสดงผลในส่วนของข้อมูลส่วนตัวพันธมิตรด้านการเชื่อมต่อระบบและในส่วนของพื้นที่หน้าจอการทำงาน ตลอดจนการแสดงรูปภาพต่างๆ ที่ผู้ให้บริการ API กำหนดและเลือกภาพที่ใช้แสดง พันธมิตรด้านการเชื่อมต่อระบบตกลงว่าข้อมูลพันธมิตรด้านการเชื่อมต่อระบบดังกล่าว ไม่มีลักษณะ หรือก่อให้เกิดภาพ สัญลักษณ์ หรือเครื่องหมายตามตัวอย่าง ดังต่อไปนี้:

- **1.8.1** ไม่ได้รับความยินยอมจากเจ้าของลิขสิทธิ์ หรือเจ้าของทรัพย์สินทางปัญญา
- **1.8.2** ไม่ได้รับความยินยอมจากบุคคลในภาพ หรือเจ้าของภาพส่วนบุคคล
- **1.8.3** ก่อให้เกิดความแตกแยก หรือมีประเด็นทางการเมือง
- **1.8.4** ไม่สุภาพ ไม่เหมาะสม หรือยั่วยวน ลามกอนาจาร
- **1.8.5** รูปภาพดารานักแสดง นักดนตรี บุคคลผู้มีชื่อเสียง หรือนักกีฬา ทั้งนี้เว้นแต่เป็นบุคคลที่ทำงานให้กับโปรแกรม Co-branding ที่ได้รับอนุญาตแล้ว
- **1.8.6** ขัดต่อความสงบเรียบร้อยของประชาชนหรือประเพณีทางวัฒนธรรมหรือศาสนา
- **1.8.7** เกี่ยวข้องการการพนัน เครื่องดื่มแอลกอฮอล์ทุกประเภทและบุหรี่
- **1.8.8** เครื่องหมายสัญลักษณ์ประเทศ พระบรมฉายาลักษณ์หรือพระบรมสาทิสลักษณ์ของพระมหากษัตริย์ และพระราชินีหรือราชวงศ์
- **1.8.9** เครื่องหมายของหน่วยงานราชการต่างๆ

ทั้งนี้ พันธมิตรด้านการเชื่อมต่อระบบตกลง ข้อมูลพันธมิตรด้านการเชื่อมต่อระบบที่ปรากฏ พันธมิตรด้านการเชื่อมต่อระบบเป็นเจ้าของและ/หรือเจ้าของลิขสิทธิ์ และ/หรือมีสิทธิโดยชอบในการนำสิ่งดังกล่าวมาใช้สำหรับการใช้บริการ Slip Verification API ตามบันทึกข้อตกลงฉบับนี้ โดยในกรณีที่พันธมิตรด้านการเชื่อมต่อระบบ ร้องขอเอกสารหลักฐานเพิ่มเติมที่ยืนยันความเป็นเจ้าของลิขสิทธิ์ และ/หรือสิทธิโดยชอบในการใช้งานข้อมูลพันธมิตรด้านการเชื่อมต่อระบบดังกล่าว พันธมิตรด้านการเชื่อมต่อระบบจะจัดส่งเอกสารหลักฐานเพิ่มเติมดังกล่าวให้ผู้ให้บริการ API ตามที่ ผู้ให้บริการ API ร้องขอ นอกจากนี้ หากพันธมิตรด้านการเชื่อมต่อระบบนำข้อมูล รูปภาพ สัญลักษณ์ เครื่องหมายการค้า เครื่องหมายบริการ หรือเครื่องหมายใดๆ ของบุคคลอื่นมาโดยไม่ได้รับอนุญาตหรือโดยผิดกฎหมาย พันธมิตรด้านการเชื่อมต่อระบบตกลงรับผิดชอบในความเสียหายที่เกิดขึ้นทั้งหมด โดยผู้ให้บริการ API ไม่มีส่วนเกี่ยวข้องใดๆ ในการกระทำผิดดังกล่าว และหากผู้ให้บริการ API ได้รับความเสียหายใดๆ พันธมิตรด้านการเชื่อมต่อระบบตกลงขดใช้ค่าเสียหายที่เกิดขึ้นให้แก่ผู้ให้บริการ API จนครบถ้วน

### 1.10 ความลับของข้อมูล
พันธมิตรด้านการเชื่อมต่อระบบจะไม่เปิดเผยข้อมูลใด ๆ ที่เกี่ยวกับข้อมูลของผู้ชำระเงิน และระบบข้อมูลของผู้ให้บริการ API แก่บุคคลอื่นใด และจะไม่นำไปใช้กับธุรกิจอื่น เว้นแต่จะได้รับความยินยอมจากผู้ให้บริการ API หรือเจ้าของข้อมูลเป็นลายลักษณ์อักษรก่อน ในกรณีที่เจ้าของข้อมูลหรือบุคคลอื่นใดได้รับความเสียหาย เนื่องจากการเปิดเผยข้อมูลของผู้ให้บริการ API ไม่ว่าด้วยเหตุจงใจหรือไม่ก็ตาม พันธมิตรด้านการเชื่อมต่อระบบตกลงขดใช้ค่าเสียหายที่เกิดขึ้นทั้งสิ้น

### 1.11 การปฏิบัติตามกฎหมาย

### 2.2 อัตราค่าบริการพิเศษ
เมื่อร้านค้าใหม่ของพันธมิตรครบกำหนดอายุการใช้งานโควตาตรวจสอบสลิปฟรี หรือใช้โควตาตรวจสอบสลิปหมดก่อนถึงระยะเวลาครบกำหนดอายุการใช้งานตามข้อ 2.1 ผู้ให้บริการ API กำหนดอัตราค่าธรรมเนียมการใช้บริการคงที่พิเศษสำหรับพันธมิตรด้านการเชื่อมต่อระบบที่ **0.125 บาท/รายการ** พร้อมระบบเรียกเก็บค่าธรรมเนียมแบบใช้งานก่อนและชำระภายหลัง (**Use First, Pay Later**)

### 2.3 การสนับสนุนทางเทคนิค (Priority Support)
พันธมิตรด้านการเชื่อมต่อระบบ จะได้รับสิทธิการสนับสนุนด้านเทคนิค (Technical Support) ตลอด 24 ชั่วโมง ในฐานะคู่สัญญาลำดับความสำคัญสูง (Priority Integration Partner) ตลอดระยะเวลาที่บันทึกข้อตกลงนี้มีผลใช้บังคับ

### 2.4 การส่งเสริมการตลาดและความร่วมมือด้านกรณีศึกษา
พันธมิตรด้านการเชื่อมต่อระบบ ตกลงแสดงข้อความ **"Slip Verified by Easy API"** บนหน้ายืนยันการชำระเงิน (Payment Confirmation)

### 2.5 การจัดทำและเผยแพร่กรณีศึกษา
ภายหลังการเปิดตัวอย่างเป็นทางการ ผู้ให้บริการ API มีสิทธิในการจัดทำและเผยแพร่กรณีศึกษา (Case Study) หรือเรื่องราวความสำเร็จ (Success Story) ที่เกี่ยวข้องกับการใช้งานระบบของพันธมิตรในรูปแบบบทความ วิดีโอ เสียงสัมภาษณ์ โดยพันธมิตรด้านการเชื่อมต่อระบบ ต้องสนับสนุนและให้ความร่วมมือในการดำเนินการให้สำเร็จลุล่วง ทั้งนี้ ต้องอนุญาตให้ ผู้ให้บริการ API เผยแพร่เนื้อหาผ่านช่องทางประชาสัมพันธ์ของผู้ให้บริการ API นอกจากนี้ ผู้ให้บริการ API จะต้องไม่เปิดเผยข้อมูลที่เป็นความลับของพันธมิตรเว้นแต่จะได้รับความยินยอมเป็นลายลักษณ์อักษรจากพันธมิตรล่วงหน้า

### 2.6 (หากมี) แต่เพียงฝ่ายเดียว
พันธมิตรด้านการเชื่อมต่อระบบตกลงรับชอบค่าธรรมเนียม ภาษีอากร และค่าใช้จ่ายต่างๆ ใดๆ ที่เกี่ยวข้องกับบริการ หากผู้ให้บริการ API จำต้องทดรองจ่ายค่าธรรมเนียม ภาษีอากร และ/หรือค่าใช้จ่ายดังกล่าวตามวรรคก่อนแทนพันธมิตรด้านการเชื่อมต่อระบบไปก่อน พันธมิตรด้านการเชื่อมต่อระบบตกลงชำระคืนให้แก่ผู้ให้บริการ API โดยพลัน

## ข้อ 3. คำรับรอง

### 3.1
พันธมิตรด้านการเชื่อมต่อระบบรับรองว่า บรรดาเอกสาร ข้อมูล รายละเอียดใดๆ ที่ปรากฏ และ/หรือที่มีการจัดส่งมาให้ผู้ให้บริการ API ไม่ว่าจะส่งมาในรูปแบบใดและไม่ว่าจะส่งด้วยตัวพันธมิตรด้านการเชื่อมต่อระบบเองหรือบุคคลที่พันธมิตรด้านการเชื่อมต่อระบบมอบหมายเป็นข้อมูลของพันธมิตรด้านการเชื่อมต่อระบบที่มีความครบถ้วน ถูกต้องแท้จริง และเป็นปัจจุบัน และเป็นข้อมูลที่ผู้ให้บริการ API สามารถนำมาใช้ในการให้ บริการตามบันทึกข้อตกลงฉบับนี้ และนำมาปรับปรุงข้อมูลในระบบของผู้ให้บริการ API ให้เป็นปัจจุบันได้ โดยพันธมิตรด้านการเชื่อมต่อระบบ มีสิทธิและมีความสามารถตามกฎหมายในการขอใช้บริการ และการทำธุรกรรมใดๆ ที่เกี่ยวข้องกับการใช้บริการตามบันทึกข้อตกลงฉบับนี้ โดยผู้ให้บริการ API ไม่มีหน้าที่ที่จะต้องตรวจสอบความครบถ้วน ถูกต้องแท้จริง และเป็นปัจจุบัน ของเอกสาร ข้อมูล และรายละเอียดต่างๆ ดังกล่าว ทั้งนี้ หากเกิดความเสียหายใดๆ เนื่องจากเอกสาร ข้อมูล หรือรายละเอียดดังกล่าวไม่ครบถ้วน ถูกต้องแท้จริง และเป็นปัจจุบัน หรือพันธมิตรด้านการเชื่อมต่อระบบไม่มีสิทธิหรือความสามารถตามกฎหมายในการขอใช้บริการ และการทำธุรกรรมใดๆ ที่เกี่ยวข้องกับการใช้บริการตามบันทึกข้อตกลงฉบับนี้ พันธมิตรด้านการเชื่อมต่อระบบตกลงเป็นผู้รับผิดชอบเพียงฝ่ายเดียว

### 3.2
พันธมิตรด้านการเชื่อมต่อระบบตกลงยินยอมให้ผู้ติดต่อหลักตามรายละเอียดการใช้บริการช่องทางการเชื่อมต่อ (API) ที่ระบุในเอกสารแนบท้ายหมายเลข 1 แนบท้ายบันทึกข้อตกลงฉบับนี้ (ถ้ามี) เป็นตัวแทนพันธมิตรด้านการเชื่อมต่อระบบในการแก้ไข เปลี่ยนแปลง และจัดส่งข้อมูลและ/หรือเอกสารใดๆ ของพันธมิตรด้านการเชื่อมต่อระบบและ/หรือที่เกี่ยวข้อง การรับรองสำเนาถูกต้องให้แก่ผู้ให้บริการ API การรับแจ้งข้อมูลและ/หรือเอกสารใดๆ ที่พันธมิตรด้านการเชื่อมต่อระบบแจ้งและ/หรือส่งให้แก่ผู้ให้บริการ API ทั้งนี้ พันธมิตรด้านการเชื่อมต่อระบบจะดำเนินการแจ้งการแก้ไขเปลี่ยนแปลงข้อมูลให้ผู้ติดต่อหลักยืนยันกลับมาทาง E-Mail Address ที่ระบุตามรายละเอียดการใช้บริการดังกล่าว โดยพันธมิตรด้านการเชื่อมต่อระบบยอมรับการกระทำของผู้ติดต่อหลักมีผลผูกพันพันธมิตรด้านการเชื่อมต่อระบบทุกประการ

### 3.3
พันธมิตรด้านการเชื่อมต่อระบบทราบดีว่า ในการให้บริการ Slip Verification API นี้ผู้ให้บริการ API เป็นเพียงผู้ให้บริการตรวจสอบข้อมูล รหัสชำระเงินที่ปรากฏบนหลักฐานการชำระเงิน (Slip) ที่พันธมิตรด้านการเชื่อมต่อระบบส่งให้ผู้ให้บริการ API ผ่านระบบของผู้ให้บริการ API และผ่านบริการ Slip Verification API เท่านั้น ผู้ให้บริการ API ไม่สามารถและไม่มีหน้าที่ในการตรวจสอบข้อมูลส่วนตัว ที่พิสูจน์และการยืนยันตัวตนของผู้ชำระเงิน ความครบถ้วน ถูกต้องแท้จริง หรือเป็นปัจจุบันของข้อมูลใดๆ ที่มีการรับส่งภายในการใช้บริการ Slip Verification API ในทุกกรณี รวมทั้งไม่ได้มีส่วนเกี่ยวข้องกับการซื้อ-ขายสินค้า/บริการระหว่างผู้ชำระเงิน กับพันธมิตรด้านการเชื่อมต่อระบบและ/หรือพันธมิตรด้านการเชื่อมต่อระบบ โดยการซื้อ-ขายสินค้า/บริการระหว่างผู้ชำระเงิน กับพันธมิตรด้านการเชื่อมต่อระบบ และ/หรือผู้ใช้บริการ เป็นไปตามข้อตกลงที่ผู้ชำระเงินได้ตกลงไว้กับผู้ให้บริการ API และ/หรือ ผู้ใช้บริการเอง

## ข้อ 4. การเปลี่ยนแปลงเงื่อนไขการให้บริการ

### 4.1
หากเป็นการเปลี่ยนแปลงเงื่อนไขที่ส่งผลให้ผู้ให้บริการ API เกิดภาระหรือความเสี่ยงที่เพิ่มขึ้น การเปลี่ยนแปลงดังกล่าวจะมีผลเมื่อผู้ให้บริการ API ให้ความยินยอม

### 4.2
หากเป็นการเปลี่ยนแปลงเงื่อนไขอื่นพันธมิตรด้านการเชื่อมต่อระบบตกลงยินยอมให้ผู้ให้บริการ API มีสิทธิเปลี่ยนแปลงได้ตามที่ผู้ให้บริการ API เห็นสมควร โดยหากเป็นการเปลี่ยนแปลงที่มีผลกระทบต่อการใช้บริการของผู้ให้บริการ API (เช่น การปรับอัตราค่าธรรมเนียมการใช้บริการที่สอดคล้องกับต้นทุนที่เพิ่มขึ้น การเปลี่ยนแปลงช่องทางในการให้บริการ การเปลี่ยนแปลงวันถึงกำหนดชำระหนี้) ผู้ให้บริการ API จะส่งสารหรือแจ้งข้อมูลอันเป็นสาระสำคัญของการเปลี่ยนแปลงที่ชัดเจนให้ผู้ให้บริการ API ทราบล่วงหน้าไม่น้อยกว่า **30 วัน** หรือระยะเวลาอื่นตามที่กฎหมายกำหนด

### 4.3
หากมีกฎหมายและ/หรือกฎเกณฑ์กำหนดให้ผู้ให้บริการ API ต้องดำเนินการเกี่ยวกับการเปลี่ยนแปลงเงื่อนไขไว้เป็นการเฉพาะเป็นอย่างอื่น พันธมิตรด้านการเชื่อมต่อระบบตกลงยินยอมให้ผู้ให้บริการ API ปฏิบัติตามกฎหมายและ/หรือกฎเกณฑ์ดังกล่าวได้

บันทึกข้อตกลงฉบับนี้มีผลใช้บังคับตั้งแต่วันที่ **1 เมษายน 2569 ถึงวันที่ 30 มีนาคม 2570** ในกรณีที่บันทึก ข้อตกลงฉบับนี้ครบกำหนดระยะเวลาข้างต้น และคู่สัญญาฝ่ายใดฝ่ายหนึ่งไม่มีการแจ้งบอกเลิกบันทึกข้อตกลงฉบับนี้ต่อคู่สัญญาอีกฝ่ายหนึ่ง ตามระยะเวลาที่กำหนดในบันทึกข้อตกลงฉบับนี้ ให้ถือว่ามีการต่ออายุบันทึกข้อตกลงฉบับนี้ออกไปอีกคราวละ **1 (หนึ่ง) ปี** โดยให้ข้อกำหนดและเงื่อนไขเป็นไปตามบันทึกข้อตกลงฉบับนี้ เว้นแต่จะมีการตกลงเพิ่มเติม/เปลี่ยนแปลงร่วมกันเป็นลายลักษณ์อักษรเป็นอย่างอื่น

## ข้อ 6. การระงับ/ยกเลิกการให้บริการ/ผลการสิ้นสุดการให้บริการ

### 6.1
พันธมิตรด้านการเชื่อมต่อระบบตกลงยินยอมให้ผู้ให้บริการ API มีสิทธิระงับ และ/หรือยกเลิกการให้บริการภายใต้บันทึกข้อตกลงนี้ได้ไม่ว่าทั้งหมดหรือบางส่วน หรือเฉพาะแต่ผู้ให้บริการ API รายใดรายหนึ่ง หรือเมื่อใดก็ได้ โดยผู้ให้บริการ API จะแจ้งให้พันธมิตรด้านการเชื่อมต่อระบบทราบล่วงหน้า เว้นแต่ในกรณีดังต่อไปนี้ พันธมิตรด้านการเชื่อมต่อระบบตกลงยินยอมให้เป็นดุลยพินิจของผู้ให้บริการ API ในการพิจารณาระงับ และ/หรือยกเลิกการให้บริการภายใต้บันทึกข้อตกลงฉบับนี้ ไม่ว่าทั้งหมดหรือบางส่วนได้ทันทีตามที่ผู้ให้บริการ API เห็นสมควรโดยไม่ต้องแจ้งให้ฝ่ายทราบล่วงหน้า และผู้ให้บริการ API ไม่ต้องรับผิดชอบในความเสียหายใดๆ ที่เกิดจากการดำเนินการตามข้อนี้ (หากมี)

**6.1.1** ข้อมูล รายละเอียด คำรับรอง หรือคำยืนยันใดๆ ที่ผู้ให้บริการ API ให้ไว้ ไม่เป็นความจริง ไม่ถูกต้องหรืออาจก่อให้เกิดความเข้าใจผิดในสาระสำคัญ

**6.1.2** ผู้ให้บริการ API ตรวจพบว่า ระบบ/อุปกรณ์ของพันธมิตรด้านการเชื่อมต่อระบบที่ใช้เชื่อมต่อกับบริการ Slip Verification API มีความเสี่ยงที่จะเกิดความเสียหายอันเนื่องมาจากการดัดแปลง แก้ไข และ/หรือ เกิดจากระบบของระบบ/อุปกรณ์ดังกล่าว

**6.1.6** พันธมิตรด้านการเชื่อมต่อระบบต้องปฏิบัติตามกฎหมาย ระเบียบ ข้อบังคับ หรือคำสั่งและ/หรือคำขอความร่วมมือศาล หรือผู้มีอำนาจตามกฎหมาย หรือหน่วยงานที่กำกับดูแลผู้ให้บริการ API

## ข้อ 7. ข้อตกลงอื่น ๆ

### 7.1
บันทึกข้อตกลงฉบับนี้ ไม่ถือว่าเป็นการก่อให้เกิดนิติสัมพันธ์ระหว่างคู่สัญญาทั้งสองฝ่ายในลักษณะ ห้างหุ้นส่วน กิจการร่วมค้า ตัวการตัวแทน แต่อย่างใดทั้งสิ้น และไม่ถือว่าเป็นการมอบอำนาจให้คู่สัญญาฝ่ายใด ในอันที่จะก่อให้เกิดภาระหน้าที่ ความรับผิด และ/หรือ หนี้สิน แทนคู่สัญญาอีกฝ่าย ในอันใดขนาด

### 7.2
คู่สัญญาทั้งสองฝ่ายต่างยอมรับว่า ตนได้เข้าทำบันทึกข้อตกลงนี้ด้วยความสมัครใจ คู่สัญญาต่างฝ่ายต่างยอมรับในค่าใช้จ่ายที่เกิดขึ้นแต่ลดจากการดำเนินการตามบันทึกข้อตกลงฉบับนี้ด้วยตนเอง

### 7.3
การล่าช้า หรืองดเว้นใดๆ ในการใช้สิทธิของผู้ให้บริการ API ตามกฎหมายหรือตามเงื่อนไขต่างๆ ตลอดจนคู่มือระเบียบ และบันทึกข้อเสียงของผู้ให้บริการ API ในระบบโทรศัพท์ ไม่ถือว่าผู้ให้บริการ API สละสิทธิ หรือความยินยอมในการดำเนินการใดๆ แก่ผู้ให้บริการ API แต่ประการใด

### 7.4
ถ้าในเวลาใดข้อกำหนดและเงื่อนไขการใช้บริการภายใต้บันทึกข้อตกลงฉบับนี้ กลายเป็นโมฆะ ไม่ชอบด้วยกฎหมาย ไม่สมบูรณ์ หรือใช้บังคับมิได้ในประการใด ให้ข้อกำหนดและเงื่อนไขข้ออื่น ที่เหลือ (แล้วแต่กรณี) ยังคงขอบด้วยกฎหมาย สมบูรณ์ และใช้บังคับได้ตามกฎหมาย และไม่เสียไปเพราะความเป็นโมฆะ ไม่ชอบด้วยกฎหมาย ไม่สมบูรณ์ หรือใช้บังคับมิได้ของข้อกำหนดและเงื่อนไขดังกล่าวนั้น

### 7.5
บรรดาหนังสือ จดหมาย คำบอกกล่าว ข้อกำหนดและเงื่อนไข ข้อมูลใด ๆ ที่คู่สัญญาฝ่ายหนึ่งได้ส่งให้แก่คู่สัญญาอีกฝ่าย ไม่ว่าจะโดยส่งเอง หรือทางไปรษณีย์ลงทะเบียนหรือไม่ลงทะเบียนไปยังที่อยู่ หรือส่งเป็น Email ไปยัง Email Address หรือส่ง Short Message Service (SMS) ไปยังหมายเลขโทรศัพท์เคลื่อนที่อื่นใด ที่ผู้ให้บริการ API ระบุในรายละเอียดการใช้บริการช่องทางการเชื่อมต่อ (API) ที่ระบุในเอกสารแนบท้ายหมายเลข 1 ที่แนบท้ายบันทึกข้อตกลงฉบับนี้ (กรณีของผู้ให้บริการ API) หรือที่ผู้ให้บริการ API ระบุไว้ในบันทึกข้อตกลงฉบับนี้ (กรณีของ) หรือผ่านบริการ/ช่องทางตามที่คู่สัญญาได้ตกลงกันไว้เป็นลายลักษณ์อักษร (ซึ่งต่อไปนี้ หากไม่กล่าวโดยเฉพาะเจาะจง จะรวมเรียกที่อยู่ Email Address หมายเลขโทรศัพท์เคลื่อนที่และบริการ/ช่องทางดังกล่าวว่า "ช่องทางรับข้อมูล") ให้ถือว่าได้รับโดยขอบแล้ว ทั้งนี้ โดยไม่จำต้องคำนึงถึงว่าจะมีผู้รับไว้หรือไม่ และแม้ว่าส่งให้ไม่ได้เพราะช่องทางรับข้อมูลถูกย้ายหรือถูกเปลี่ยนแปลงไปหรือถูกรื้อถอนโดยคู่สัญญาฝ่ายใดไม่แจ้งการย้าย การเปลี่ยนแปลง หรือการรื้อถอนนั้นให้คู่สัญญาอีกฝ่ายทราบก็ดี หรือส่งให้ไม่ได้เพราะหาช่องทางรับข้อมูลไม่พบก็ดี ให้ถือว่าคู่สัญญาได้รับและทราบหนังสือ จดหมาย คำบอกกล่าว หรือข้อมูลดังกล่าวแล้วโดยขอบ ทั้งนี้ หากมีการย้าย การเปลี่ยนแปลง หรือการรื้อถอนช่องทางรับข้อมูล คู่สัญญาฝ่ายนั้นจะต้องแจ้งให้คู่สัญญาอีกฝ่ายทราบเป็นลายลักษณ์อักษรล่วงหน้าไม่น้อยกว่า **30 (สามสิบ) วัน**

### 7.6
บันทึกข้อตกลงฉบับนี้จัดขึ้นเพื่อแสดงเจตนารมณ์ร่วมกันในการประสานความร่วมมือทางเทคนิคและเชิงพาณิชย์ โดยให้ใช้บังคับและตีความตามกฎหมายไทย หากมีข้อพิพาทเกิดขึ้นคู่สัญญาตกลงจะเจรจาเพื่อหาข้อยุติร่วมกันก่อน หากไม่สามารถตกลงกันได้ภายใน **30 (สามสิบ) วัน** ให้นำข้อพิพาทนั้นเข้าสู่กระบวนการพิจารณาของศาลไทยที่มีเขตอำนาจ

# เอกสารแนบท้ายหมายเลข 2
## วิธีการเรียกเก็บค่าธรรมเนียมการใช้บริการ Slip Verification API

พันธมิตรด้านการเชื่อมต่อระบบตกลงยินยอมปฏิบัติตามข้อกำหนดและเงื่อนไขในการชำระค่าธรรมเนียมตามรายละเอียด ดังนี้:

### 1. การเรียกเก็บรายเดือน
พันธมิตรด้านการเชื่อมต่อระบบตกลงชำระค่าธรรมเนียมการใช้บริการ Slip Verification API เป็นรายเดือนตามรอบปฏิทิน ให้แก่ผู้ให้บริการ API โดยเริ่มคิดค่าธรรมเนียมการใช้บริการ (รวมภาษีมูลค่าเพิ่ม) ตั้งแต่วันที่ **1 เมษายน 2569** เป็นต้นไป

กรณีที่พันธมิตรด้านการเชื่อมต่อระบบมีความประสงค์จะเปลี่ยนแปลงแพ็กเกจการใช้บริการ พันธมิตรด้านการเชื่อมต่อระบบจะต้องแจ้งความประสงค์เป็นลายลักษณ์อักษร ตามแบบคำขอเปลี่ยนแปลงแพ็กเกจการใช้บริการช่องทางการเชื่อมต่อ (API) ที่ผู้ให้บริการ API กำหนด ลงนามโดยผู้มีอำนาจกระทำการแทนพันธมิตรด้านการเชื่อมต่อระบบ และนำส่งมายังผู้ให้บริการ API ล่วงหน้าอย่างน้อย **30 วันทำการ** โดยการเปลี่ยนแปลงจะมีผลวันที่ 1 ของเดือนที่ถัดจากวันที่ครบกำหนด 30 วันทำการของพันธมิตรด้านการเชื่อมต่อระบบแล้ว เช่น หากแจ้งมายังผู้ให้บริการ API วันที่ 15 มกราคม การเปลี่ยนแปลงจะเริ่มมีผลในวันที่ 1 มีนาคม เป็นต้น

### 2. การแจ้งสรุปรายเดือนและออกใบแจ้งหนี้
ผู้ให้บริการ API จะจัดส่งสรุปรายละเอียดการใช้บริการและสรุปค่าธรรมเนียมการใช้บริการของแต่ละเดือนไปยัง E-Mail Address ของผู้ติดต่อหลักตามที่ระบุในเอกสารแนบท้ายหมายเลข 1 รายละเอียดการใช้บริการช่องทางการเชื่อมต่อ (API) สำหรับบริการตรวจสอบหลักฐานการชำระเงิน (Slip Verification) ภายใน **7 (เจ็ด) วันทำการ**ของผู้ให้บริการ API ใน เดือนถัดจากเดือนที่มีการใช้บริการ ทั้งนี้ หากพันธมิตรด้านการเชื่อมต่อระบบเห็นว่าค่าธรรมเนียมการใช้บริการที่ผู้ให้บริการ API แจ้งไม่ถูกต้อง พันธมิตรด้านการเชื่อมต่อระบบจะต้องแจ้งรายการที่ถูกต้องกลับมายังผู้ให้บริการ API ภายใน **3 (สาม) วันทำการ**ของพันธมิตรด้านการเชื่อมต่อระบบ นับแต่วันที่ผู้ให้บริการ API ส่งสรุปรายละเอียดการใช้บริการและสรุปค่าธรรมเนียมการใช้บริการของเดือนนั้น หากพ้นระยะเวลาดังกล่าวแล้ว ถือว่าอัตราค่าธรรมเนียมการใช้บริการนั้นถูกต้องและผู้ให้บริการ API จะจัดส่งใบแจ้งหนี้ไปยัง E-Mail Address ของผู้ติดต่อหลักต่อไป

### 1. บทนำและพันธสัญญาการให้บริการ (Introduction & Commitment)
เอกสารฉบับนี้จัดทำขึ้นเพื่อกำหนดข้อตกลงระดับการให้บริการ (SLA) ของทาง Easy Slip เพื่อให้มั่นใจถึงคุณภาพ ประสิทธิภาพ และความต่อเนื่องของบริการตรวจสอบข้อมูลสลิปโอนเงิน (Slip Verification Service) เราตระหนักดีว่าในโลกธุรกิจดิจิทัล การตรวจสอบยอดเงินที่แม่นยำและรวดเร็วเป็นปัจจัยสำคัญต่อการขับเคลื่อนธุรกิจของท่าน เราจึงมุ่งมั่นที่จะส่งมอบบริการที่มีเสถียรภาพสูงสุด พร้อมทีมงานผู้เชี่ยวชาญที่พร้อมสนับสนุนและแก้ไขปัญหาอย่างมืออาชีพ เพื่อให้ท่านดำเนินธุรกิจได้อย่างไร้กังวล

### 2. การรับประกันความเสถียรภาพของระบบ (System Uptime Guarantee)
- **ระดับความพร้อมใช้งาน (Uptime Guarantee):** เรารับรองความพร้อมใช้งานของระบบ API ที่ระดับ **99.9% ต่อเดือน**
- **หมายเหตุ:** ตัวเลขนี้ไม่รวมถึงช่วงเวลาการปิดปรับปรุงระบบตามแผน (Scheduled Maintenance) ซึ่งจะมีการแจ้งเตือนล่วงหน้าอย่างน้อย **3 วันทำการ**

การรันตีนี้สะท้อนถึงโครงสร้างพื้นฐาน (Infrastructure) ที่เราออกแบบมาอย่างรัดกุม มีระบบสำรอง (Redundancy) และการเฝ้าระวังตลอด 24 ชั่วโมง เพื่อให้การทำธุรกรรมของลูกค้าท่านไม่สะดุด

### 3. ระดับความรุนแรงของปัญหาและระยะเวลาการให้บริการ (Severity Levels & Response Time)

เมื่อเกิดเหตุขัดข้อง เรามีการจัดลำดับความสำคัญของปัญหา (Incident Priority) เพื่อให้มั่นใจว่าปัญหาที่ส่งผลกระทบต่อธุรกิจหลักจะได้รับการแก้ไขเร็วที่สุด โดยแบ่งเกณฑ์ดังนี้:

| ระดับความรุนแรง (Severity Level) | นิยามและตัวอย่างปัญหา (Definition & Examples) | ระยะเวลาตอบกลับ (Response Time) | ระยะเวลาแก้ไข (Target Resolution Time) |
|---|---|---|---|
| **High (วิกฤต)** | ปัญหาร้ายแรงที่ทำให้ใช้บริการไม่ได้ทั้งหมด ส่งผลกระทบต่อธุรกิจของลูกค้าทันที ตัวอย่าง:<br>- ระบบ API ล่ม ไม่สามารถเชื่อมต่อได้ (HTTP 500)<br>- ระบบไม่สามารถอ่านรูปสลิปโอนเงินได้ทุกรายการ<br>- ข้อมูลในระบบฐานข้อมูลสูญหายหรือไม่ถูกต้องในวงกว้าง | ภายใน **15 นาที** | ภายใน **4 ชั่วโมง** |

### 5. ข้อยกเว้นความรับผิดของผู้ให้บริการ

**5.1 ความขัดข้องจากระบบภายนอกที่อยู่นอกเหนือการควบคุม (Third-Party Dependency)**
เนื่องจากบริการตรวจสอบสลิปโอนเงินต้องมีการเชื่อมต่อข้อมูลกับสถาบันการเงินโดยตรง บริษัทขอสงวนสิทธิ์ไม่นับรวมช่วงเวลา Downtime ที่เกิดจาก สาเหตุดังต่อไปนี้:
- การปิดปรับปรุงระบบ หรือความขัดข้องทางเทคนิคของ ระบบธนาคาร (Bank System) หรือแอพพลิเคชัน Mobile Banking ของธนาคารต้นทาง
- ความขัดข้องของโครงข่ายอินเทอร์เน็ตสาธารณะ (Public Internet Backbone) หรือผู้ให้บริการอินเทอร์เน็ต (ISP) ฝั่งลูกค้า

**5.2 การบำรุงรักษาระบบตามแผน (Scheduled Maintenance)**
ช่วงเวลาที่บริษัทได้แจ้งเตือนลูกค้าล่วงหน้าเพื่อทำการบำรุงรักษา ปรับปรุงประสิทธิภาพ หรืออัปเดตระบบความปลอดภัย (โดยปกติจะแจ้งล่วงหน้าอย่างน้อย **3-5 วันทำการ** และมักดำเนินการในช่วงเวลาที่มีการใช้งานน้อยที่สุด)

**5.3 ความผิดพลาดจากการใช้งานของผู้ใช้บริการ (Client-Side Issues)**
ความขัดข้องที่เกิดจากอุปกรณ์ ซอฟต์แวร์ หรือการกระทำของลูกค้าเอง เช่น:
- การส่งข้อมูล API Request ที่ผิดรูปแบบ (Bad Request / Invalid Format)
- ปัญหาจากระบบ Firewall หรือ Network Configuration ภายในของลูกค้าเองที่ปิดกั้นการเชื่อมต่อ
- การนำ Access Token หรือ API Key ไปใช้อย่างไม่ถูกต้อง หรือถูกระงับสิทธิ์เนื่องจากปัญหาด้านความปลอดภัย

**5.4 เหตุสุดวิสัย (Force Majeure)**
เหตุการณ์ที่ไม่สามารถคาดการณ์หรือควบคุมได้ เช่น ภัยธรรมชาติ อัคคีภัย ภัยสงคราม การจลาจล การเปลี่ยนแปลงกฎระเบียบของรัฐบาล หรือเหตุการณ์อื่นใดที่มีผลกระทบในวงกว้างและอยู่นอกเหนือความรับผิดชอบของบริษัท
