## 2. โครงสร้างที่สังเกตได้

### ระดับ SME (Start → Silver)
- ราคา/สลิปสูง (0.133–0.247) → GP กว้าง แม้คิดที่ cost 0.11
- **Start ทำกำไรดีที่สุด** (56%) เพราะ premium pricing สำหรับลูกค้าเล็ก
- ใช้เป็น loss leader ดึงลูกค้าเข้าก่อนได้

### ระดับกลาง (Gold → Premium-1)
- ราคา/สลิปกดลงเหลือ 0.111–0.114
- **GP บางมาก (1–4%)** ถ้า Others 100% → margin จะหายทันทีถ้า traffic mix แย่
- **ต้องตรวจ mix ก่อนรับลูกค้า**

### ระดับใหญ่ (Premium-2 → Premium-4)
- ราคา/สลิป 0.093–0.100 → **ต่ำกว่า cost 0.11**
- **ขาดทุนถ้าลูกค้าใช้ Others 100%** → ห้ามรับโดยไม่ lock mix
- ออกแบบสำหรับลูกค้าที่ traffic เป็น K-source เป็นหลัก (เช่น ร้านที่ลูกค้าโอนจาก KBank app)

## 4. ข้อสังเกตเปรียบเทียบ API vs BOT

| ประเด็น | EasySlip **API** | EasySlip **BOT** |
|---|---|---|
| ลูกค้าเป้าหมาย | Developer / business ที่ทำเอง | ร้านค้า/SME ใช้บอทตรวจให้ |
| Pricing ระดับเริ่มต้น | Start ฿99/250 สลิป (0.40 บ./slip) | Start ฿99/400 สลิป (0.247 บ./slip) |
| Pricing ระดับสูงสุด | Premium 3 ฿40,000/320,000 (0.125) | Premium-4 ฿14,000/150,000 (0.093) |
| 10 สาขา | ❌ ไม่ระบุ | ✅ ทุกแพ็ก |
| ระดับ Premium ขาดทุน @ 0.11 | Premium 1–3 (margin บาง) | **Premium-2 ถึง Premium-4 ติดลบ** |
| Value-add | API ดิบ | บอท + จัดการสาขา + UI พร้อมใช้ |

→ BOT **ราคา/สลิปถูกกว่า API ในระดับ enterprise** เพราะถือว่าลูกค้า BOT มี mix ดีกว่า (สแกนผ่านแอปธนาคารเอง)
→ API ตั้งราคาสูงกว่าได้เพราะลูกค้า "ไม่รู้/ไม่ควบคุม" mix

## 5. Sales Notes

### ทำได้เลย
- **Start–Silver** ขายตรงได้ไม่ต้องเช็ค mix — ปลอดภัยทุกกรณี
- **Gold–Premium-1** ขายได้ แต่บอก sales: "ดูดีกว่าถ้าลูกค้าใช้ KBank app เป็นหลัก"

### ต้องระวัง
- **Premium-2 ขึ้นไป**: ต้องเช็ค traffic mix ลูกค้าก่อน
  - ถ้า Others > 50% → margin ติดลบหรือบางมาก → ปฏิเสธหรือ upsell
  - ถ้า K-source ≥ 70% → ปล่อยขายได้ ทำกำไร 41%+

### คอมเซลส์ (proposed — ใช้ pattern เดียวกับ API)
- GP% ≥ 60% → 15% (Start, Basic, Starter)
- 50–60% → 12% (Beginner, Silver, Gold, Diamond, Premium-1)
- 40–50% → 7% (Premium-2, Premium-3, Premium-4)

ต้องการ Excel commission สำหรับ BOT แบบเดียวกับ API ให้สร้างต่อได้

### Quick-quote ลูกค้าที่ถามรอบ 3 / 6 เดือน
- บอกตรงๆ ว่า "ราคา × N ไม่มีส่วนลดทั้ง 2 รอบ"
- จุดขายของรอบยาวคือ **ไม่ต้องมาต่ออายุบ่อย** + **สลิปสะสมได้** (3/6 เดือน)
- ถ้าลูกค้ายื้อเรื่องส่วนลด → upsell tier ใหญ่ขึ้นแทน หรือ bundle BOT + API

### ความเสี่ยงเฉพาะรอบยาว (Premium-2 ขึ้นไป)
- ขาดทุน Others 100% **คูณตามรอบ** — ติดสัญญายาว = เสียหายหนักขึ้น
  - Premium-4: 1mo −฿2,500 / 3mo −฿7,500 / 6mo −฿15,000 / **12mo −฿30,000**
- ถ้าจะปิด Premium-2+ รอบ 6/12 เดือน ต้อง **lock K-source ≥ 50% ใน contract** ไม่งั้นเสียเปล่าทั้งปี
- เสนอเงื่อนไข: ถ้า K-src < 50% → EasySlip มีสิทธิ์ปรับราคาให้สะท้อน cost จริง หรือ terminate contract
