# Thunder Solution — บริบทธุรกิจ (ฉบับเปิดเผยได้)

## 1. บริษัทและกลุ่มธุรกิจ

- **บริษัท ธันเดอร์ โซลูชั่น จำกัด** (Thunder Solution Co., Ltd.) — ตั้งอยู่ที่จังหวัดขอนแก่น ประเทศไทย
- ธุรกิจหลัก: **บริการตรวจสอบสลิปโอนเงิน (slip verification)** + **LINE chatbot** สำหรับร้านค้าออนไลน์ / e-commerce ไทย
- เป็นหนึ่งในสองแบรนด์ของ **Thunder Group**:
  - **Thunder Solution** — โฟกัสลูกค้า enterprise / รายใหญ่
  - **EasySlip** — โฟกัสลูกค้า SMB (บริษัท อีซี่สลิป จำกัด ขอนแก่น)
- บริษัทในเครือ/สินค้าใต้กลุ่มเดียวกัน: EasyCRM (loyalty/CRM), BoostSMS (SMS marketing)

## 2. โมเดลธุรกิจ (เชิงหลักการ)

- รายได้มาจาก 2 ช่องทางหลัก:
  1. **Slip Verification API** — คิดตามการใช้งาน (per-slip)
  2. **LINE chatbot** — รูปแบบ subscription
- เก็บเงินแบบ **pre-paid billing cycle** (ลูกค้าเติมเงินล่วงหน้า)
- ต้นน้ำ: ตรวจสอบสลิปข้ามธนาคารผ่านระบบกลางของธนาคาร (เช่น ITMX / บริการ API ของธนาคาร) — ครอบคลุมสลิปหลายธนาคาร (กสิกร KBank, กรุงไทย KTB ฯลฯ)

## 3. สินค้า / บริการ (positioning เชิงคุณภาพ)

### Thunder BOT — LINE chatbot
- สินค้ารายได้หลักของแบรนด์ · ฐานลูกค้าใหญ่ที่สุดในกลุ่ม
- ARPU ต่ำ (ลูกค้าเยอะแต่จ่ายต่อรายน้อย)
- แนวโน้ม: "flatten–decline" — โตช้าลง/ทรงตัวถึงถดถอยช้า
- โจทย์เชิงกลยุทธ์ที่เปิดอยู่: ตัดสินใจ **pivot หรือ harvest**

### Thunder API — Slip Verification API
- **ดาวรุ่ง / growth vector หลักของแบรนด์** — โตแรงสุดในกลุ่ม (สัญญาณ Product-Market Fit แบบ S-curve)
- ลูกค้าเป็นกลุ่ม VIP · ARPU สูง
- โจทย์: validate ว่า PMF เป็นของจริงไม่ใช่ one-off + audit unit economics

### Thunder Flex
- สินค้าเสริม (minor product) · สัดส่วนรายได้เล็ก · แนวโน้มทรงตัว

### Thunder Corporate API (ขายผ่าน MOU)
- แพ็กเกจ **Enterprise** ที่ขายผ่านสัญญา MOU — ไม่ใช่แพ็กเกจมาตรฐานบนเว็บ
- แบ่งเป็นหลาย tier ตามปริมาณสลิป/เดือน · สัญญาเลือกได้ทั้งแบบรายเดือนและ 12 เดือน
- มี Sales Playbook สำหรับทีมเซลส์

### API Gen QR (แผนงาน)
- ฟีเจอร์สร้าง QR สำหรับรับชำระเงิน · ครอบทั้งฝั่ง Thunder API และ Easy API
- ช่วงวางแผน: รองรับพร้อมเพย์ก่อน · Phase 1 มี Dev Doc แล้ว
- (สถานะปัจจุบันควร verify ก่อนสื่อสารภายนอก)

## 4. Insight เชิงกลยุทธ์ (ไม่มีตัวเลข)

- **Thunder API คือเครื่องยนต์การเติบโต** — ทิศทางคือเร่ง API และผลักดันเป็นตัวนำ
- **Thunder BOT กำลังถดถอยช้า** ทั้งที่เป็นรายได้หลัก → ต้องตัดสินใจทิศทาง
- **Upsell path:** migrate ลูกค้า BOT → API
- **Re-engagement opportunity:** ฐานลูกค้าส่วนใหญ่ยัง inactive → โอกาสกระตุ้นการใช้งานซ้ำมีสูง
- ข้อควรจำเชิง ops: รอบบิลแบบ pre-paid ทำให้ margin ต้นเดือนดู "ติดลบ" เป็นเรื่องปกติ ไม่ใช่วิกฤต

## 5. งานบริการลูกค้า (CS) — โทนและแนวปฏิบัติสาธารณะ

### เสียง/โทนของทีม (voice signature)
- ลงท้าย "ค่ะ / นะคะ" เกือบทุกข้อความ (โทน CS สุภาพ)
- ใช้ 🙏 ท้ายชื่อแบรนด์บ่อย · เปิดหัวเรื่องด้วย ⚠️ / ✅ / 📢 / 📌 / ⏰
- เมื่อเป็นปัญหาจากธนาคาร / LINE OA / Cloudflare จะ **ยืนยันตรงๆ ว่า "ไม่ใช่ปัญหาที่ระบบเรา"**
- แนะนำทางเลี่ยงระหว่างระบบขัดข้อง: "ตรวจ SMS หรือประวัติธุรกรรมในแอปธนาคาร"
- ปิดเคสด้วย ✅ + ยืนยันใช้งานปกติ + เบอร์ติดต่อ
- วลีประจำ: "ขออภัยในความไม่สะดวกมา ณ ที่นี้ 🙏"

### ประเภทเคสที่ประกาศบ่อย
1. ธนาคารต้นทางส่งข้อมูลใน QR ผิด (KTB → ITMX)
2. สลิปกสิกร (KBank) แสดง "ไม่พบข้อมูล / หมดอายุ"
3. ช่วงธุรกรรมสูง (เช่น 19:00–20:00) ระบบธนาคารอัปเดตช้า
4. ระบบภายในล่าช้า / ปิดปรับปรุงระบบชั่วคราว
5. LINE OA ขัดข้อง · Cloudflare ขัดข้อง · Login ล่าช้า (แต่ตรวจสลิปยังทำงาน)

- ค้นจากแหล่งสาธารณะ: เว็บไซต์ทางการของบริษัท, หน้า Facebook / LINE OA, ข่าว/บทความ, ข้อมูลจดทะเบียนบริษัท (DBD), รีวิวจากลูกค้า, คู่แข่งในตลาด slip verification ไทย
